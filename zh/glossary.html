📖 术语表

用简单的话解释复杂的AI概念。如果您在课程中遇到不认识的词 — 在这里查找。

🔤 关键术语

API

「数据插座」— 程序之间互相连接的方式

如果网站A想从程序B获取数据，它就使用API。例如，当课程计算器从中央银行网站获取当前汇率。

开源 (Open-source)

源代码开放的软件 — 可以免费下载和使用

开发者公开了代码。任何人都可以下载、学习和修改。示例：n8n、LibreOffice。但是：支持和培训可能需要付费（如Red Hat）。

本地部署 (Self-hosted)

「在您自己的服务器上」— 程序在企业内部运行，不在云端

数据永远不会离开办公室。需要服务器和IT专家进行配置。许可证免费，但硬件和维护不是。

VPN

虚拟专用网络 — 远程工作的安全隧道

允许在家工作的员工安全地连接到企业网络。对于办公室的本地AI，需要局域网；对于远程工作 — 需要VPN。

防火墙 (Firewall)

网络屏障 — 防止不受欢迎的连接

就像入口处的守卫：只放行获得授权的人。由IT部门配置。

定时任务 (Cron / Schedule)

按计划执行任务：每天、每小时等

例如：「每周一上午9:00检查供应商价格。」在n8n中这叫Schedule，在编程中叫cron。

JSON

程序的数据格式 — 「文本形式的表格」

AI经常以JSON格式提供答案，然后转换为Excel/表格。对人类来说，它看起来像带花括号的文本。

CSV

表格格式 — 在Excel中打开

简单的文本格式，数据用逗号分隔。任何表格都可以保存为CSV格式。

LLM (大语言模型)

大语言模型 — 理解并生成文本的「AI大脑」

ChatGPT、Claude、Kimi — 都是LLM。通过在大量文本上训练，理解问题并给出相关回答。

Token (令牌)

AI的文本计量单位 — 大约¾个词

AI计算的不是词，而是token。1000个token ≈ 750个词。请求的成本取决于token数量。

RAG (检索增强生成)

搜索 + 生成 — AI先找到事实，再回答

AI不仅依靠记忆，还在您的文档中搜索信息，然后基于这些信息回答。降低了编造的风险。

向量数据库

AI按意义搜索，而不是按确切的词搜索的存储

如果您搜索「昂贵的供应商」，它也会找到「价格高」和「不便宜」。它理解上下文，而不仅仅是措辞。

提示词 (Prompt)

给AI的请求或指令 — 您在聊天中写的内容

提示词越精确，回答越好。好的提示词包含：角色、任务、格式、约束、示例。

AI代理 (Agent)

按照给定指令自主执行任务的程序

它不只是回答问题，还会行动：搜索、比较、发送邮件、填写文档。

幻觉 (Hallucination)

当AI自信地提供编造的信息时

可能编造不存在的法律、虚假引用或错误事实。始终验证关键数据。

微调 (Fine-tuning)

在您的数据上训练模型以完成特定任务

预训练模型被「调整」到您的文档、术语和风格。比正常使用更贵，但更精确。

边界情况 (Edge cases)

极端、非标准的情况 — 当一切不如预期时

例如：预算=0、空文件、供应商不回复。好的自动化也会考虑这些场景。

MCP (模型上下文协议)

AI连接外部系统的标准

允许AI代理访问您的数据库、CRM和邮件。就像不同设备的通用连接器。

本地部署 (On-premise)

「在现场」— 一切都在企业内部运行，不在云端

数据留在您的围墙内。安全，但维护成本更高。适合机密信息。

工作流 (Workflow)

自动化步骤序列

例如：收到邮件 → 提取数据 → 写入表格 → 发送通知。在n8n和Zapier中，工作流可以可视化创建。

Webhook

程序之间的「信号」— 即时通知

当事件发生时（例如新订单），系统A立即通知系统B。无需不断轮询。

⚙️ 参数、自动化与审计

上下文窗口

「模型的记忆」— AI一次能记住多少文本

如果文档比窗口长，模型会忘记开头。要分析50页的合同，需要大窗口（200K+ token）或分块技术。

温度 (Temperature)

AI的「创意水平」— 从严格到富有想象力

0.1 — 可预测，几乎确定性（用于合同、报告）。0.7 — 有创意，生成变体（用于头脑风暴、营销）。1.0+ — 可能开始编造。

嵌入 (Embedding)

「意义的数字指纹」— AI如何理解两个文本相似

每个文本片段被赋予一组数字（向量）。AI通过「数学距离」找到相关文档，即使用词完全不同。

分块 (Chunking)

「切成小块」— 将长文档分成部分

100页的合同被切成每块500 token的段落，以便每块都能放入模型内存。然后AI找到需要的块并组装答案。

解析 (Parsing)

「从混乱中提取结构」— 将非结构化数据转为表格

从供应商PDF目录中提取价格、货号、计量单位到Excel。或从邮件往来中提取日期、金额和地址。

SaaS (软件即服务)

通过互联网订阅的软件 — 无需安装，付费即用

ChatGPT、n8n Cloud、Zapier — 都是SaaS。通过浏览器访问，数据在供应商服务器上。方便，但对于机密文档需要本地替代方案。

评分 (Scoring)

积分评估 — 客观选择供应商或质量控制

供应商获得85/100 — 通过，60/100 — 淘汰。标准：价格、交期、声誉、证书、经验。AI可以按给定标准自动计算。

合规 (Compliance)

符合规则 — 法律、标准、内部规章

检查合同是否违法、估算是否符合标准、采购是否通过了所有审批。AI将文档与需求清单进行交叉核对。

SLA (服务等级协议)

服务等级协议 — 响应时间、可用性、罚金

「请求响应 — 2小时内，可用性99.9%，停机罚金 — 月费的1%。」AI可以监控SLA遵守情况并报告违规。

流水线 (Pipeline)

「传送带」— 数据处理的步骤链

步骤1：收到邮件 → 步骤2：提取数据 → 步骤3：写入CRM → 步骤4：发送通知。在n8n和Zapier中，流水线可以可视化构建，无需编程。

🏷️ 课程缩写

GENAi

基于OpenWebUI的企业内部服务

在您的服务器上运行（本地）。对机密文档安全。使用GENAi处理：合同、报告、个人数据、内部协议。

NDA (保密协议)

保密协议 — 不披露合同

与AI供应商签署，以确保他们不会使用您的数据训练模型。

ROI (投资回报率)

投资回报 — 投资的盈利能力

显示AI实施相对于成本能带来多少收益。ROI = (利润 − 成本) / 成本 × 100%。

CRM (客户关系管理)

客户关系管理系统

存储客户信息、交易、联系方式。示例：amoCRM、Bitrix24、Salesforce。

ERP (企业资源计划)

企业资源计划系统

统一会计、生产、采购和销售。示例：1C、SAP、Oracle。

🤖 代理系统术语

OpenClaw

开源框架，用于在自有服务器上运行AI代理

可创建数字员工，在Telegram、邮件或企业通讯工具中工作。数据不离开公司——代理在本地运行。使用配置文件（SOUL.md、SKILL.md、HEARTBEAT.md）定义行为。

Hermes Agent

自学习AI代理，来自Nous Research

能从经验中创建新技能（SKILL.md）并记住用户偏好。像资深员工一样，每完成一个任务就变得更「聪明」。

GoClaw

企业级多代理平台，支持数据隔离

允许不同部门拥有不同权限的代理。例如，财务代理看不到工程师的数据。适合大型制造企业。

SOUL.md

代理的「灵魂」文件——定义其性格和角色

例如：「你是一位经验丰富的工艺工程师。说话简洁，使用过程控制系统数据。没有事实不做假设」。这就像数字员工的岗位说明书。

SKILL.md

代理的技能文件——典型任务的分步指南

例如：「如何分析故障日志：1）从过程控制系统加载数据，2）按设备分组，3）找出重复故障，4）给出建议」。代理执行此类技能，如同执行规章制度。

HEARTBEAT.md

代理的周期性检查时间表

定义代理自动执行什么以及何时执行。例如：每天早上7:30检查开采计划与实际对比，偏差>5%时发送汇总。类似于值班表和例会安排。

MEMORY.md

代理的长期记忆——存储事实、决策和经验教训

代理将重要事实记录在此，以免会话间遗忘。例如：「3号炉在温度>1200°时产生次品」或「47号井在压力<15大气压时不稳定」。这就像资深员工的知识库。

AGENTS.md / USER.md / TOOLS.md

代理系统的配置文件——规则、用户档案和工具

AGENTS.md — 所有代理的通用规则（行为准则）。USER.md — 用户档案：他是谁、做什么、哪些操作无需请示。TOOLS.md — 可用工具清单：SAP、1C、Grafana、企业通讯工具、邮件。

企业通讯工具

企业内部的即时通讯平台

如钉钉、企业微信、飞书等。用于团队沟通、通知推送和AI代理的消息收发。数据不离开公司网络。

过程控制系统 (SCADA)

工业自动化中的数据采集与监控系统

实时收集传感器和设备数据：温度、压力、流量、设备状态。是分析代理和预测性维护的数据来源。

本地部署 (On-premise)

「在现场」— 程序和数据在企业内部运行，不在云端

对生产数据至关重要：冶炼参数、开采计划、故障日志。本地代理和模型运行在企业服务器上——数据不会传输给外部提供商。

向量搜索

按语义而非精确词汇进行搜索

AI将文本转化为多维空间中的「点」。意义相近的词语（「违约金」「罚款」「滞纳金」）在空间中靠近。当您提问时，AI寻找最近的点——即使表述不同也能找到相关文档。重要：不能通过在提示词中「开启」向量搜索——它在文档上传时自动工作，或需要IT部门配置RAG系统。
