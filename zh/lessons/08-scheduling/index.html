8.3 · 工作计划与资源：进度、人员、设备

🎯 无代码 · ✅ 交互式测验

🎯 为什么企业需要这个

施工组织员（工长、现场主管）每天解决：谁做什么、按什么顺序、什么在延误。AI帮助构建进度、优化资源、预测延误。

案例： AI分析了3个月进度并预警：5层混凝土工作将在塔吊从4层释放前开始。延误：2天。组织员将开工推迟2天——进度未受影响。

📚 学习内容

AI为施工组织员做什么

  • 📅 进度： 构建/调整网络图、关键路径
  • 👷 资源： 按项目分配班组、设备、材料
  • ⚠️ 风险： 延误、过载、资源冲突预警
  • 📊 报告： 施工日志、完工单、计划/实际对比
  • 🔮 预测： 按当前速度何时完工？
  • 📋 派工单： 基于进度自动生成

AI输入数据

数据格式AI分析内容
网络进度（MS Project、Excel）.mpp、Excel关键路径、资源、依赖关系
施工日志（日报）PDF、Excel、文本计划/实际、完成百分比、延误
估算/本地估算PDF、Excel工程量、工时、工期
资源可用性（班组、设备）Excel、ERP、SAP负载、冲突、可用性
天气/季节限制API、Excel室外作业天气预报

AI进度请求示例

进度提示词
"分析施工进度（附件）。

1. 找出关键路径（不能延误的工作）
2. 检查资源负载：是否有过载（2个班组在同一项目）？
3. 检查依赖：前项工作是否完成后才开始？
4. 与天气预报对比：哪些室外作业会遇雨？
5. 预测：按当前速度何时完工？

输出：
- 风险表：工作 | 风险 | 级别 | 建议
- 修正后的更新进度
- 下周派工单"

🛠️ 如何应用

场景：每周进度审查

流程
1. 将进度（MS Project/Excel）、周报、资源上传AI
2. 请求："检查进度。关键路径？风险？资源负载？"
3. AI分析并输出：
   - 关键路径（标红）
   - 风险：3.15塔吊过载、混凝土延迟到场
   - 建议：调整工作、增加班组
4. 工长调整进度
5. AI生成下周派工单
6. 发给班组长

计划/实际对比

AI每日对比：

  • 计划工程量vs实际完成
  • 计划开始/完成日期vs实际
  • 材料消耗vs定额
  • 工时vs计划

偏差>10%时——预警并建议。

👨‍💻 如何委派给开发人员

任务模板

"需要施工进度管理AI工具。

输入：
- 进度：MS Project / Excel（网络图）
- 施工日志：日报（PDF、Excel、文本）
- 资源：班组、设备、材料（Excel、ERP、SAP）
- 估算：本地估算（PDF、Excel）
- 天气：API（或Excel预报）

功能：
1. 分析关键路径
2. 检查资源负载（冲突、过载）
3. 计划/实际对比
4. 预测完工日期
5. 生成派工单
6. 风险预警（延误、天气、资源）

输出：
- PDF报告（周报）
- Excel表（更新进度）
- 邮件/短信通知（严重风险）

集成：[MS Project / ERP / SAP / Excel]
期限：[日期]"

✅ 结果验收检查清单

  • 关键路径正确？（与手工计算核对）
  • 风险真实？（在进度上检查2-3个）
  • 资源负载合理？（无>100%过载）
  • 完工预测现实？
  • 派工单与进度一致？
  • 格式方便日常使用？

📝 理解测验

问题1： 什么是关键路径？

A) 延误会导致整个项目延期的工序链
B) 去工地最长的路
C) 从仓库到工地的路线

问题2： AI分析进度需要什么数据？

A) 进度、施工日志、资源、估算、天气
B) 只有进度
C) 只有估算

问题3： 当计划/实际偏差>10%时AI做什么？

A) 预警并建议调整
B) 忽略——这很正常
C) 自动停工

🎯 课程要点

AI是施工的预测分析师。它看到人容易忽略的风险：资源过载、进度冲突、天气延误。不替代工长，但提供数据驱动的决策工具，而非凭感觉。
