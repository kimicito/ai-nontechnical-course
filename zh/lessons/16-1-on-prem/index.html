第16级 — 合规

16.1 内部任务：本地部署

⏱ 25分钟 · 面向合规官、律师、首席合规官
封闭式环境（本地部署）
本课程中的所有任务都涉及具体姓名、交易、交易对手、个人数据、举报者投诉、调查。本地部署是合规工作的默认规则。

为什么合规是最封闭职能之一

合规兼具多个职能的敏感性：像内部控制一样——存在漏洞地图和举报者风险；像律师一样——享有特权并与监管后果相关；像安全部门一样——调查具体人员。其特殊之处在于："具体调查了谁以及为什么"这一事实本身就是敏感信息，无论调查结果如何。

本地部署的强制适用场景：
  • 分析制裁限制对具体交易/交易对手的适用性
  • KYC/KYB——收集关于具体交易对手的信息（受益所有人、股权结构）
  • 交易分析以识别可疑活动迹象（金融数据）
  • 准备SAR/STR——向监管机构提交的可疑交易报告
  • 交易监控警报调查
  • 第三方反腐尽职调查（具体合作伙伴）
  • 礼品/招待费用分析（员工个人数据）
  • 腐败信号调查（与内部控制部门合作）
  • 热线投诉的初步分析（举报者身份泄露风险）
  • 调查材料整理、管理层报告
  • 董事会/审计委员会的合规报告
  • 回应监管机构问询
  • 准备外部合规审计（内部流程全貌）
  • 针对真实业务流程的合规风险评估（漏洞地图）

1. 交易制裁适用性分析

解释针对具体交易对手和交易的筛查结果——决定是否可以进行操作。

提示词示例（本地部署）
分析制裁限制对交易的适用性：

交易数据（机密）：
- 交易对手：[请填写名称]
- 交易对手所在国：[请填写]
- 商品原产国：[请填写]
- 商品/服务类型：[请填写]
- 金额：[请填写]
- 币种：[请填写]
- 支付方式：[请填写]

筛查结果（内部系统）：
- OFAC：[请填写]
- 欧盟：[请填写]
- 英国：[请填写]
- 国家名单：[请填写]

请完成：
1. 交易风险评估（高/中/低/禁止）
2. 适用的限制性措施
3. 需要进行的额外核查
4. 交易可行条件（如可行）
5. 合规委员会依据
6. 文档记录（需记录内容）

结果：有据可依的交易决策，节省2–3小时。

2. 交易对手KYC/KYB

收集和整理关于受益所有人、股权结构、关联方的信息。

提示词示例
对交易对手进行KYC/KYB分析：

交易对手：[请填写名称]

内部数据：
- 创始人：[请填写]
- 受益所有人：[请填写]
- 股权结构：[请填写]
- 关联方：[请填写]
- 变更历史：[请填写]

请完成：
1. 股权结构图（文字描述）
2. 最终受益所有人（UBO）
3. 关联方及其状态（PEP、制裁）
4. 风险评估（高/中/低）
5. 合作条件建议
6. 信息更新计划

结果：结构化的交易对手档案，节省3–4小时。

3. 交易可疑活动分析

解释交易监控警报，准备论证依据。

提示词示例
分析交易监控警报：

警报（内部系统）：
- ID：[请填写]
- 类型：[请填写：结构化/循环流动/交易量骤增]
- 账户：[请填写（如需要可匿名化）]
- 期间：[请填写]
- 金额：[请填写]
- 交易对手：[请填写]

账户历史：
- 月均交易额：[请填写]
- 典型交易对手：[请填写]
- 业务依据：[请填写]

请完成：
1. 可疑模式描述
2. 与账户典型行为对比
3. 可能解释（合法与非法）
4. 需要的额外数据
5. 建议（SAR/进一步分析/关闭）
6. 档案论证

结果：有据可依的警报结论，节省2–3小时。

4. SAR/STR准备

向监管机构提交的可疑交易报告——具有法律意义的文件。

提示词示例
准备SAR（可疑活动报告）：

操作数据（机密）：
- 操作日期：[请填写]
- 金额：[请填写]
- 币种：[请填写]
- 发送方：[请填写]
- 接收方：[请填写]
- 怀疑依据：[请填写]
- 关系历史：[请填写]
- 既往警报：[请填写]

请完成：
1. 可疑活动摘要
2. 交易详细描述（时间线）
3. 怀疑论证（为何认为可疑）
4. 关联方和账户
5. 既往历史（如有）
6. 公司已采取措施
7. 附件（文件清单）

格式：符合[请填写监管机构]要求

结果：完成的SAR，节省3–4小时。

5. 腐败信号调查

整理材料，为道德委员会准备结论。

提示词示例
对腐败信号进行调查：

信号来源：[请填写：热线/审计/内部控制]

信号内容：
[请填写要点（如为举报者则隐去身份）]

内部数据：
- 涉事人员：[请填写]
- 礼品/招待费用历史：[请填写]
- 交易对手：[请填写]
- 文件：[请填写]

请完成：
1. 事件时间线
2. 事实分析（哪些可确认，哪些不可）
3. 违规评估（违反了哪些政策）
4. 涉及人员
5. 建议（纪律/行政/刑事）
6. 整改措施计划
7. 道德委员会报告

结果：结构化的调查报告，节省4–5小时。

6. 热线投诉分析

对举报者投诉进行初步分类和优先级排序。

提示词示例
分析热线投诉：

投诉（匿名）：
[请插入投诉文本]

请完成：
1. 违规类别（腐败/金融舞弊/安全违规/歧视/其他）
2. 严重程度（严重/高/中/低）
3. 投诉中提及的部门/人员
4. 调查所需初步步骤
5. 风险（报复、泄露、声誉）
6. 调查期限建议
7. 相对于其他案件的优先级

注意：保护举报者匿名性。

结果：分类和行动方案，节省1–2小时。

7. 董事会合规报告

为最高管理机构准备结构化报告。

提示词示例
准备董事会合规报告：

期间：[请填写]

内部数据：
- 检查次数：[请填写]
- 发现的违规：[请填写]
- 已提交SAR/STR：[请填写]
- 制裁警报：[请填写]
- 热线投诉：[请填写]
- 调查：[请填写]
- 培训完成率：[请填写]%

请完成：
1. 执行摘要（1页）
2. 本期关键风险
3. 趋势（与上期对比）
4. 重大事件（匿名化）
5. 整改措施状态
6. 管理层建议
7. 下期计划

语气：专业、冷静、聚焦风险。

结果：完成的报告，节省3–4小时。

总结表

任务所需资料输出格式节省时间
交易制裁分析交易数据、筛查结果合规决策2–3小时

交易分析警报、账户历史警报结论2–3小时

知识测验

问题1：为什么举报者投诉的初步分析必须仅在本地部署进行？

存在举报者身份泄露和整个热线系统被渗透的风险
仅当月投诉少于10件时
如果匿名化投诉文本，可以在云端进行

问题2：SAR/STR适用哪种环境？

本地部署——具有法律意义的文件，包含机密数据
外部AI——监管机构公布了SAR要求
混合——结构在云端，事实在内部

问题3：为什么KYC/KYB必须在本地部署进行？

受益所有人和股权结构属于个人数据
公开登记信息人人可得，因此可以在云端进行
如果匿名化名称，可以在云端进行
