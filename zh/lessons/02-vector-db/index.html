2.2 · 向量数据库：AI如何"理解"语义

🎯 无代码 · ✅ 交互式测验

🎯 为什么企业需要这个

普通搜索查找精确词汇。如果你搜索"违约金"，而合同写的是"逾期罚款" — 普通搜索找不到。

向量搜索按语义查找。它理解"违约金"、"罚款"、"滞纳金"是同一个意思。

示例： 员工问"如何处罚逾期供应商？" — 向量搜索会找到关于"违约金"的章节，即使文档中没有"处罚"或"逾期"这个词。

📚 学习内容

普通搜索 vs 向量搜索

❌ 普通搜索

  • 查找精确词汇
  • "违约金" ≠ "罚款"
  • 遗漏同义词
  • 不理解上下文
  • 需要精确措辞

✅ 向量搜索

  • 按语义查找
  • "违约金" = "罚款" = "滞纳金"
  • 找到同义词
  • 理解上下文
  • 支持自然语言

工作原理（无数学）

类比： 想象每个词都是空间中的一个点。

  • "国王"和"女王" — 接近的点（亲属）
  • "国王"和"汽车" — 遥远的点（无关联）
  • "违约金"和"罚款" — 接近的点（同义词）

AI将你的问题转化为点，并在文档片段中寻找最近的点。

生活示例

💡 问题示例
"如果供应商迟到怎么办？"

普通搜索： 找不到任何内容（合同中没有"迟到"这个词）

向量搜索： 找到关于"违约金"、"交货逾期"、"罚款条款"的章节

商业应用场景

场景如何工作

👨‍💻 如何委派给开发人员

任务模板

要求：
1. 语义搜索（不仅精确匹配）
2. 支持中英文
3. 可扩展性：从1000到100000文档
4. 与[现有系统]集成
5. 响应时间 < 2秒

示例查询：[提供]
预期结果：[提供]

首选数据库：[Pinecone/Chroma/Weaviate — 或留给开发人员选择]
期限：[日期]"

✅ 结果验收检查清单

  • 搜索能找到同义词？（"违约金" = "罚款"）
  • 搜索理解上下文？（不同措辞的问题给出相同结果）
  • 响应时间可接受？（< 3秒）
  • 结果相关？（前3个答案有用）
  • 系统随文档增长可扩展？
  • 可按日期/文档类型过滤？

🎯 课程要点

向量搜索让AI理解语义，而非查找精确词汇。这对文档处理至关重要：员工用自己的话提问，AI找到正确答案。管理者不需要选择数据库 — 这是开发人员的任务。需要理解的是为什么需要它以及如何检查结果。
