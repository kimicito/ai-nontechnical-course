16.2 外部任务：外部AI

⏱ 20分钟 · 面向合规官
公开数据（外部AI）
本课程中的所有任务都涉及公开制裁名单、法规文件、公开框架、标准政策。可以使用Claude、ChatGPT、Gemini。

可以在外部AI中做什么

外部AI在以下场景中是安全的：以法规领域本身为工作对象——存在哪些名单、监管机构要求什么、方法论如何构建——在将其应用于真实姓名或交易之前。

主要规则：如果请求中没有具体姓名、公司名称、金额、账户号码、交易日期——可以使用外部AI。

1. 制裁制度监控

汇总OFAC、欧盟、英国、联合国和国家监管机构制裁名单的变化。

提示词示例（外部AI + 网络搜索）
分析过去一个月的制裁制度变化：

司法管辖区：OFAC（美国）、欧盟、英国、联合国

请完成：
1. 新加入名单的个人/组织
2. 行业制裁的变化
3. 商品/技术的新限制
4. 监管机构的解释（指导更新）
5. 趋势（值得关注的内容）
6. 对公司的普遍建议

格式：分析备忘录（1-2页）

结果：变化概览，节省3–4小时。

2. 出口管制结论（通用）

以通用形式分析出口管制规范的适用性，不涉及具体交易。

提示词示例
解释出口管制的适用性：

主题：[请填写：两用商品/技术限制/军事用途]
司法管辖区：[请填写：美国/欧盟/俄罗斯/中国]

请完成：
1. 适用的出口管制制度
2. 商品/技术类别
3. 许可要求
4. 例外和豁免
5. 许可获取程序
6. 实践建议（通用）
7. 法规依据链接

注意：不涉及公司具体交易。

结果：出口管制参考，节省2–3小时。

3. 基于标准框架的合规政策制定

基于国际标准（ISO 37001、英国《反贿赂法》、美国《反海外腐败法》）制定政策。

提示词示例
基于国际标准制定合规政策：

政策类型：[请填写：反腐败/反洗钱/制裁/利益冲突]
基础框架：[请填写：ISO 37001/英国《反贿赂法》/美国《反海外腐败法》指导]

请完成：
1. 政策结构（章节）
2. 主要条款（原则）
3. 员工义务（按层级）
4. 文件记录程序
5. 违规调查程序
6. 违规处罚
7. 审查周期
8. 附件（表格、检查清单）

格式：可供公司调整使用的现成文件。

结果：政策草稿，节省4–5小时。

4. 商业道德准则

基于最佳实践制定准则的通用部分。

提示词示例
制定商业道德准则：

公司：大型制造企业（B2B）
地域：国际业务

请完成：
1. 公司价值观（5-7条）
2. 与交易对手交往行为规范
3. 礼品和招待规则
4. 利益冲突（定义、示例）
5. 资产和信息保护
6. 职业道德（歧视、骚扰）
7. 报告违规的义务
8. 调查程序
9. 责任

语气：易懂、具体示例、不抽象。

结果：完成的准则，节省3–4小时。

5. 合规培训项目

准备通用内容的培训材料。

提示词示例
制定合规培训课程：

主题：[请填写：反洗钱/反腐败/制裁/利益冲突]
受众：[请填写：全体员工/管理层/财务部门]
格式：10页幻灯片 + 测试（5题）

请完成：
1. 培训目标
2. 基本概念（简单语言）
3. 员工义务
4. 警示信号（示例）
5. 怀疑时的处理方式
6. 测试（5道选择题）
7. 自查清单

语气：易懂、结合生活实例。

结果：培训现成材料，节省3–4小时。

6. 行业合规实践基准测试

基于公开数据与国际集团比较合规实践。

提示词示例（网络搜索）
进行合规实践基准测试：

行业：[请填写]
参数：
- 合规职能结构
- 技术应用（筛查自动化）
- 培训项目（每员工小时数）
- 举报者渠道
- 外部审计（频率）
- 公开报告（ESG/反腐败报告）

请完成：
1. 3-5家国际公司实践概览
2. 对比表
3. 最佳实践
4. 改进建议
5. 合规职能趋势

来源：公开报告、ESG披露、道德准则。

结果：行业实践概览，节省4–5小时。

知识测验

问题1：可以在外部AI中检查具体交易对手是否在制裁名单上吗？

可以，名单是公开的
不行，"检查过谁"这一事实本身属于敏感信息
如果匿名化名称，就可以

问题2：基于ISO 37001制定合规政策——适用哪种环境？

外部AI——框架是公开的
本地部署——政策是内部的
混合——结构在云端，调整在内部

问题3：合规实践基准测试——适用哪种环境？

外部AI——使用公开报告
本地部署——与竞争对手比较
混合——公开数据 + 内部分析
