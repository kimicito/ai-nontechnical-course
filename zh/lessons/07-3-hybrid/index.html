7.3 混合任务：数据匿名化

15分钟 · 面向采购专员 · 边界轮廓

有些任务处于边界状态：它们可以在云端完成，但前提是数据必须经过匿名化处理。将供应商名称替换为"供应商A"，删除具体价格——这样任务对外部模型来说就变得安全了。

在本课中：如何匿名化数据、哪些任务允许以混合模式执行，以及如何为开发人员准备自动化请求。

规则：先匿名化，再上云端

绝不能发送到云端的内容（即使部分信息）：
  • 报价单中的具体价格
  • 供应商名称（除非是西门子等巨头）
  • 特定物品的采购量
  • 合同条款（违约金、折扣、付款延期）
  • 谈判立场、BATNA、供应商弱点
匿名化后可以发送到云端的内容：
  • 报价结构比较（不含价格）——"供应商A vs 供应商B"
  • 对技术任务书响应完整性的分析（不含名称）
  • 角色扮演谈判排练（匿名场景）
  • 准备一般性论据（不含具体数字）

匿名化检查清单（5个步骤）

  1. 替换名称："PromTechSnab有限责任公司" → "供应商A"
  2. 将价格替换为相对值："1,200,000 ₽" → "基础价格+20%"或"价格X"
  3. 删除数量："50件" → "中等批量"
  4. 删除合同日期："至2025年12月31日" → "合同至年底"
  5. 检查：如果你将匿名化后的文本发送给同事——他们能理解内容，但不知道供应商是谁？如果是——就可以发送到云端。

1. 角色扮演谈判排练

采购专员正在准备复杂的谈判。他们需要排练场景。数据可以匿名化，云端模型可以作为"对手"使用。

示例提示（匿名化）
与我进行一次角色扮演谈判排练。

场景（匿名化）：
- 我是工业公司的采购专员
- 我需要降低设备供应的价格
- 我有2个替代供应商，价格低15%
- 当前供应商（供应商A）上次延迟了交货
- 我需要3周的交货期，他们的标准是6周
- 我愿意讨论数量，但不透露具体数字

你扮演供应商A的角色。你的目标是维持价格和交货期。
我的目标是降低价格并加快交货期。

开始谈判。像真正的供应商一样回应——
提出异议、反驳、试图转移话题。
结果：与"真实"对手进行15-20分钟的排练。采购专员练习论据，学习反驳异议，为实际对话做准备。

2. 报价结构比较（不含价格）

你需要了解哪个供应商对技术任务书的响应更完整，而不是谁更便宜。删除价格，只比较结构和完整性。

示例提示（匿名化）
按结构和响应完整性比较3份商业报价。

数据匿名化：
- 供应商A、供应商B、供应商C
- 价格已删除，仅保留相对值："低于市场"、"市场"、"高于市场"

按标准比较：
1. 对技术任务书的响应完整性（是否涵盖所有要点）
2. 技术描述的存在（是/否，质量）
3. 证书的存在（列表）
4. 保修条款（期限、范围）
5. 附加服务（安装、培训、维修）

不提及具体价格。给出完整性建议。

3. 准备一般性论据

采购专员需要一份一般性的谈判论据清单——不针对特定交易。云端模型生成基础内容，然后本地模型补充具体细节。

示例提示（匿名化）
我正在准备工业设备采购的谈判。

背景（匿名化）：
- 市场：3-4家大型供应商，竞争激烈
- 我的立场：大客户，定期采购
- 问题：供应商想提高价格和交货期

准备谈判的一般性论据：
1. 降价论据（市场、数量、竞争）
2. 加快交货期论据（计划、储备、违约金）
3. 对典型供应商异议的反驳
4. 坚定立场的礼貌表述

不含具体价格、名称或数量。
只提供模板，我会自行调整。

向开发人员请求匿名化自动化

如果采购专员经常手动匿名化文档——这可以自动化。以下是向IT团队请求的任务模板。

开发人员请求模板
任务：创建一个脚本/应用程序，用于在发送到云端LLM之前匿名化采购文档。

输入：商业报价文件（PDF、Word、Excel）。
输出：相同文件，但数据已替换。

替换规则：
1. 公司名称 → "供应商A"、"供应商B"等
2. 金额 → 相对值（"基础价格"、"+15%"、"-20%"）
3. 数量 → "小/中/大批量"
4. 日期 → 相对值（"本季度"、"至年底"）
5. 联系方式 → 完全删除
6. 合同编号 → "合同N"

附加：检查——输出文件不包含原始数据（按关键词搜索）。

集成：拖放文件 → 5秒内获得匿名化版本。
保存匿名化历史。

优先级：高（每天5-10名采购专员使用）。
💡 提示：即使IT无法构建完整的应用程序，你也可以从简单的Python脚本开始。它将替代80%的手工工作。然后逐步完善。

知识检测

问题1：以下哪项即使在匿名化后也不能发送到云端？

供应商名称 → "供应商A"
有效合同中的具体违约金条款
"价格高于市场10%"

问题2：匿名化检查清单的第一步是什么？

删除所有日期
将公司名称替换为"供应商A"、"供应商B"
删除文档中的所有数字

问题3：如何检查文档是否充分匿名化？

发送到云端，看系统是否阻止
给同事看——他们理解大意，但不知道供应商是谁？
删除所有数字和名称——这就够了

问题4：以下哪项是混合任务（匿名化 + 云端）？

检查合同与模板的偏差
使用匿名场景进行角色扮演谈判排练
在互联网上搜索新供应商

问题5：如果采购专员经常手动匿名化文档，应该怎么做？

准备向开发人员请求脚本自动化
继续手动操作——更可靠
将原始文件发送到云端——更快
