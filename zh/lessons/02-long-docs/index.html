2.3 · 长文档处理：限制与解决方案

🎯 无代码 · ✅ 交互式测验

🎯 为什么企业需要这个

合同、报告、招标文件 — 通常数百页。AI处理文本的容量有限制。

问题： 如果文档太长，AI读到末尾时"忘记"开头。或者干脆拒绝处理。

解决方案： 分块、RAG、具有大上下文的特殊模型。

📚 学习内容

模型限制（上下文窗口）

模型支持页数大致说明
GPT-4o-mini~50页一个中等合同
Claude Sonnet~200页长合同 + 附件
Kimi（月之暗面）~1000+页书籍、完整档案
Gemini 1.5 Pro~1500+页多本书籍

*大致数字。取决于文本密度。

处理长文档的3种方法

方法1：选择大上下文模型

简单方案：使用Kimi或Gemini处理长文档。

💡 何时使用
  • 文档 < 500页
  • 需要快速分析无需配置
  • 无敏感数据（云服务）

方法2：分块（Chunking）

将文档分成逻辑部分，逐部分分析。

💡 合同分块示例
第1部分：前言、合同标的
第2部分：双方权利义务
第3部分：价格与结算方式
第4部分：责任与违约金
第5部分：不可抗力与争议解决
第6部分：最终条款

每部分单独分析，然后合并结果。

方法3：RAG（检索增强生成）

使用向量搜索：AI找到相关片段并仅分析它们。

💡 何时使用
  • 数百份文档的档案
  • 需要语义搜索
  • 定期处理文档

🛠️ 如何应用

场景：招标文件分析（300页）

任务： 分析招标文件是否符合要求。

算法
  1. 将文档上传到Kimi或Gemini（支持大文件）
  2. 设定提示词："分析招标文件。提取供应商关键要求、评分标准、期限。以表格输出。"
  3. 如果回答被截断 — 追问："继续分析[章节名称]"

场景：合同档案分析（50份文档）

任务： 找出所有违约金高于1%的合同。

算法
  1. 将所有合同上传到RAG系统
  2. 提问："哪些合同规定逾期每天违约金超过1%？"
  3. 核对答案（AI会注明来源）

👨‍💻 如何委派给开发人员

任务模板

"需要长文档处理系统。

要求：
1. 支持文档多达[N]页
2. 自动分成逻辑部分
3. 部分间保持上下文
4. 可对整份文档提问
5. 注明来源（页码、章节）

文档类型：[合同 / 招标 / 报告]
数量：[N]份
平均大小：[N]页

首选方案：[指定或留给开发人员]
期限：[日期]"

✅ 结果验收检查清单

  • AI完整处理文档，无截断？
  • 章节间保持上下文？
  • 注明来源（页码、章节）？
  • 可对文档提出澄清问题？
  • 处理时间可接受？（100页 < 1分钟）
  • 支持PDF、DOCX、TXT格式？

🎯 课程要点

长文档不是问题，只要知道工具。Kimi和Gemini处理数千页。档案用RAG。中等文档用分块。关键是：始终检查AI没有遗漏重要章节。
