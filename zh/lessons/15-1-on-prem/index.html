第15级 — 法务职能

15.1 内部任务：本地部署

⏱ 25分钟 · 面向律师、公司秘书、合规专员
封闭回路（本地部署）
本课所有任务涉及律师保密义务、内幕信息、个人数据、公司战略规划。请使用内部模型或本地大语言模型。

为什么本地部署对律师至关重要

法务职能是默认最封闭的领域之一。律师保密义务、attorney-client privilege、交易和争议的内幕信息——所有这些都要求严格的访问控制。如果案件材料或争议中的谈判立场被法院或监管机构要求提供，使用外部云服务可能会引发保密性违规的问题，并在某些司法管辖区影响 privilege 的保护。

默认规则：如果材料与具体争议、交易、交易对手或员工相关——仅限本地部署。只能将涉及公开法律领域的工作放在外部回路中。

本地部署的强制场景：
  • 审查 incoming 合同并准备修改意见（真实商业条款、谈判策略）
  • 交易对手的尽职调查及并购交易（内幕信息，泄露=市场操纵）
  • 准备起诉状、答辩状、上诉状（律师保密义务、辩护策略）
  • 形成争议中的法律立场（证人/对方律师角色扮演演练）
  • 准备证据基础，系统化案件文件（具体争议）
  • 真实案件的索赔函往来（具体金额、违规行为、要求）
  • 公司文件（会议纪要、决议）——发布前的管理信息
  • 针对具体交易对手/交易的制裁合规筛查
  • 回复监管机构问询（战略敏感）
  • 针对具体业务问题的法律意见（真实事实、金额、风险）
  • 劳动争议，雇主立场（个人数据、潜在冲突）
  • 合同登记簿管理（完整商业关系清单）
  • 重组及设立子公司（战略敏感）

1. 审查 incoming 合同

将 incoming 合同与公司标准对比，识别风险条款，准备修改意见。

提示词示例（本地部署）
审查 incoming 合同是否符合公司标准：

Incoming 合同（文本）：
[请粘贴合同文本]

公司标准（核心条款）：
- 供应商责任：不低于批次价值的[填写]%
- 付款期限：不得早于交付后[填写]天
- 不可抗力：依据[填写法律]
- 索赔程序：[填写期限]天
- 仲裁地：[填写地点]

请完成：
1. 偏离标准条款的清单（注明条款编号）
2. 每项偏离的风险评估（高/中/低）
3. 建议的修改文本
4. 与交易对手谈判的论据
5. 替代措辞（折中方案）

结果：审查完毕的合同及修改意见，节省2–3小时。

2. 交易对手尽职调查

基于内部数据+公开来源分析交易对手，评估交易风险。

提示词示例
对交易对手[填写名称]进行尽职调查：

内部数据：
- 合作历史：自[填写年份]起，营业额[填写]，逾期[填写]
- 当前负债：[填写金额]
- 诉讼纠纷：[填写]
- 关联方：[填写]

公开数据（另行核实）：
- 国家企业登记：状态[填写]
- 仲裁案件：[填写]
- 制裁名单：[填写]

请完成：
1. 风险摘要（高/中/低）
2. 交易建议（同意/有条件同意/拒绝）
3. 最小化风险的必要条件
4. 预警信号（red flags）
5. 交易后的持续监控计划

结果：附带建议的风险分析，节省3–4小时。

3. 准备起诉状

构建立场、准备文件、角色扮演排练。

提示词示例
为以下争议准备起诉状：

背景：
- 原告方：[填写公司名称]
- 被告方：[填写]
- 争议标的：[填写：合同未履行、追讨欠款等]
- 主要事实：
  * [填写事实1]
  * [填写事实2]
  * [填写事实3]
- 证据：[填写清单]
- 诉求：[填写]
- 诉讼金额：[填写]

请完成：
1. 起诉状结构
2. 事实陈述（附合同引文）
3. 法律依据（条款、司法实践）
4. 附件清单
5. 角色扮演：法官典型问题及回答

结果：准备就绪的起诉状，节省4–5小时。

4. 制裁合规筛查

检查交易对手及交易中的制裁风险，解读结果。

提示词示例
分析交易的制裁风险：

交易数据：
- 交易对手：[填写名称]
- 交易对手所在国：[填写]
- 商品原产国：[填写]
- 目的国：[填写]
- 商品/服务类型：[填写]
- 金额：[填写]
- 币种：[填写]
- 付款方式：[填写]

筛查结果（内部系统）：
- OFAC：[填写结果]
- 欧盟：[填写结果]
- 英国：[填写结果]
- 国内名单：[填写结果]

请完成：
1. 交易风险评估（高/中/低）
2. 适用的限制性措施
3. 需进行的额外核查
4. 建议（批准/有条件批准/拒绝）
5. 合规委员会审议依据

结果：制裁风险分析，节省2–3小时。

5. 回复监管机构问询

准备对反垄断机构、税务机关或其他监管机构问询的结构化回复。

提示词示例
准备回复[填写监管机构]的问询：

问询：
[请粘贴问询文本]

原始数据（内部）：
- 事实：[填写]
- 文件：[填写清单]
- 法律立场：[填写]
- 风险：[填写]

请完成：
1. 回复结构（按问询要点逐一回复）
2. 每个要点的回复文本
3. 法律依据引用
4. 附件（清单）
5. 风险分析：监管机构后续可能追问什么
6. 管理层建议

结果：准备好的回复，节省3–4小时。

6. 法律意见书（legal opinion）

分析具体法律情况并为业务提供建议。

提示词示例
准备以下问题的法律意见书：

问题：
[填写具体问题]

事实（内部）：
- [填写事实1]
- [填写事实2]
- [填写事实3]

适用法律：[填写司法管辖区]

请完成：
1. 事实简述
2. 法律分析（适用条款）
3. 司法实践（类似案例）
4. 风险及其评估
5. 建议（行动方案）
6. 建议实施计划

结果：结构化的意见书，节省4–5小时。

7. 劳动争议 — 雇主立场

结合员工个人数据准备劳动争议立场。

提示词示例
准备劳动争议中雇主的立场：

员工数据（保密）：
- 职位：[填写]
- 工龄：[填写]
- 员工诉求：[填写]
- 纪律处分记录：[填写]
- 文件：[填写]

员工要求：
[填写]

请完成：
1. 要求的法律分析（合理性）
2. 雇主立场（论据）
3. 败诉风险（概率、金额）
4. 和解方案（诉讼/调解/庭前）
5. 策略建议

结果：准备就绪的立场，节省2–3小时。

汇总表

知识检验

问题1：为什么准备起诉状必须本地部署？

因为包含律师保密义务、争议策略和保密事实
仅在诉讼金额超过特定门槛时才需要
如果在云端匿名化处理当事人名称就可以

问题2：制裁筛查可以在云端进行吗？

可以，制裁名单是公开数据
不行——针对具体交易对手和交易的结果解读属于内部信息
混合模式——名单在外部，决定内部

问题3：为什么回复监管机构问询必须本地部署？

因为需要特殊格式的回复
因为揭示了公司的战略立场，可能影响当前争议
仅当要求24小时内回复时才需要
