第15级 — 法务职能

15.3 混合任务

⏱ 20分钟 · 面向律师
混合方法
通用法律逻辑、文件结构、标准论证在外部AI中准备。
真实事实、金额、当事人名称、战略立场在本地部署中添加。

律师的混合方法原则

法务职能需要特别谨慎：律师保密义务和 attorney-client privilege 在使用外部服务时可能受到威胁。混合方法允许在不违反保密性的前提下发挥两个回路的优势。

云端匿名化规则：
必须删除可以保留
当事人公司名称当事人类型（供应商、买方、雇主）
具体金额和价格损失计算方法
具体交易的日期和期限行业典型期限
员工、证人姓名参与者职位
案件编号、登记号案件类型（追讨、未履行）
争议中的战略立场通用法律逻辑

1. 法律意见书（混合模式）

云端：通用法律逻辑、适用条款、类似实践。
本地部署：真实事实、金额、具体业务风险。

步骤1：云端——法律逻辑
准备典型情况的法律分析：

情况（匿名化）：
[填写行业类型]行业的公司面临以下问题：
[填写通用法律问题]

司法管辖区：[填写]

请完成：
1. 适用法律条款（条文、法律）
2. 类似问题的司法实践（匿名化）
3. "支持"与"反对"的论证
4. 典型风险
5. 通用建议（不针对具体事实）

格式：法律分析（2–3页）
步骤2：本地部署——最终意见书
用真实数据完善法律意见书：

法律分析（来自云端）：
[粘贴云端结果]

真实事实：
- 公司：[填写名称]
- 交易金额：[填写]
- 交易对手：[填写]
- 关系历史：[填写]
- 具体风险：[填写]

请调整分析：
1. 将法律逻辑应用于真实事实
2. 添加具体数字（罚款、违约金、期限）
3. 评估风险概率
4. 提供具体优先建议
5. 行动计划（谁、做什么、何时）

节省时间：云端分析（2小时）+ 本地部署最终意见书（1.5小时）。

2. 起诉状 / 法律立场（混合模式）

云端：文件结构、该类型案件的典型论证。
本地部署：真实事实、证据、金额、当事人名称。

步骤1：云端——结构与论证
准备某类案件起诉状的结构：

案件类型：[填写：追讨欠款/合同未履行/劳动争议]
司法管辖区：[填写]

请编制：
1. 起诉状结构（逐条）
2. 原告典型论据
3. 被告典型反驳
4. 通常需要的证据
5. 通常适用的法律条款
6. 司法实践（典型法院做法）

[补充：行业、典型合同条件]
步骤2：本地部署——具体案件
将结构适配到具体案件：

案件：
- 原告：[填写]
- 被告：[填写]
- 标的：[填写]
- 诉讼金额：[填写]
- 事实：[填写]
- 证据：[填写]

结构（来自云端）：
[粘贴结构]

请编制：
1. 事实陈述（附日期、引用）
2. 法律依据（附具体条款）
3. 证据基础（附事实关联）
4. 诉讼金额计算（附公式）
5. 角色扮演：法官问题及回答

节省时间：云端结构（1.5小时）+ 本地部署案件（2小时）。

3. 尽职调查（混合模式）

云端：公开部分（登记册、媒体、司法数据库、典型风险）。
本地部署：结合内部风险评估、战略决策。

步骤1：云端——公开调研
对典型交易对手进行公开尽职调查：

交易对手类型：[填写：生产商/分销商/承包商]
行业：[填写]
国家：[填写]

公开来源：
1. 国家企业登记/类似登记
2. 仲裁案件（公开数据库）
3. 媒体与新闻
4. 制裁名单（公开）
5. 行业评级

请完成：
1. 公开核查清单
2. 行业典型 red flags
3. 风险评估方法论
4. 公开部分 DD 报告模板
步骤2：本地部署——内部评估
将结果与内部评估结合：

交易对手：[填写名称]
公开数据（来自云端）：
[粘贴结果]

内部数据：
- 合作历史：[填写]
- 当前负债：[填写]
- 关联方：[填写]
- 内部风险：[填写]

请完成：
1. 风险汇总表（公开+内部）
2. 最终评估（批准/有条件批准/拒绝）
3. 建议交易条件
4. 签约后监控计划

节省时间：云端公开部分（2小时）+ 本地部署内部评估（1.5小时）。

混合任务汇总表

任务云端本地部署节省时间
法律意见书法律逻辑、条款事实、金额、风险2 + 1.5小时
起诉状结构、典型论证具体案件、证据1.5 + 2小时
尽职调查公开登记册、媒体内部风险评估2 + 1.5小时

知识检验

问题1：法律意见书的混合原则是什么？

法律逻辑和条款在云端，真实事实和金额在本地部署
只要匿名化处理，整个法律意见书都可以在云端完成
法律意见书始终只能在本地部署

问题2：起诉状——哪些步骤在本地部署？

具体事实、金额、当事人名称、证据
文件结构和典型论据
适用法律条款

问题3：尽职调查——在放入云端前需要删除什么？

公司名称、交易金额、合同编号
交易对手类型（供应商/买方）
交易对手注册国家
