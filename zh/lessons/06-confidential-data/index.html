6.1 · 机密数据：什么不能发给AI

🎯 无代码 · ✅ 交互式测验

🎯 为什么企业需要这个

当您将文档发送给ChatGPT时，它可能成为模型训练数据的一部分。这意味着您的机密数据可能出现在其他用户的回复中。

真实案例： 员工将包含商业条款的合同发给了ChatGPT。一个月后，竞争对手从AI那里收到了类似的合同——AI"记起"了这些数据。损失：竞争优势丧失。

📚 学习内容

什么不能发给云端AI

类别示例
个人数据护照、身份证号、地址、电话
商业秘密价格、利润率、供应商、客户
银行信息账户信息、付款单据
保密协议(NDA)所有受NDA保护的内容
医疗数据诊断、病史

如何安全地使用AI

级别1：去标识化

发送前替换机密数据。

示例
原文："ABC公司，税号7701234567，价格50万元"
发送："公司A，税号XXXXXXXXXX，价格X元"

级别2：本地解决方案

在自己的服务器上运行AI。数据不离开公司。

  • Ollama — 本地运行模型
  • Llama 3、Mistral — 开源模型
  • 需要：GPU服务器

级别3：企业版合约

带有保密保证的企业版本。

  • ChatGPT Enterprise — 数据不用于训练
  • Microsoft Copilot — 在您的Azure环境中
  • Claude for Work — 企业协议

🛠️ 如何应用

发送前检查算法

  1. 文本中是否有个人数据？（姓名、身份证号、护照）
  2. 是否有商业秘密？（价格、客户、供应商）
  3. 是否有NDA数据？
  4. 如果任一回答"是" → 去标识化或使用本地解决方案

去标识化模板

去标识化前
合同编号45，日期2025.03.15
甲方：ABC科技有限公司，税号7701234567
乙方：XYZ建筑公司，税号7707654321
标的：设备供应，金额250万元

甲方：公司A，税号[XXXXXXXXXX]

✅ 结果验收检查清单

  • 所有员工知道什么不能发送？
  • 有去标识化流程？
  • 机密数据已配置本地解决方案？
  • 日志记录正常工作？（可查看历史）
  • 有自动泄露检查？
  • 政策已由所有员工签署？

📝 理解测验

问题1： 可以将带价格的合同发给免费版ChatGPT吗？

A) 可以，如果删除公司名称
B) 不可以，价格是商业秘密
C) 可以，ChatGPT是安全的

问题2： 什么是去标识化？

A) 将机密数据替换为占位符
B) 删除文档
C) 加密文件

问题3： 哪种保护级别最可靠？

A) 本地解决方案（数据不离开公司）
B) 云端AI的企业版
C) 去标识化

🎯 课程要点

保密不是可选项，而是义务。不要将个人数据、商业秘密和NDA内容未经保护地发送到云端AI。使用去标识化、本地解决方案或企业版合约。一次泄露可能让公司损失声誉和数百万资金。
