12.2 MRO外部任务

⏱ 15分钟 · 面向可靠性工程师和机械师
公开数据(外部AI)
Все задачи этого урока — с публичными источниками, общими методологиями, открытым рынком. Можно использовать Claude, ChatGPT, Gemini.

Почему облако лучше on-prem для ТОиР

Облачные модели выигрывают, когда нужно:

  • Веб-поиск: мониторить стандарты ТОиР, ГОСТ на оборудование, отраслевые требования
  • Перевод: работать с зарубежными каталогами запчастей, ISO, API
  • Генерация: создать обучающие материалы с нуля, типовые регламенты
  • Анализ публичных данных: LME на металлы, цены на оборудование, бенчмаркинг
  • Поиск аналогов: импортозамещение, кросс-номера запчастей
重要: перед отправкой в облако — проверь, что в запросе нет конкретных инвентарных номеров, данных о простоях, финансовых показателей завода. Если есть — обезличь или отправь в on-prem.

1. Разработка и актуализация графиков ППР

Общая методология планирования планово-предупредительных ремонтов — без привязки к конкретному заводу.

Пример промпта (внешний ИИ)
Подготовь методологию разработки графика ППР для металлургического предприятия:

1. Классификация оборудования по критичности (A — критичное, B — важное, C — вспомогательное)
2. Определение периодичности ТО и ремонтов по типам оборудования
   - Текущий ремонт: ежемесячно/ежеквартально
   - Средний ремонт: 1–2 раза в год
   - Капитальный ремонт: через наработку или календарный срок
3. Резервирование ресурсов (бригады, запчасти, оборудование)
4. Методика корректировки графика при внеплановых отказах
5. KPI контроля выполнения (% выполнения по плану, простои)
6. Формы документации (график, наряд, акт)

Без конкретных инвентарных номеров и заводских данных.

Результат: методология для внедрения на предприятии, экономит 4–5 часов.

2. Разработка регламентов ТОиР в общем виде

Структура документа для типового оборудования (насос, мельница, компрессор) — без привязки к конкретному заводу.

Пример промпта (Claude с веб-поиском)
Подготовь структуру регламента ТОиР для центробежного насоса горизонтального типа (общего назначения, перекачка воды/шлама):

Структура регламента:
1. Область применения (типы насосов, условия эксплуатации)
2. Типы и периодичность ТО (ежедневный осмотр, ТО-1, ТО-2, ТО-3, КР)
3. Перечень работ для каждого типа ТО (контрольные точки)
4. Нормативы допустимых параметров (вибрация, температура, расход)
5. Критерии приёмки после ремонта
6. Формы учёта (журнал осмотров, акты)
7. Приложения (схема смазки, схема уплотнений)

Без конкретных инвентарных номеров и заводских данных.

Результат: готовая структура регламента, экономит 2–3 часа.

3. Инструкции по эксплуатации — общая часть

Базовые принципы работы с оборудованием, если нет специфичных для завода модификаций.

Пример промпта
Подготовь инструкцию по эксплуатации пластинчатого теплообменника (общего назначения):

Разделы:
1. Назначение и принцип работы
2. Технические характеристики (общие диапазоны)
3. Требования к персоналу
4. Порядок пуска и остановки
5. Контрольные параметры в работе (температура, давление, перепад)
6. Профилактический осмотр (что проверять, с какой периодичностью)
7. Типичные неисправности и действия (без привязки к конкретным случаям)
8. Требования безопасности

Язык: простой, для операторов и механиков.

Результат: готовая инструкция, экономит 3–4 часа.

4. Обучающие материалы для ремонтного персонала

Базовые принципы, безопасность, типовые процедуры — без привязки к реальным инцидентам завода.

Пример промпта
Подготови обучающий модуль «Безопасность при ремонте вращающегося оборудования» для механиков:

Формат: 10 слайдов (текст для каждого)
Содержание:
1. Типы опасностей (механические, электрические, химические)
2. LOTO (Lockout/Tagout) — процедура отключения
3. Средства индивидуальной защиты
4. Контрольные вопросы (5 вопросов с ответами)

Без конкретных случаев с завода — только общие принципы.

Результат: готовый модуль, экономит 4 часа.

5. Поиск аналогов оборудования и запчастей (импортозамещение)

Открытый рынок — поиск кросс-номеров, аналогов, заменителей.

Пример промпта (веб-поиск)
Найди аналоги подшипника SKF 22320 EK/C3 для импортозамещения:

Требования:
- Российский производитель или страны ЕАЭС
- Аналогичные размеры: d=100, D=215, B=73 мм
- Тип: сферический роликовый, с бронзовым сепаратором
- Допустимая нагрузка: не ниже оригинала

Выведи таблицу: производитель, обозначение, размеры, примерная цена, наличие на складе (если есть данные).

Результат: список аналогов для закупки, экономит 1–2 часа.

6. Бенчмаркинг практик ТОиР в отрасли (RCM, TPM)

Открытая методология — лучшие практики в металлургии, ГОКах, энергетике.

Пример промпта (веб-поиск)
Проведи бенчмаркинг практик ТОиР в металлургии (2024–2026):

问题ы:
1. Какие крупные металлургические компании внедрили RCM (Reliability-Centered Maintenance)?
2. Какие результаты заявлены (снижение простоев, экономия)?
3. Какие ИТ-системы используют для ТОиР (SAP PM, 1С ТОиР, Infor)?
4. Какие показатели KPI применяют (MTBF, MTTR, OEE)?
5. Какие конференции по ТОиР в металлургии проходят в 2026?

Сделай структурированный отчёт с источниками.

Результат: обзор отраслевых практик, экономит 3–4 часа.

7. Разработка методологии перехода на RCM/TPM

Общий фреймворк — публичные подходы, без привязки к конкретному заводу.

Пример промпта
Подготовь план внедрения TPM (Total Productive Maintenance) для металлургического предприятия:

Этапы (по месяцам, 12-месячный план):
1. Подготовка: обучение персонала, создание команды TPM
2. Автономное обслуживание (операторы выполняют базовый уход)
3. Плановое техническое обслуживание (графики, регламенты)
4. Плановый ремонт (КР, ТР)
5. Обучение и развитие (тренинги, сертификация)
6. Раннее управление проектами (учёт надёжности при закупке)
7. Безопасность и качество (5S, визуальный контроль)

Для каждого этапа: цели, метрики успеха, типичные проблемы.

Результат: готовый план внедрения, экономит 5–6 часов.

8. Подготовка тендерной документации — общие разделы

Типовые разделы ТЗ на ремонтные работы — без коммерческих условий конкретной сделки.

Пример промпта
Подготовь типовую структуру ТЗ на капитальный ремонт оборудования:

Разделы:
1. Общие положения (область действия, нормативные ссылки)
2. Техническое описание объекта (без конкретных инвентарных номеров)
3. Перечень работ (типовые виды: дефектовка, ремонт, испытания)
4. Требования к качеству (приёмочные критерии, испытания)
5. Требования к документации (акты, сертификаты, фото)
6. Гарантийные обязательства (типовые сроки)
7. Требования безопасности
8. Форма отчёта исполнителя

Место для вставки конкретных данных: [УТОЧНИТЬ ПРОИЗВОДСТВО].

Результат: типовой шаблон ТЗ, экономит 2 часа.

知识测验

问题 1: Почему поиск аналогов запчастей — во внешний ИИ?

Потому что это публичная информация — каталоги, кросс-номера, цены
Потому что поставщики — внешние контрагенты
Только если это импортозамещение

问题 2: Какая задача ТОиР точно в on-prem?

Обоснование «ремонт vs замена» для конкретного компрессора
Разработка методологии RCM для металлургии
Подготовка типовой структуры ТЗ на капремонт

问题 3: Обучающие материалы для механиков — какой контур?

Внешний ИИ — базовые принципы, без конкретных инцидентов завода
On-prem — любые материалы для персонала завода
Гибрид — базовые принципы снаружи, кейсы завода внутри
