2.4 · 摘要：压缩文本不丢失意义

🎯 无代码 · ✅ 交互式测验

🎯 为什么企业需要这个

管理者将30-40%时间用于阅读：报告、邮件、会议纪要、合同。摘要将体积压缩5-10倍，保留关键信息。

节省示例：

  • 50页报告 → 1页结论
  • 会议纪要（10页）→ 决策清单和负责人
  • 合同（20页）→ 5个关键条款

节省： 每天2-3小时

📚 学习内容

3种摘要类型

📄 提取式（Extractive）

AI选择并复制文本中最重要的句子。

示例
原文："公司宣布推出新产品。产品3月1日上市。价格为5000元。订单满1万元免运费。"

摘要：
- 推出新产品
- 3月1日上市
- 价格：5000元
- 满1万元免运费

🔄 生成式（Abstractive）

AI用自己的话重写文本，压缩和概括。

示例
原文：[同上]

摘要：
"公司将于3月1日推出5000元产品，订单满1万元享免运费。"

🎯 定向式（Tailored）

针对特定任务或受众的摘要。

示例
给管理者："产品推出，价格5000元，3月上市"
给营销："价格5000元，满1万元免运费活动"
给物流："满1万元免运费，待确认发货量"

🛠️ 如何应用

场景1：每日邮件汇总

提示词
你是管理者助理。
分析下方15封邮件并创建汇总。

格式：
🔴 紧急（需今天回复）：
🟡 重要（本周内回复）：
🟢 信息（仅供参考）：

每封邮件：发件人、要点、是否需要回复、截止日期。

场景2：会议纪要

提示词
你是董事会秘书。
基于会议记录创建纪要。

格式：
1. 参会人员（列表）
2. 决议（决定了什么）：
   - 决议 | 负责人 | 期限
3. 未决问题（未决定的）：
   - 问题 | 谁应准备 | 期限
4. 后续步骤（下一步）

场景3：分析报告摘要

提示词
你是商业分析师。
分析季度销售报告（50页）。

为管理者创建执行摘要（1页）：
1. 关键数字（收入、利润、增长）
2. 前3成就
3. 前3问题
4. 建议（3点）

使用表格和项目符号。无废话。

👨‍💻 如何委派给开发人员

任务模板

"需要自动文档摘要系统。

要求：
1. 支持格式：PDF、DOCX、TXT、邮件
2. 3种摘要模式：
   - 简短（原文10%）
   - 中等（原文25%）
   - 详细（原文50%）
3. 定向摘要按角色：
   - 给管理者（聚焦数字和决策）
   - 给执行者（聚焦任务和期限）
   - 给法务（聚焦风险和责任）
4. 保留结构（标题、列表）
5. 导出为PDF / Word / 邮件

示例输入和输出文档（附件）。
期限：[日期]"

✅ 结果验收检查清单

  • 摘要保留关键信息？
  • 无事实扭曲？（核对数字、日期、姓名）
  • 长度符合要求？（短/中/长）
  • 结构便于阅读？（标题、列表）
  • 定向摘要聚焦正确方面？
  • 可导出为所需格式？

📝 理解测验

问题1： 哪种摘要类型用自己的话重写文本？

A) 提取式
B) 生成式（Abstractive）
C) 定向式

问题2： 摘要节省多少时间？

A) 10-15%
B) 60-80%
C) 20-30%

问题3： 摘要中需要检查什么？

A) 事实、数字、日期
B) 仅拼写
C) 无需检查，AI不会出错

🎯 课程要点

摘要是AI为管理者最强大的工具之一。50页 → 1页结论。关键是始终检查事实：数字、日期、姓名。AI可以压缩文本，但有时会扭曲细节。节省：每天2-3小时阅读时间。
