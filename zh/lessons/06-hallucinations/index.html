6.2 · AI幻觉：当机器 convincing 地撒谎

🎯 无代码 · ✅ 交互式测验

🎯 为什么企业需要这个

AI分不清事实与虚构。它生成的文本听起来可信，但可能完全是编造的。

真实案例： 律师使用AI准备法庭案件。AI编造了6个从未存在的先例。法官核查后 — 案件败诉，声誉受损。

危险： 幻觉在商业中特别危险，因为AI听起来自信且专业。必须始终核查。

📚 学习内容

什么是幻觉

幻觉 — 当AI生成看起来可信但实际是虚假或无法验证的信息时。

幻觉类型：

  • 事实： "X公司成立于1995年"（实际是2005年）
  • 来源： "据《福布斯》文章..."（文章不存在）
  • 引语： "正如马斯克所说..."（马斯克没说过）
  • 统计： "市场增长了150%..."（数字是编造的）

AI为什么会幻觉

原因解释
没有数据AI不知道答案，但生成"最可能的"
数据过时训练在特定日期截止
混淆不同来源的事实混在一起
想讨好用户AI倾向于给出"完整"回答，添加细节

如何识别幻觉

  • ✅ 没有来源的过于具体细节
  • ✅ 回答听起来"太完美" — 没有细微差别
  • ✅ AI断言与您的知识矛盾的内容
  • ✅ 数字是"整数"或过于精确
  • ✅ 事实没有来源链接

🛠️ 如何应用

对抗幻觉的方法

1. 要求来源

提示词
"给出带来源的回答。如果不确定 — 说'不知道'。
不要编造事实。"

2. RAG（在您的文档中搜索）

AI仅基于您的文档回答，而非通用知识。

3. 核查关键事实

数字、日期、姓名、法律 — 始终手动核查。

4. 使用专业模型

在法律/医疗数据上训练的模型，在其领域内幻觉更少。

场景：核查法律分析

核查流程
1. AI分析了合同
2. 核查每个事实：
   - 法律条款：与官方法规库核对
   - 先例：在判例库中检查
   - 期限：与日历核对
3. 如果AI说"根据民法第123条" → 
   打开民法，查找该条款
4. 可疑之处 → 交给律师

👨‍💻 如何委派给开发人员

任务模板

"需要为AI回复建立事实核查系统。

要求：
1. AI对每个事实必须标明来源
2. 自动核查：日期、数字、法律引用
3. 如果未找到来源 — 标记为'需核查'
4. 与[数据库/CRM/判例库]集成
5. 已核查事实百分比的仪表板

关键领域：[法律/财务/医疗]
期限：[日期]"

✅ 结果验收检查清单

  • 所有事实都已核查？（特别是数字、日期、姓名）
  • 每个论断都有来源？
  • AI在不确定时说了"不知道"？
  • 检查了边界情况？（罕见情形）
  • 有关键决策的核查流程？
  • 用户知道AI可能出错？

B) 完全信任AI
C) 不使用AI

问题3： 如果AI引用法律条款怎么办？

A) 在官方数据库中核查
B) 轻信

🎯 课程要点

AI convincing 地撒谎。这不是bug，而是技术特性。幻觉不可避免，但可控。要求来源、核查事实、使用RAG。永远不要仅凭AI的一个回答就做出关键决策。信任但要核查 — 这不是偏执，而是专业素养。
