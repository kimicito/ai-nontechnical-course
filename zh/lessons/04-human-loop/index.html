4.4 · 人工介入：AI何时应请求许可

🎯 无代码 · ✅ 交互式测验

🎯 为什么企业需要这个

AI会犯错。有时这些错误代价高昂：合同中的错误价格、错误的采购决策、机密数据泄露。

真实案例： AI代理自动同意了一笔高价采购。损失：200万元。原因：代理在价格超过100万元时未请求负责人确认。

解决方案： "人工介入"（Human-in-the-loop）— AI执行任务，但在关键点暂停并等待人工确认。

📚 学习内容

3个控制级别

🟢 完全自主

AI无需确认即可行动

适用：日常任务、低风险

示例：每日邮件汇总

🟡 部分控制

AI建议，人工确认

适用：重要决策、中等风险

示例：采购额 > 10万元

🔴 完全控制

AI仅准备，人工决策

适用：关键决策、高风险

示例：签署合同、解雇员工

何时必须有人工介入

情况原因
价格 > 阈值多付风险
新供应商需核查声誉
合同条款变更法律风险
机密数据安全问题
非标准情况AI未训练过此类案例

🛠️ 如何应用

场景：采购自动化

带人工控制
AI分析供应商报价：
↓
如果金额 < 5万元 → 自动同意
如果金额 5万-50万元 → 发送给采购员确认
如果金额 > 50万元 → 发送给总监确认
如果新供应商 → 始终发送检查

结果： 日常事务自动化，重要决策受控。

如何设置"人工介入"

  1. 定义阈值： 在什么条件下需要确认？
  2. 选择通知方式： 邮件、Telegram、企业通讯工具、CRM任务
  3. 设定响应时间： 等待确认多久？超时怎么办？
  4. 设置升级： 如果第一人未回复——转给谁？

👨‍💻 如何委派给开发人员

任务模板

"需要为AI代理[名称]创建'人工介入'机制。

停止条件（需确认）：
- [列举：金额 > X、新供应商等]

通知方式：[邮件 / Telegram / 企业通讯工具]
响应时间：[N分钟/小时]
升级：如果未回复 → 发送给[谁]

批准时动作：[AI做什么]
拒绝时动作：[AI做什么]

场景示例：[描述]
期限：[日期]"

✅ 结果验收检查清单

  • 所有关键操作都需要确认？
  • 通知及时送达？
  • 每个控制级别有明确阈值？
  • 升级机制有效？（如果第一人未回复）
  • 超时设置合理？（不太短）
  • 有所有决策日志？（谁何时批准）

🎯 课程要点

人工介入 = 保险。AI自动化日常事务，但人工控制重要决策。设置阈值：简单任务完全自主，重要任务部分控制，关键任务完全控制。这能防止昂贵的错误。
