1.3 · 提示词链：从简单到复杂

🎯 无代码 · ✅ 交互式测验

🎯 为什么企业需要这个

复杂任务无法用一个提示词解决。试图一次性要求所有内容，会导致肤浅的回答。

示例： 供应商市场分析包括：搜索 → 对比 → 风险评估 → 推荐。每一步都基于上一步。

节省： 4个提示词链 = 30分钟。手动分析 = 4小时。

📚 学习内容

什么是提示词链

提示词链是提示词的序列，其中每个后续提示词使用前一个的结果。

🔍 收集
→
📊 结构化
→
🔎 分析
→
✅ 决策

示例：选择包装供应商

步骤1：收集信息
你是FMCG采购员。
找出莫斯科5大纸板包装制造商。
每家：名称、网站、电话、最小订单量、生产周期。
步骤2：结构化
基于上一步结果创建表格：
| 供应商 | 网站 | 电话 | 最小订单 | 周期 | 备注 |

标记周期 > 14天的供应商。
步骤3：风险分析
分析表格：
1. 你看到哪些风险？（周期、最小订单、未知玩家）
2. 谁性价比最高？
3. 还需要额外检查什么？
步骤4：推荐
基于分析推荐2个供应商进行首次联系。
说明选择理由。列出电话时应问的问题。

🛠️ 如何应用

规则：一个提示词 = 一个任务

❌ 错误： "找到供应商，对比他们，评估风险并推荐最好的"

✅ 正确： 拆分为4个独立提示词

原因：当被要求一次性做太多时，AI会给出肤浅的回答。

何时使用提示词链

特征示例
任务需要 > 3个动作市场分析 + 对比 + 推荐
每一步依赖上一步未收集数据前无法对比
需要中间结果分析前可检查的表格
单个提示词回答太长回答被截断，丢失细节

业务提示词链模板

链「招标」

  1. 收集供应商要求
  2. 制定评估标准列表
  3. 收集报价（或查找供应商）
  4. 按标准对比
  5. 评估风险
  6. 形成推荐意见

链「报告」

  1. 收集周期数据
  2. 计算关键指标
  3. 发现趋势和异常
  4. 与上期对比
  5. 形成结论和建议

链「合同」

  1. 分析关键条款
  2. 为各方发现风险
  3. 与公司模板/历史合同对比
  4. 形成意见
  5. 准备向对方提出的问题清单

👨‍💻 如何委派给开发人员

任务模板

"需要自动化[流程]的分析链。

当前链（手动）：
1. [步骤1] → [步骤2] → [步骤3]

要求：
1. 步骤间保存中间结果
2. 每步后可暂停检查结果
3. 若某步结果不正确 — 可重试
4. 最终报告格式：[指定]
5. 完成时邮件通知

集成：[邮件 / 1C / CRM]
期限：[日期]"

✅ 结果验收检查清单

  • 每个步骤都有明确的输入和输出任务？
  • 可以检查中间步骤的结果？
  • 若步骤2不正确 — 可以只重试它？
  • 最终结果基于所有前序步骤？
  • 没有可以合并而不损失质量的步骤？
  • 链覆盖从起点到终点的完整任务？

🛠️ 工具

工具用途

n8n / Zapier自动链：步骤A → 步骤B → 步骤C

🎯 课程要点

复杂任务拆分为链：收集 → 结构化 → 分析 → 推荐。检查每一步的结果。一个任务 = 一个提示词。这是单条提示无法达到的质量。
