11.3 混合质量任务: 匿名化数据

15分钟 · 面向质量专员 · 云端+本地混合

Гибридные задачи — это сценарии, где часть работы делается в облаке (общая структура, методология, литературный обзор), а финальная версия с реальными данными — в on-prem (закрытом контуре).

Принцип: Обезличенное описание + общая методология → облако. Конкретные параметры, рецептура, номера партий → on-prem.

混合方案示意图

│  ОБЛАКО (публичные данные)              │
│  • Структура метода                     │

                  │

│  ON-PREM (производственная тайна)       │

│  • Рецептура, химсостав                │

└─────────────────────────────────────────┘

1. Root-cause + FMEA (гибрид)

Облако: структура Ishikawa-диаграммы, 8D-шаблон, FMEA-матрица. On-prem: заполнение реальными параметрами конкретной партии.

步骤1: 云端 — 模板准备
Подготовь шаблон 8D-расследования для металлургического производства:

D0: Критерии запуска (когда начинать 8D)
D1: Роли команды (руководитель, инженер по качеству, технолог, производство, лаборатория)
D2: Форма описания проблемы (5W2H)
D3: Варианты временных сдерживающих действий
D4: Шаблон Ishikawa (6M: Man, Machine, Material, Method, Milieu, Measurement)
D5-D7: Форматы для корректирующих и превентивных действий
D8: Критерии закрытия

Все поля — заполнить placeholder'ами [УТОЧНИТЬ].
步骤2: 本地 — 填入真实数据
Заполни шаблон 8D для конкретного случая:

- Партия: П-2026-1847
- Дефект: раковины на поверхности слитка
- Линия: №3
- Дата: 2026-07-15
- Параметры: [фактические данные из MES]

Проведи root-cause, заполни Ishikawa, 5 Why, D5-D7.
Результат: готовый шаблон в облаке (экономит 2 часа) + заполненный 8D в on-prem (экономит ещё 4 часа).

2. ТУ на продукцию (гибрид)

Облако: общая структура документа, типовые разделы. On-prem: конкретные технические параметры, допуски, рецептура.

Шаг 1: Облако — структура ТУ
Подготовь структуру ТУ на алюминиевый сплав для авиапрома:

1. Область применения (пример: детали авиационных конструкций)
2. Технические требования:
   - Химсостав (таблица: элемент, мин, номинал, макс)
   - Мехсвойства (σв, δ, HB, E)
   - Допуски на размеры
3. Методы контроля (список стандартных методов)
4. Упаковка и маркировка (общие требования)
5. Технологические указания (без конкретных параметров — [УТОЧНИТЬ])
6. Критерии приёмки (статистические, 100%)

Все числовые значения — [УТОЧНИТЬ ПРОИЗВОДСТВО].
Шаг 2: On-prem — заполнение параметрами
Заполни ТУ для сплава "СуперАлюм-3000":

Химсостав:
- Al: [94.0-95.0]% — номинал 94.5%
- Mg: [2.5-3.0]% — номинал 2.8%
- Si: [0.8-1.5]% — номинал 1.2%
... [конкретные данные из лаборатории]

Мехсвойства:
- σв: ≥350 МПа (наши данные: 385)
- δ: ≥10% (наши данные: 13%)
... [конкретные данные из испытаний]

Технологические указания:
- Температура литья: 720-740°C (оптимум 730°C)
- Скорость охлаждения: 15-20°C/мин
... [конкретные параметры из MES]

3. R&D сплавов (гибрид)

Облако: литературный обзор, общие металлургические принципы. On-prem: сопоставление с реальными экспериментальными данными и принятие решения.

Шаг 1: Облако — литературный обзор
Проведи литературный обзор по алюминиевым сплавам с Scandium (Sc):

1. Какие сплавы с Sc описаны в литературе (2020-2026)?
2. Какие механические свойства заявлены (σв, δ, усталость)?
3. Какие методы получения (литьё, прокатка, экструзия)?
4. Какие области применения (авиа, автопром, спорт)?
5. Какие проблемы (свариваемость, коррозия, стоимость Sc)?
6. Какие компании внедряют (публичная информация)?

Сделай сравнительную таблицу: сплав × свойства × проблемы.
步骤2: 本地 — 与我们的数据对比
Сравни наши экспериментальные данные с литературными:

Наш сплав: Al-Mg-Sc (0.3% Sc)
Наши данные:
- σв = 428 МПа (среднее, 3 партии)
- δ = 10%
- Проблема: растрескивание при сварке (2/3 партий)

Литературные данные: [из облачного обзора]

Сделай:
1. Соответствуют ли наши результаты литературным?
2. Если нет — в чём различия и почему?
3. Рекомендация: продолжить / изменить состав / отказаться?
4. Следующие шаги экспериментов (если продолжаем)

4. Обучающие материалы (гибрид)

Облако: базовый контент, общие принципы. On-prem: кейсы с реальными инцидентами завода как иллюстрации.

Шаг 1: Облако — базовый контент
Подготови обучающий модуль "Контроль качества в плавке":

1. Основные дефекты плавки (типы, причины, признаки)
   - Газовые включения
   - Окисные плёнки
   - Неравномерность химсостава
   - Раковины (внутренние и поверхностные)
2. Методы контроля (визуальный, УЗК, химанализ)
3. Критерии приёмки (общие, без конкретных цифр)
4. Действия при обнаружении дефекта (общий алгоритм)

Без привязки к конкретному заводу — только общие принципы.
步骤2: 本地 — 添加真实案例
Добавь в модуль реальные кейсы нашего завода:

Кейс 1: "Газовые включения в партии П-2026-1847"
- Что произошло: [описание, 匿名化数据е]
- Причина: [реальная причина из расследования]
- Как нашли: [метод контроля]
- Что сделали: [корректирующие действия]
- Чему научились: [урок для других мастеров]

Кейс 2: "Неравномерность химсостава в партии П-2026-1956"
- [аналогичная структура]

重要: обезличить (убрать имена, конкретные клиенты, точные параметры, если чувствительно).

云端匿名化规则

上传云端前必须删除的内容:
  • Названия клиентов ("АвтоПром ООО" → "Клиент А")
  • Номера партий ("П-2026-1847" → "Партия X")
  • Конкретные химсоставы ("Al 94.5%" → "основной элемент ~95%")
  • Названия поставщиков ("УралСырье" → "Поставщик 1")
  • ФИО сотрудников ("Иванов П.С." → "мастер смены 2")
  • Геолокация ("цех №3" → "цех Y")
可以保留的内容:
  • Тип дефекта ("раковины", "пористость")
  • Общий тип процесса ("плавка", "прокатка")
  • Общие методы контроля ("УЗК", "химанализ")
  • Структура документа (разделы ТУ, форма 8D)

知识测验

问题 1: Какой принцип гибридного подхода?

Всё делать в облаке, а потом удалить данные
Общая структура в облаке, реальные данные в on-prem
Всё делать в on-prem, облако не нужно

问题 2: Можно ли в облаке подготовить FMEA для реального процесса?

Да, FMEA — это публичный метод
Нет, только шаблон. Реальные данные процесса — в on-prem
Можно, если обезличить название завода

问题 3: Что из списка обязательно обезличить перед облаком?

Номера партий, названия клиентов, конкретные химсоставы
Типы дефектов ("раковины", "пористость")
Общие методы контроля ("УЗК", "химанализ")

问题 4: Какой шаг R&D сплавов делается в on-prem?

Литературный обзор по Scandium-сплавам
Сопоставление наших экспериментальных данных с литературными
Поиск патентов по Al-Mg-Sc

问题 5: Какой формат обучающих материалов — гибрид?

Всё в on-prem, включая базовые принципы
Базовый контент в облаке + реальные кейсы (обезличенные) в on-prem
Всё в облаке, кейсы тоже без обезличивания
