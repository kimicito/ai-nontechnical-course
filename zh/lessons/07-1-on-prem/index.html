第7级 — 采购

7.1 内部任务：本地部署

15分钟 · 面向采购专员 · 闭环

本课程涵盖采购专员在公司内部执行的任务，涉及无法发送到互联网的数据。价格、条款、谈判立场——所有这些都是商业秘密。

解决方案： 在公司封闭边界内部署本地大语言模型（如通义千问）。数据永远不会离开边界。

为什么采购需要本地部署

不能发送到云端的内容：
  • 报价中的具体供应商价格
  • 合同条款和违约金
  • 谈判立场（BATNA、筹码）
  • 各品类的采购量
  • 历史供应商数据
可以在本地完成的内容：
  • 在2分钟内比较5份报价的价格、交期和条款
  • 检查供应商的提案是否符合技术规格
  • 准备谈判论据
  • 发现合同与模板的偏差
  • 分析2年的采购模式

1. 报价对比分析

采购专员收到3到5份商业报价。以前：在Excel中手动汇总，比较价格、交货时间、付款条件、保修期。现在：上传到本地大语言模型并提问。

提示示例
我有3份水泵供应的商业报价。
文件Q1、Q2、Q3已附。

创建对比表：
- 单价
- 交货时间
- 付款条件
- 保修期
- 运费

突出每个标准中的最佳报价和总体最优方案。
结果： 对比表 + 最佳方案的可视化高亮。采购专员在常规工作上节省20到30分钟。

2. 提案与技术规格核对

供应商经常回应技术规格，但带有附带条件或偏差。大语言模型可以比人工更快地找到这些偏差。

提示示例
对比技术规格（文件TS.docx）和供应商的回应（文件Response.docx）。

查找：
1. 与技术规格的完全偏差（不匹配的内容）
2. 部分偏差（有附带条件的匹配内容）
3. 遗漏项（供应商未处理的内容）

列出具体的技术规格条款作为参考。

3. 谈判准备

这是最敏感的任务。BATNA（谈判协议的最佳替代方案）、筹码、供应商弱点——这些都不能暴露在云端。本地部署解决了这个问题。

提示示例
我正在准备与供应商X就设备采购进行谈判。

以下是我的数据：
- 有2家替代供应商，价格低15%
- 该供应商上季度延迟交货2周
- 我需要在3周内交货，但他们的标准周期是6周
- 订单量是50件，这在该品类中是大订单

准备：
1. 我降价的论据
2. 筹码点（礼貌但有说服力）
3. BATNA——如果我们无法达成协议该怎么做
4. 谈判方案（分步进行）

4. 合同审查

律师审查合同需要一周。大语言模型在几分钟内就能发现与模板的偏差——然后律师验证这些发现。

提示示例
对比供应商的合同（文件Contract.docx）和我们的标准模板（文件Template.docx）。

查找以下方面的偏差：
- 双方责任
- 违约金
- 交货条款
- 付款条件
- 争议解决程序

对于每项偏差，请注明：在模板中的位置、在合同中的位置、对我们的风险。

5. 物料编码/品名标准化

在公司中，5个部门对同一种产品的叫法不同："格兰富水泵32-160"、"Gründfos水泵32-160"、"GRUNDFOS 32-160"。大语言模型将它们统一为标准化格式。

提示示例
以下是3个部门的品名清单（文件Nomenclature.xlsx）。

统一为标准化格式：
- 标准化品牌名称（拉丁字母）
- 标准化计量单位
- 删除重复项
- 创建映射表：旧名称 → 新名称

将结果保存为Excel。

6. 采购历史分析

采购专员想弄清楚：固定供应商的价格在上涨吗？有季节性吗？哪些品类的涨价速度最快？

提示示例
我有2年的采购历史数据（文件History.xlsx）。

分析：
1. 前10大供应商的价格走势（上涨/下跌）
2. 季节性（哪些月份更贵）
3. 涨价幅度最大的前5个品类
4. 各供应商从下单到交货的平均时间
5. 建议：哪些地方有涨价风险，需要关注什么

7. 供应商风险评分

内部供应商数据 + 风险评估模板的组合。大语言模型计算评分并解释原因。

提示示例
根据以下数据评估供应商"PromTekhSnab"的风险：

- 合作3年
- 去年有2次交货延迟（5天和12天）
- 1次质量投诉（缺陷率3%，由他们承担费用）
- 价格低于市场8%
- 财务状况：收入增长，无诉讼
- 合同年底到期，之后重新招标

在1到10的评分范围内：
1. 总体风险概况
2. 运营风险（交期违约）
3. 财务风险
4. 质量风险

给出建议：续约、重新招标，还是寻找备用供应商。

8. 运输单据处理

运单、发票、CMR——大语言模型提取数据并与订单交叉核对。全部在边界内部完成。

提示示例
文件Waybill.pdf——解析内容：

- 运单编号、日期
- 发货人和收货人
- 商品清单及数量、重量
- 货物价值
- 发货日期和交货日期（实际与计划）

与订单（文件Order.xlsx）交叉核对，并识别：
- 短缺
- 错误品类
- 交货时间偏差

9. 管理层报告

采购专员经常花整晚做PPT。大语言模型在10分钟内起草报告——只需核对数字。

提示示例
根据文件Purchases_Q3.xlsx的数据，为董事会准备报告草稿：

1. 总采购量（金额、与第二季度相比的增长）
2. 按金额排名前5的品类
3. 关键项目的状态（按时 / 有风险 / 逾期）
4. 针对2到3个问题的建议
5. 第四季度计划（简要）

格式：结构化文本，可直接粘贴到演示文稿中。

知识检测

问题1： 为什么报价对比不能发送到云端ChatGPT？

因为ChatGPT运行缓慢
因为供应商价格是商业秘密
因为ChatGPT无法读取Excel

问题2： 准备包含BATNA的谈判应该使用什么工具？

Google文档
闭环内的本地大语言模型
数据删除后的ChatGPT

问题3： 大语言模型在对比5份报价时节省多少时间？

没有节省时间，手动更快
30分钟 → 2分钟
5分钟 → 4分钟

问题4： 什么是供应商风险评分？

基于内部数据的1到10分风险评估
银行信用记录查询
在线搜索供应商评价

问题5： 以下哪份文件 definitely 属于本地部署范围？

无价格的技术规格草案
包含具体违约金和条款的合同
从英文翻译的说明书
