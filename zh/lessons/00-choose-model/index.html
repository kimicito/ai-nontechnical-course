0.2 · 如何选择模型：ChatGPT、Claude、Kimi、Qwen

🎯 无代码 · ✅ 末尾有测验

🎯 为什么企业需要这个

不同的模型就像不同的工具。用螺丝刀钉钉子不行，用锤子拧螺丝也不行。选择错误的模型 = 浪费时间、金钱和精力。

本课后您将能够：
  • 在2分钟内选择适合任务的模型
  • 理解为什么ChatGPT并非总是最佳选择
  • 节省开支，避免为简单任务购买昂贵的模型

📚 模型对比

🟢 ChatGPT (OpenAI)

✅ 通用型。很好地理解上下文。庞大的社区。集成无处不在。

⚠️ 有时会「平滑」回答，回避有争议的话题。可能比竞争对手更贵。

适合： 需要通用助手、通信、创意生成、简单分析时。

🟣 Claude (Anthropic)

✅ 长文档之王。一次可以「阅读」100+页。评估更诚实。

⚠️ 无法实时搜索互联网。集成较少。

适合： 处理合同、报告、审计等大量文本工作。

🔵 Kimi (Moonshot AI)

✅ 超大上下文（200万令牌）。中文和英文表现出色。价格便宜。

⚠️ 俄语较弱。商业集成较少。

适合： 处理中国供应商/市场、分析海量数据时。

🟠 Qwen (阿里巴巴)

✅ 多语言（25+种）。开源模型。适合集成到企业系统中。

⚠️ 设置更复杂。需要更多「调整」来适应任务。

适合： 多语言项目、本地部署、与现有系统集成。

🛠️ 选择算法（1-2-3）

步骤1：什么任务？

  • 文档 > 50页 → Claude或Kimi
  • 通信、创意、简单分析 → ChatGPT
  • 中国市场/供应商 → Kimi或Qwen
  • 机密数据 → Qwen（本地部署）或GENAi

步骤2：什么预算？

  • 免费 → ChatGPT Free、Claude Free（有限制）
  • $20/月 → ChatGPT Plus或Claude Pro
  • $50-200/月 → API + 集成（需要开发人员）
  • 本地部署 → 服务器 + 许可证（参见课程0.3）

步骤3：在实际任务上测试

  • 取一个真实的工作文档/请求
  • 用2-3个模型（免费版）测试
  • 对比质量、速度、便利性
  • 选择最佳方案并订阅

👨‍💻 如何向开发人员分配任务

模板： 
"需要为[任务描述]选择LLM。

标准：
- 语言：[RU/EN/CN]
- 输入数据量：每天[N]页/请求
- 保密性：[是/否]
- 集成系统：[系统名称]
- 预算：每月[金额]
- 是否需要本地部署：[是/否]

请比较2-3个方案并给出推荐
及理由。需要在[N]天内完成POC。"

✅ 结果验收检查清单

  • 模型能处理实际工作任务吗？
  • 成本可预测且在预算内？
  • 有备选方案（不依赖单一供应商）？
  • 开发人员确认集成可行？
  • 如果供应商更改条款，已验证数据迁移流程？

🧰 选择工具

  • Chatbot Arena (lmarena.ai) — 模型盲测，为最佳回答投票
  • Artificial Analysis — 价格、速度、质量对比
  • Vellum — 跨模型的提示词比较平台

📝 理解测验

问题1： 分析200页合同应选哪个模型？

A) Claude或Kimi — 擅长处理长文档
B) ChatGPT — 最通用
C) 任何模型都一样

问题2： 可以用ChatGPT处理机密数据吗？

A) 可以，购买付费版即可
B) 不行，数据会发送到OpenAI服务器。需要本地部署（Qwen + GENAi）
C) 可以，ChatGPT不保存数据

问题3： 为什么不总是选最强大的模型？

A) 强大的模型总是更好
B) 更贵更慢。简单任务可以用便宜模型完成
C) 因为很难找到

🎯 课程要点

没有「最好」的模型，只有适合任务的模型。Claude适合文档，ChatGPT适合通用任务，Kimi适合中文和大数据量，Qwen适合本地部署和多语言。选择前务必在真实任务上测试。
