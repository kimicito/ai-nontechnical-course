4.1 · 什么是AI代理：从聊天到自主行动

🎯 无代码 · ✅ 交互式测验

🎯 为什么企业需要这个

普通AI等待您的命令。代理自行行动：设定目标、规划步骤、执行、检查结果。

示例： 您对代理说"准备包装市场报告"。代理自行：

  1. 在网上搜索信息
  2. 分析找到的内容
  3. 制作对比表格
  4. 撰写结论与建议
  5. 发送完成的文档给您

您花费： 5分钟布置任务。代理花费： 20分钟执行。

🛠️ 提示词 vs 技能 vs 子代理：如何选择工具

并非所有任务都需要复杂处理。一次性任务用一次性提示词。常规流程用契约或技能。需要干净上下文时用子代理。

机制何时使用示例
提示词（Prompt）当前对话中的一次性任务"对比3个供应商的报价"
项目指令项目中几乎所有工作的规则"始终写得简洁，不要废话"
技能（Skill）有边界和检查的可重复流程"每周五按此合同检查报告"
子代理（Subagent）需要干净上下文或并行工作"独立检查这份合同，同时我做其他事"

快速选择路线

  1. 一个对话中的一个回答？ → 提示词
  2. 整个项目的规则？ → 项目指令
  3. 有始有终的可重复流程？ → 技能
  4. 需要独立执行者或干净上下文？ → 子代理
💡 演进示例

第1周："分析这份报告" — 提示词（一次性）

第3周："每周五按此合同检查报告" — 技能（常规）

第5周："独立于主任务检查报告" — 子代理（隔离）

💡 决策规则

从最简单的、能提供所需可重复性的机制开始。一次性任务用提示词。持续规范用项目指令。可重复流程用技能。需要隔离用子代理。

📚 学习内容

聊天机器人 vs 代理

💬 聊天机器人

  • 等待问题
  • 仅回复一次
  • 不记住上下文
  • 不执行操作
  • 需要持续关注

🤖 代理

  • 自己设定目标
  • 执行一系列操作
  • 记住状态
  • 使用工具
  • 自主工作

代理的组成部分

组件功能类比
大脑（LLM）思考、规划、决策员工的大脑
记忆记住上下文、历史、事实员工的记事本
工具执行操作：搜索、邮件、Excel员工的双手
规划器决定下一步做什么项目经理

🛠️ 如何应用

商业代理示例

🕵️ 市场分析师代理

目标： 每周竞争对手报告

操作： 搜索竞争对手价格 → 分析 → 制作表格 → 发送报告

节省： 每周4小时

📋 采购助理代理

目标： 处理 incoming 报价

操作： 接收邮件 → 提取数据 → 与数据库对比 → 生成建议 → 发送给采购员

节省： 每天2小时

🔍 审计代理

目标： 检查合同是否符合标准

操作： 接收合同 → 与模板对比 → 发现偏差 → 生成报告 → 发送给法务

节省： 每份文档3小时

⛏️ 采矿控制代理

目标： 每日检查实际产量是否符合计划

操作： 获取过程控制系统 (SCADA)数据 → 与计划对比 → 发现偏差 → 分析原因 → 发送汇总到企业通讯工具

节省： 每天2小时手动核对

- [上网搜索 / 邮件 / Excel / API / 等]

限制：
- [API请求预算]
- [执行时间]
- [什么需要人工确认]

✅ 结果验收检查清单

  • 代理完全达成目标？（从头到尾）
  • 没有陷入单步循环？
  • 正确使用工具？
  • 步骤间记住上下文？
  • 在预算和时间内完成？
  • 关键决策有"人工介入"？

🎯 课程要点

代理 = 自主员工。它有大脑（LLM）、记忆、工具和规划器。它不仅回答问题——还从头到尾解决问题。从简单代理开始，逐步增加复杂度。始终保留"人工介入"。
