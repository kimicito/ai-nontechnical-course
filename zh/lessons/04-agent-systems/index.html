第4级 — 智能体

4.2 企业智能体系统

15分钟 · 数字员工的工作原理

生产型企业中90%的任务可以在企业内网完成——使用自有模型，无需将数据发送到云端。为此，智能体系统应运而生——这些框架可以在企业服务器上运行AI智能体。

本课不涉及安装和代码。它讲的是数字员工如何运作以及它由哪些"文件"组成——这样您就能向IT部门布置任务并验收结果。

🎯 为什么需要这个

本地化部署 = 数据不离开公司。故障日志、冶炼参数、开采计划、商业预测——所有数据都留在内部。

智能体系统 = 有记忆的数字员工。它不仅能回答问题——还能记住、学习、按计划执行任务，并与您的系统（1C、SAP、过程控制系统 (SCADA)）集成。

📚 学习内容

三种智能体系统

系统 用途 类比
OpenClaw 通用智能体，支持即时通讯（Telegram、企业通讯工具、Email）。在您的服务器上运行。 全渠道个人助理
Hermes Agent 自学习智能体。从经验中创建技能，记住用户偏好。 在实践中学习的老员工
GoClaw 企业级多智能体平台，部门间数据隔离。 企业人力资源管理系统

智能体的关键文件

智能体不是"黑箱"。它由文档组成，就像员工由工作说明书和人事档案组成。以下是每个文件的含义：

文件 说明 商业类比
SOUL.md 智能体的"灵魂"——它的性格、沟通风格、角色。例如："你是一位经验丰富的工艺工程师，说话简洁、直击要点"。 岗位职责 + 企业文化
SKILL.md 技能——针对典型任务的分步说明。例如："如何分析故障日志"。 规章制度 / 标准操作流程
HEARTBEAT.md 定期检查计划。例如：每天早上8:00检查开采计划。 值班表和计划会
MEMORY.md 长期记忆——事实、决策、经验教训。例如："3号炉在温度>1200°时产生废品"。 知识库 / 观察日志
AGENTS.md 系统中所有智能体的通用规则——什么可以做，什么不可以做，红线。 公司行为准则
USER.md 用户画像——您是谁、做什么、哪些事情可以不经请示就做。 员工档案 + 权限
TOOLS.md 智能体可访问的工具列表：SAP、1C、Grafana、Email、企业通讯工具。 可用程序和系统清单

🛠️ 如何应用

示例：开采监控智能体

智能体配置

SOUL.md："你是开采分析师。说话简洁，使用过程控制系统 (SCADA) 的数据。没有事实依据不做假设。"

HEARTBEAT.md：每天7:30检查实际开采量与计划对比，发现偏差>5%，发送简报。

SKILL.md："开采日志分析：1）从过程控制系统 (SCADA) 加载数据，2）与钻井计划对比，3）标出偏差，4）给出建议，5）发送到企业通讯工具。"

MEMORY.md："47号井在压力<15个大气压时产量不稳定。2026年3月曾发生抽油杆断裂。"

结果：总工程师在班前通过企业通讯工具收到简报——包含事实、偏差和建议。人工核对时间从2小时缩短到5分钟。

示例：质量控制智能体

智能体配置

SOUL.md："你是质量工程师。检查冶炼参数是否符合标准。只指出偏差。"

HEARTBEAT.md：每4小时检查温度、压力、最新批次的化学成分。出现偏差时立即通知。

TOOLS.md：可访问过程控制系统 (SCADA) 数据库，企业通讯工具用于告警，Excel用于报告。

结果：废品在发货前就被实时发现，而非等到发货时才察觉。节省退货和重熔成本。

✅ 验收清单

  • 智能体运行在企业服务器上（本地化部署）？
  • 数据不会发送到外部云端？
  • SOUL.md是否反映了业务所需的角色和风格？
  • HEARTBEAT.md是否覆盖关键检查（频率不过高）？
  • SKILL.md是否像规章制度一样分步描述任务？
  • MEMORY.md是否积累了有用的信息（而非垃圾）？
  • TOOLS.md是否仅限于必要的系统（最小权限原则）？
  • 关键决策是否保留"人在回路"？

📝 理解测试

问题 1：SOUL.md 文件定义了什么？

智能体可访问的工具列表
智能体的性格、沟通风格和角色
定期检查计划

问题 2：HEARTBEAT.md 的用途是什么？

存储智能体的长期记忆
设定智能体的定期任务计划
描述技能的分步说明

问题 3：为什么生产数据必须使用本地化智能体？

本地化智能体比云端运行更快
数据（开采参数、计划、故障记录）不离开公司内网
本地化智能体维护成本更低

💡 总结

智能体系统不是魔法，而是一组文件（SOUL、SKILL、HEARTBEAT、MEMORY），它们定义了数字员工的行为。

本地化部署确保您的生产数据留在公司内部。

管理者的关键技能：能够编写这些文件的内容——就像为人编写岗位职责说明书，只不过对象换成了智能体。
