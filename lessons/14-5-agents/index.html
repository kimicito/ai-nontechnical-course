14.5 Агентские сценарии для Продаж

⏱ 12 мин · скоринг, КП, прогнозирование
Чем агент отличается от языковой модели?
LLM пишет текст по запросу. Агент работает циклически: читает CRM, оценивает лиды, формирует КП, отправляет — без участия менеджера в каждом шаге. Подробнее — Уровень 4.

Сценарий 1: Агент скоринга лидов (n8n + CRM)

Задача: оценивать новые лиды из CRM, приоритизировать, назначать менеджерам.

[Триггер: новый лид в CRM]
↓
[n8n: сбор данных о лиде]
├─→ Источник (сайт, выставка, рекомендация)
├─→ Данные компании (отрасль, размер)
├─→ Запрос (продукт, объём, срок)
└─→ История взаимодействий
↓
[LLM: скоринг]
├─→ Бюджет (1–5)
├─→ Авторитет (1–5)
├─→ Потребность (1–5)
├─→ Срок (1–5)
└─→ Итоговый балл BANT
↓
[Приоритизация]
├─→ A (≥80 баллов): срочно → старший менеджер
├─→ B (50–79): в течение дня → менеджер
└─→ C (<50): nurture → email-рассылка
↓
[Создание задачи в CRM + уведомление]
Результат: менеджеры тратят время на лиды с вероятностью конверсии >30%, а не на все подряд.

Сценарий 2: Агент подготовки КП (OpenClaw-агент)

Задача: по запросу клиента автоматически подготовить персонализированное коммерческое предложение.

[Запрос клиента: email / форма сайта / CRM]
↓
[Агент: извлечение требований]
├─→ Продукт, объём
├─→ Специфические требования (сертификаты, упаковка)
├─→ Сроки
└─→ История покупок клиента
↓
[On-prem LLM: формирование КП]
├─→ Обращение по контексту
├─→ Предложение с учётом истории
├─→ Условия (цена, срок, оплата)
├─→ Преимущества для данного клиента
└─→ Призыв к действию
↓
[PDF-генерация]
↓
[Email клиенту + задача менеджеру: «КП отправлено, следите за ответом»]

Сценарий 3: Агент прогнозирования спроса (гибридный)

Задача: анализировать историю продаж, рыночные данные, формировать прогноз для планирования.

[Cron: еженедельно]
↓
[Сбор данных]
├─→ История продаж (ERP)
├─→ Портфель заказов (CRM)
├─→ Сезонность
└─→ Рыночные данные (внешние: индексы, отраслевая статистика)
↓
[On-prem агент: анализ трендов]
├─→ Рост/падение по продуктам
├─→ Новые тренды
└─→ Аномалии (крупные разовые заказы)
↓
[LLM: формирование прогноза]
├─→ На месяц
├─→ На квартал
└─→ Риски и сценарии
↓
[Отчёт: отделу продаж + производству + закупкам]
Важно: история продаж и портфель заказов — коммерческая тайна. Анализ трендов — on-prem. Рыночные данные можно получать извне.
