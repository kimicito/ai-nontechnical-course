Уровень 7 — Закупки

7.1 Внутренние задачи: on-prem

15 мин · для закупщиков · закрытый контур

В этом уроке — задачи, которые закупщик выполняет внутри компании, с данными, которые нельзя отправлять в интернет. Цены, условия, переговорная позиция — всё это коммерческая тайна.

Решение: on-prem LLM (например, Qwen) в закрытом контуре компании. Данные не покидают периметр.

Зачем on-prem для закупщика

Что нельзя отправлять в облако:
  • Конкретные цены поставщиков из КП
  • Условия договоров и штрафы
  • Переговорная позиция (BATNA, рычаги)
  • Объёмы закупок по позициям
  • Исторические данные о поставщиках
Что можно сделать в on-prem:
  • Сравнить 5 КП по цене, срокам, условиям — за 2 минуты
  • Проверить, соответствует ли предложение ТЗ
  • Подготовить аргументы для переговоров
  • Найти отклонения в договоре от шаблона
  • Проанализировать паттерны закупок за 2 года

1. Сравнительный анализ КП

Закупщик получает 3–5 коммерческих предложений. Раньше: вручную сводить в Excel, сравнивать цену, срок поставки, условия оплаты, гарантии. Сейчас: загрузить в on-prem LLM и задать вопрос.

Пример промпта
У меня 3 коммерческих предложения на поставку насосов. 
Приложены файлы КП1, КП2, КП3.

Сделай сравнительную таблицу:
- Цена за единицу
- Срок поставки
- Условия оплаты
- Гарантийный срок
- Стоимость доставки

Выдели лучшее предложение по каждому критерию и итоговый победитель.
Результат: таблица сравнения + визуальное выделение лучших позиций. Закупщик экономит 20–30 минут на рутине.

2. Сверка предложения с ТЗ

Часто поставщик отвечает на ТЗ, но с оговорками или отклонениями. LLM может найти эти отклонения быстрее человека.

Пример промпта
Сравни техническое задание (файл ТЗ.docx) и ответ поставщика (файл Ответ.docx).

Найди:
1. Полные отклонения от ТЗ (что не соответствует)
2. Частичные отклонения (что соответствует с оговорками)
3. Пропущенные пункты (на что поставщик не ответил)

Сделай список с указанием конкретных пунктов ТЗ.

3. Подготовка к переговорам

Это самая чувствительная задача. BATNA (лучшая альтернатива без сделки), рычаги давления, слабые места поставщика — всё это нельзя светить в облаке. On-prem решает проблему.

Пример промпта
Я готовлюсь к переговорам с поставщиком X по закупке оборудования.

Вот мои данные:
- Есть 2 альтернативных поставщика с ценами на 15% ниже
- Текущий поставщик задержал поставку в прошлом квартале на 2 недели
- Мне нужна поставка за 3 недели, а у него стандартно 6 недель
- Объём закупки — 50 единиц, это крупный заказ для его категории

Подготовь:
1. Мои аргументы для снижения цены
2. Рычаги давления (вежливо, но убедительно)
3. BATNA — что делать, если не договоримся
4. Сценарий переговоров (по шагам)

4. Проверка контрактов

Юрист проверяет договор неделю. LLM находит отклонения от шаблона за минуты — юрист потом валидирует находки.

Пример промпта
Сравни договор поставщика (файл Договор.docx) и наш стандартный шаблон (файл Шаблон.docx).

Найди отклонения по:
- Ответственности сторон
- Штрафным санкциям
- Срокам поставки
- Условиям оплаты
- Порядку разрешения споров

Для каждого отклонения укажи: где в шаблоне, где в договоре, риск для нас.

5. Нормализация номенклатуры / SKU

В компании 5 подразделений называют один товар по-разному: «насос Grundfos 32-160», «насос Грюндфос 32-160», «GRUNDFOS 32-160». LLM приводит к единому виду.

Пример промпта
Вот список номенклатуры из 3 подразделений (файл Номенклатура.xlsx).

Приведи к единому виду:
- Унифицируй названия брендов (латиница)
- Стандартизируй единицы измерения
- Убери дубли
- Создай таблицу соответствия: старое название → новое название

Сохрани результат в Excel.

6. Анализ истории закупок

Закупщик хочет понять: растут ли цены у постоянного поставщика? Есть ли сезонность? Какие позиции дорожают быстрее всего?

Пример промпта
У меня история закупок за 2 года (файл История.xlsx).

Проанализируй:
1. Динамика цен по топ-10 поставщикам (тренд роста/падения)
2. Сезонность (какие месяцы дороже)
3. Топ-5 позиций с максимальным ростом цены
4. Среднее время между заказом и поставкой по поставщикам
5. Рекомендации: где есть риск роста цен, на что обратить внимание

7. Risk-скоринг поставщика

Комбинация внутренних данных о поставщике + шаблон оценки рисков. LLM считает скоринг и объясняет почему.

Пример промпта
Оцени риск поставщика "ПромТехСнаб" по данным:

- Работаем 3 года
- 2 задержки поставок за последний год (на 5 и 12 дней)
- 1 претензия по качеству (брак 3%, устранили за свой счёт)
- Цена на 8% ниже рынка
- Финансовые показатели: выручка растёт, нет судебных дел
- Контракт до конца года, потом перетендер

По шкале от 1 до 10:
1. Общий риск-профиль
2. Риск операционный (срыв сроков)
3. Риск финансовый
4. Риск качества

Дай рекомендацию: продлевать, перетендеровать, искать резерв.

8. Разбор транспортных документов

ТТН, инвойсы, CMR — LLM извлекает данные и сверяет с заказом. Всё внутри контура.

Пример промпта
Файл ТТН.pdf — разбери содержимое:

- Номер ТТН, дата
- Отправитель и получатель
- Перечень груза с количеством и весом
- Сумма груза
- Дата отгрузки и дата доставки (факт vs план)

Сверь с заказом (файл Заказ.xlsx) и выяви:
- Недостачи
- Пересорты
- Отклонения по срокам

9. Отчёты для руководства

Закупщик тратит вечер на PowerPoint. LLM делает черновик за 10 минут — остаётся только доверить цифры.

Пример промпта
На основе данных из файла Закупки_Q3.xlsx подготовь черновик отчёта для совета директоров:

1. Общий объём закупок (сумма, рост vs Q2)
2. Топ-5 категорий по сумме
3. Статус по ключевым проектам (в срок / с рисками / просрочены)
4. Рекомендации по 2–3 проблемным зонам
5. План на Q4 (кратко)

Формат: структурированный текст, готовый к вставке в презентацию.

Проверка знаний

Вопрос 1: Почему сравнение КП нельзя отправлять в облачный ChatGPT?

Потому что ChatGPT работает медленно
Потому что цены поставщиков — коммерческая тайна
Потому что ChatGPT не умеет читать Excel

Вопрос 2: Какой инструмент использовать для подготовки к переговорам с BATNA?

Google Docs
On-prem LLM в закрытом контуре
ChatGPT с удалением данных после

Вопрос 3: Сколько времени экономит LLM на сравнении 5 КП?

Не экономит, вручную быстрее
30 минут → 2 минуты
5 минут → 4 минуты

Вопрос 4: Что такое risk-скоринг поставщика?

Оценка рисков по шкале 1–10 на основе внутренних данных
Проверка кредитной истории в банке
Поиск отзывов о поставщике в интернете
