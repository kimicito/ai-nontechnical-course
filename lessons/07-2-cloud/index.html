Уровень 7 — Закупки

7.2 Внешние задачи: облако

15 мин · для закупщиков · открытые данные

В этом уроке — задачи, где закупщик работает с публичной информацией: поиск поставщиков, бенчмаркинг цен, подготовка обезличенных документов. Эти данные не содержат коммерческой тайны — их можно отправлять в облачные модели.

Решение: облачные API: Claude, ChatGPT, Kimi, Qwen API. Нужен веб-поиск — значит, модель должна быть подключена к интернету.

Когда облако лучше on-prem

Облачные модели выигрывают, когда нужно:
  • Веб-поиск: найти новых поставщиков, проверить цены на рынке
  • Перевод: работать с зарубежными документами
  • Генерация: создать шаблонный текст с нуля
  • Анализ публичных данных: индексы сырья, курсы валют, таможенные коды
Важно: перед отправкой в облако — проверь, что в запросе нет конкретных цен, названий поставщиков, объёмов. Если есть — обезличь или отправь в on-prem.

1. Поиск аналогов и альтернативных поставщиков

Закупщик знает 2–3 поставщика на позицию. Нужно найти ещё — для конкуренции или резерва. On-prem не имеет доступа к интернету, облако — имеет.

Пример промпта (Claude с веб-поиском)
Найди производителей промышленных центробежных насосов 
серии 32-160 в России и ближнем зарубежье (Казахстан, Беларусь).

Укажи:
- Название компании и сайт
- Где производство (город)
- Есть ли сертификаты (ГОСТ, ISO)
- Примерный диапазон цен (если публично)
- Контакты для закупок (отдел продаж)

Сделай таблицу сравнения.
Результат: таблица потенциальных поставщиков с контактами. Закупщик тратит 10 минут вместо 2 часов ручного поиска.

2. Бенчмаркинг рыночных цен и индексов сырья

Закупщик договаривается о цене, но не знает, где рынок сейчас. LLM с веб-поиском находит актуальные индексы.

Пример промпта
Какие сейчас (2024-2025) рыночные цены на:
1. Стальную арматуру А500С (опт, тонна)
2. Портландцемент М500 (опт, тонна)
3. Медь катодную (LME + премия)

Также укажи:
- Динамику за последние 6 месяцев (рост/падение %)
- Прогноз на ближайший квартал (если есть)
- Основные факторы, влияющие на цену
Важно: в запросе нет конкретных цен твоих поставщиков — только рыночные ориентиры. Это безопасно для облака.

3. Подготовка обезличенного ТЗ (черновик)

ТЗ с конкретными цифрами — в on-prem. А черновик структуры ТЗ, общие требования, типовые параметры — можно сделать в облаке, а потом довести внутри.

Пример промпта
Подготовь структуру технического задания на закупку 
промышленного насосного оборудования для химического производства.

Без конкретных цифр (объёмов, цен, сроков). 
Только типовые параметры и требования.

Структура:
1. Общие требования
2. Технические параметры (мощность, производительность, материалы)
3. Требования к качеству и сертификации
4. Условия поставки (типовые)
5. Требования к документации
6. Критерии приёмки

Укажи, какие пункты нужно будет дополнить конкретными данными внутри компании.

4. Тендерная документация (общие разделы)

Инструкция для участников, критерии оценки, форма заявки — всё это шаблонное и не содержит данных конкретной сделки.

Пример промпта
Создай шаблон тендерной документации для закупки 
промышленного оборудования методом открытого конкурса.

Включи:
1. Инструкция для участников (правила подачи, сроки)
2. Форма заявки (общие поля)
3. Критерии оценки (цена, сроки, опыт) — без весовых коэффициентов
4. Форма договора (типовые условия)
5. Перечень документов для заявки

Укажи, какие разделы нужно согласовать с юристом и заказчиком.

5. Автоответы клиентам и площадкам

Статус груза, сроки поставки, запросы на КП — часто шаблонные. LLM генерирует ответ по статусу из ERP, но без конкретных цифр.

Пример промпта
Напиши шаблон письма ответа на запрос статуса поставки.

Условия:
- Заказ в пути
- Поставщик сообщил о задержке
- Нужно извиниться, объяснить причину, назвать новый срок
- Предложить компенсацию (типовая)

Без конкретных дат, сумм, названий поставщиков. 
Получатель: заказчик внутри компании.

6. Перевод и локализация документов

Зарубежные поставщики присылают инструкции, спецификации, договоры на английском, китайском, немецком. Облачный перевод быстрее и точнее стандартных переводчиков для технических текстов.

Пример промпта
Переведи с английского на русский техническую спецификацию 
на промышленный насос (файл Spec.pdf).

Требования:
- Сохрани структуру и нумерацию разделов
- Технические термины — по ГОСТ/ISO
- Единицы измерения — в СИ (м³/ч, бар, кВт)
- Сохрани таблицы в формате markdown
- Отметь непонятные места для уточнения у поставщика

7. Проверка таможенных требований (общая нормативка)

Код ТН ВЭД, требования сертификации, пошлины — это публичная информация. LLM находит актуальные требования для категории товаров.

Пример промпта
Какие таможенные требования и сертификаты нужны 
для ввоза в РФ промышленных центробежных насосов (код ТН ВЭД 8413 70)?

Укажи:
- Обязательная сертификация (ГОСТ-Р, ТР ТС)
- Нужен ли СБКТС
- Таможенная пошлина (ставка)
- НДС при ввозе
- Требования к документам поставщика (инвойс, упаковочный лист)
- Особенности для конкретных стран (КНР, Германия, Италия)

Без конкретного груза и сумм — только общая нормативка.

Проверка знаний

Вопрос 1: Почему поиск новых поставщиков лучше делать в облаке, а не в on-prem?

Потому что on-prem не имеет доступа к интернету для поиска
Потому что on-prem работает медленнее
Потому что облако бесплатное

Вопрос 2: Какую модель НЕ стоит использовать для бенчмаркинга цен?

Claude с веб-поиском
On-prem LLM без интернета
ChatGPT с браузингом

Вопрос 3: Какой запрос безопасен для облака?

"Сравни цены моих 3 поставщиков: А, Б, В" (с конкретными суммами)
"Какие рыночные цены на арматуру А500С в 2025 году?"
"Проверь договор с поставщиком X на соответствие шаблону"

Вопрос 4: Какую задачу можно делать в облаке, если ТЗ содержит конкретные объёмы?

Отправить полное ТЗ с объёмами в ChatGPT
Сделать черновик структуры ТЗ без цифр, а объёмы добавить в on-prem
Нельзя использовать ИИ для ТЗ вообще

Вопрос 5: Какая модель лучше для перевода технических спецификаций?

Claude или GPT-4 — они лучше с техническими терминами
Любая бесплатная модель — перевод это простая задача
Только on-prem модель, потому что в спецификации могут быть цены
