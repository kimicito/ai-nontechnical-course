17.3 Гибридные задачи HR

⏱ 15 мин · типовое/обезличенное снаружи, персональное внутри
Принцип гибрида для HR
Общая структура, методология, типовые формулировки — снаружи. Конкретные ФИО, оклады, оценки, факты о сотруднике — внутри.

Задача 1: Reference check — структура вопросов снаружи, оценка с именем внутри

Гибрид: типовая структура reference check и критерии оценки — универсальны. Финальная оценка конкретного кандидата по реальным ответам рекомендателей — on-prem.

Этап 1 — Внешний ИИ (структура)

Промпт:
Разработай структуру reference check для позиции [название] уровня [junior/middle/senior]. Включи: круг рекомендателей (руководитель, коллеги, клиенты), вопросы по компетенциям [список], шкалу оценки, форму фиксации результатов. Формат: готовый шаблон для HR.

Этап 2 — On-prem (оценка конкретного кандидата)

Промпт:
Результаты reference check кандидата [ФИО] по структуре выше: [записи]. Проведи оценку по каждой компетенции. Сформируй итоговое заключение с рекомендацией (приём/отказ/дополнительное интервью).
Экономия: 30–40 минут на подготовку структуры (делается 1 раз на тип позиции). Оценка конкретного кандидата — 10 минут вместо 30.

Задача 2: Индивидуальный план развития — шаблон снаружи, привязка к сотруднику внутри

Гибрид: общий шаблон ИПР, типовые треки развития — универсальны. Привязка к результатам оценки конкретного сотрудника — on-prem.

Этап 1 — Внешний ИИ (шаблон)

Промпт:
Разработай шаблон ИПР для [уровень: специалист/руководитель] на [период]. Включи: структуру (цели, компетенции, мероприятия, сроки, критерии), типовые треки развития (технические, управленческие, soft skills). Формат: заполняемый шаблон.

Этап 2 — On-prem (персонализация)

Промпт:
Заполни шаблон ИПР для сотрудника [ФИО]. Данные: должность [название], результаты оценки [данные], цели сотрудника [цели]. Составь конкретный план с измеримыми критериями.

Задача 3: Опросы вовлечённости — вопросы снаружи, анализ ответов внутри

Гибрид: формулировки вопросов — типовые, могут быть публичными. Анализ реальных ответов сотрудников — персональные данные + управленческая чувствительность.

Этап 1 — Внешний ИИ (вопросы)

Промпт:
Разработай опрос вовлечённости сотрудников. 15 вопросов: 10 закрытых (шкала 1–5), 5 открытых. Темы: удовлетворённость работой, отношения с руководителем, развитие, баланс работы/личной жизни, рекомендация компании как работодателя. Формат: готовый опросник.

Этап 2 — On-prem (анализ)

Промпт:
Проанализируй результаты опроса вовлечённости за [период]. Данные: [агрегированные результаты, без ФИО по отдельным ответам]. Выдели: общий индекс, проблемные зоны по подразделениям, тренды vs прошлый период, рекомендации.
Важно: анализ открытых ответов, где сотрудники могут указать ФИО или описать конкретные ситуации — только on-prem.

Задача 4: Обратная связь по оценке — принципы снаружи, факты о сотруднике внутри

Гибрид: общие принципы формулирования конструктивной обратной связи — универсальны. Содержание с реальными фактами о конкретном сотруднике — on-prem.

Этап 1 — Внешний ИИ (методология)

Промпт:
Разработай гайд для руководителя по проведению review-встречи. Включи: структуру разговора, примеры конструктивной обратной связи (модель SBI: Situation-Behavior-Impact), примеры формулировок для разных сценариев (превышение ожиданий, соответствие, зона роста). Формат: памятка руководителя.

Этап 2 — On-prem (подготовка конкретной встречи)

Промпт:
Подготовь talking points для review-встречи с сотрудником [ФИО]. Данные: должность [название], результаты оценки [данные], цели периода [цели]. Используй модель SBI. Сформулируй обратную связь по каждой компетенции.

Задача 5: Офферы и трудовые договоры — шаблон снаружи, реальные условия внутри

Гибрид: типовой шаблон оффера/договора — универсальный. Реальные условия (оклад, ФИО, дата) — on-prem.

Этап 1 — Внешний ИИ (шаблон)

Промпт:
Составь типовой шаблон оффера на позицию [название] в [отрасль]. Включи: структуру, обязательные разделы (должность, оклад, испытательный срок, соцпакет), формулировки о конфиденциальности, условия о неполной занятости (если применимо). Формат: шаблон с маркерами [ФИО], [ОКЛАД].

Этап 2 — On-prem (заполнение)

Промпт:
Заполни шаблон оффера для кандидата [ФИО]. Данные: должность [название], оклад [сумма], дата выхода [дата], испытательный срок [срок]. Проверь соответствие ТК РФ.

Проверка знаний

Вопрос 1: Почему опрос вовлечённости — гибрид, а не полностью on-prem?

Вопросы типовые (можно в облаке), анализ ответов — on-prem
Потому что опрос проводится онлайн
Потому что опрос анонимный

Вопрос 2: Какой этап reference check — on-prem?

Разработка структуры вопросов
Оценка конкретного кандидата по реальным ответам
Выбор шкалы оценки (1–5)

Вопрос 3: Почему ИПР — гибрид?

Потому что цели — личные
Шаблон универсален (облако), привязка к конкретному сотруднику — on-prem
Потому что сроки разные у всех
