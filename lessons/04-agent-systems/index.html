Уровень 4 — Агенты

4.2 Агент-системы для корпоративного ИИ

15 минут · Как устроен цифровой сотрудник изнутри

До 90% задач в производственной компании можно решать внутри периметра — собственными моделями, без отправки данных в облако. Для этого существуют агент-системы — фреймворки, которые запускают AI-агентов на корпоративных серверах.

Этот урок не про установку и код. Он про то, как устроен цифровой сотрудник и из каких «документов» он состоит — чтобы вы могли ставить задачи ИТ-отделу и проверять результат.

🎯 Зачем это бизнесу

On-premise = данные не покидают компанию. Журналы отказов, параметры плавки, планы добычи, коммерческие прогнозы — всё остаётся внутри.

Агент-система = цифровой сотрудник с памятью. Он не просто отвечает на вопросы — он запоминает, учится, действует по расписанию и интегрируется с вашими системами (1С, SAP, АСУ ТП).

📚 Что изучаем

Три типа агент-систем

Система Для чего Аналогия
OpenClaw Универсальный агент с мессенджерами (Telegram, Русал Пульс, Email). Работает на вашем сервере. Личный помощник на всех каналах связи
Hermes Agent Самообучающийся агент. Создаёт навыки из опыта и запоминает предпочтения пользователя. Опытный сотрудник, который учится на задачах
GoClaw Enterprise-платформа для команд агентов с изоляцией данных между отделами. Корпоративная система управления персоналом

Ключевые файлы агента

Агент — это не «чёрный ящик». Он состоит из документов, как сотрудник из инструкций и личного дела. Вот что значит каждый файл:

Файл Что это Бизнес-аналогия
SOUL.md «Душа» агента — его характер, стиль общения, роль. Например: «Ты опытный инженер-технолог, говоришь кратко и по делу». Должностная инструкция + корпоративная культура
SKILL.md Навык — пошаговая инструкция для типовой задачи. Например: «Как анализировать журнал отказов». Регламент / стандартная операционная процедура
HEARTBEAT.md Расписание периодических проверок. Например: каждое утро в 8:00 — проверить план добычи. График дежурств и планёрки
MEMORY.md Долговременная память — факты, решения, уроки. Например: «Печь №3 даёт брак при температуре >1200°». База знаний / журнал наблюдений
AGENTS.md Общие правила для всех агентов в системе — что можно, что нельзя, красные линии. Кодекс поведения компании
USER.md Профиль пользователя — кто вы, чем занимаетесь, что можно делать без спроса. Личное дело сотрудника + полномочия
TOOLS.md Список инструментов, к которым агент имеет доступ: SAP, 1С, Grafana, Email, Русал Пульс. Перечень доступных программ и систем

🛠️ Как применить

Пример: Агент для контроля добычи

Конфигурация агента

SOUL.md: «Ты аналитик по добыче. Говоришь кратко, используешь данные АСУ ТП. Не делаешь предположений без фактов.»

HEARTBEAT.md: Каждый день в 7:30 — проверить фактическую добычу против плана, выявить отклонения >5%, отправить сводку.

SKILL.md: «Анализ журнала добычи: 1) Загрузить данные из АСУ ТП, 2) Сравнить с планом по буровым, 3) Выделить отклонения, 4) Дать рекомендации, 5) Отправить в Русал Пульс.»

MEMORY.md: «Скважина №47 даёт нестабильный дебит при давлении <15 атм. В марте 2026 был обрыв штанги.»

Результат: Главный инженер получает сводку в Русал Пульс до начала смены — с фактами, отклонениями и рекомендациями. Время ручной сверки сокращается с 2 часов до 5 минут.

Пример: Агент для контроля качества

Конфигурация агента

SOUL.md: «Ты инженер по качеству. Проверяешь соответствие параметров плавки нормативу. Указываешь только отклонения.»

HEARTBEAT.md: Каждые 4 часа — проверить температуру, давление, химсостав последней партии. При отклонении — немедленное уведомление.

TOOLS.md: Доступ к базе АСУ ТП, Русал Пульс для алертов, Excel для отчётов.

Результат: Брак выявляется в реальном времени, а не при отгрузке. Экономия на возвратах и переплавке.

✅ Чеклист принятия результата

  • Агент работает на корпоративном сервере (on-prem)?
  • Данные не уходят во внешние облака?
  • SOUL.md отражает роль и стиль, нужные бизнесу?
  • HEARTBEAT.md покрывает ключевые проверки (не слишком часто)?
  • SKILL.md описывает задачи пошагово, как регламент?
  • MEMORY.md накапливает полезные факты (не мусор)?
  • TOOLS.md ограничен нужными системами (принцип минимальных привилегий)?
  • Есть «человек в цикле» для критичных решений?

📝 Проверка понимания

Вопрос 1: Что определяет файл SOUL.md?

Список инструментов, к которым агент имеет доступ
Характер, стиль общения и роль агента
Расписание периодических проверок

Вопрос 2: Для чего нужен HEARTBEAT.md?

Хранить долговременную память агента
Задать расписание периодических задач агента
Описать пошаговые инструкции для навыков

Вопрос 3: Почему для производственных данных критично использовать on-prem агентов?

On-prem агенты работают быстрее облачных
Данные (параметры добычи, планы, отказы) не покидают периметр компании
On-prem агенты дешевле в обслуживании

💡 Итого

Агент-система — это не магия, а набор документов (SOUL, SKILL, HEARTBEAT, MEMORY), которые определяют поведение цифрового сотрудника.

On-prem гарантирует, что ваши производственные данные остаются внутри компании.

Ключевой навык менеджера: уметь формулировать содержание этих документов — как должностные инструкции для человека, только для агента.
