Уровень 16 — Комплаенс

16.1 Внутренние задачи: on-prem

⏱ 25 мин · для комплаенс-офицеров, юристов, CCO
Закрытый контур (on-prem)
Все задачи этого урока работают с конкретными именами, сделками, транзакциями, персональными данными, whistleblower-обращениями, расследованиями. On-prem — правило по умолчанию для комплаенса.

Почему комплаенс — одна из самых закрытых функций

Комплаенс сочетает чувствительность сразу нескольких функций: как у СВК — карта уязвимостей и whistleblower-риск, как у юриста — privilege и связь с регуляторными последствиями, как у безопасности — расследования конкретных людей. Дополнительная специфика: сам факт «кого именно и зачем проверяли» — это чувствительная информация вне зависимости от результата проверки.

Когда on-prem обязателен:
  • Анализ применимости санкционных ограничений к конкретной сделке/контрагенту
  • KYC/KYB — сбор информации о конкретных контрагентах (бенефициары, структура собственности)
  • Анализ транзакций на признаки подозрительной активности (финансовые данные)
  • Подготовка SAR/STR — сообщений о подозрительных операциях для регулятора
  • Расследование алертов транзакционного мониторинга
  • Due diligence третьих лиц на коррупционные риски (конкретные партнёры)
  • Анализ подарков/представительских расходов (персональные данные сотрудников)
  • Расследование сигналов о коррупции (совместно с СВК)
  • Первичный анализ обращений с горячей линии (риск деанонимизации заявителя)
  • Структурирование материалов расследования, отчёты для руководства
  • Комплаенс-отчётность для совета директоров/аудиткомитета
  • Ответы на запросы регуляторов
  • Подготовка к внешним комплаенс-аудитам (полная картина внутренних процессов)
  • Комплаенс риск-оценка применительно к реальным бизнес-процессам (карта уязвимостей)

1. Анализ применимости санкций к сделке

Интерпретация результатов скрининга по конкретному контрагенту и сделке — решение, можно ли проводить операцию.

Пример промпта (on-prem)
Проанализируй применимость санкционных ограничений к сделке:

Данные сделки (конфиденциально):
- Контрагент: [УКАЖИТЕ НАЗВАНИЕ]
- Страна контрагента: [УКАЖИТЕ]
- Страна происхождения товара: [УКАЖИТЕ]
- Вид товара/услуги: [УКАЖИТЕ]
- Сумма: [УКАЖИТЕ]
- Валюта: [УКАЖИТЕ]
- Способ оплаты: [УКАЖИТЕ]

Результаты скрининга (внутренние системы):
- OFAC: [УКАЖИТЕ]
- EU: [УКАЖИТЕ]
- UK: [УКАЖИТЕ]
- Национальные списки: [УКАЖИТЕ]

Сделай:
1. Оценка риска сделки (высокий/средний/низкий/запрещено)
2. Применимые ограничительные меры
3. Необходимые дополнительные проверки
4. Условия, при которых сделка возможна (если возможна)
5. Обоснование для compliance-комитета
6. Документальный след (что зафиксировать)

Результат: обоснованное решение по сделке, экономит 2–3 часа.

2. KYC/KYB контрагента

Сбор и структурирование информации о бенефициарах, структуре собственности, связанных лицах.

Пример промпта
Проведи KYC/KYB анализ контрагента:

Контрагент: [УКАЖИТЕ НАЗВАНИЕ]

Внутренние данные:
- Учредители: [УКАЖИТЕ]
- Бенефициары: [УКАЖИТЕ]
- Структура собственности: [УКАЖИТЕ]
- Связанные лица: [УКАЖИТЕ]
- История изменений: [УКАЖИТЕ]

Сделай:
1. Схема структуры собственности (текстовое описание)
2. Конечные бенефициары (UBO)
3. Связанные лица и их статус (PEP, санкции)
4. Риск-оценка (высокий/средний/низкий)
5. Рекомендации по условиям сотрудничества
6. План обновления информации

Результат: структурированный профиль контрагента, экономит 3–4 часа.

3. Анализ транзакций на подозрительную активность

Интерпретация алертов транзакционного мониторинга, подготовка обоснования.

Пример промпта
Проанализируй алерт транзакционного мониторинга:

Алерт (внутренняя система):
- ID: [УКАЖИТЕ]
- Тип: [УКАЖИТЕ: структурирование/круговое движение/резко увеличившийся объём]
- Счёт: [УКАЖИТЕ (обезличенно, если нужно)]
- Период: [УКАЖИТЕ]
- Сумма: [УКАЖИТЕ]
- Контрагенты операций: [УКАЖИТЕ]

История счёта:
- Среднемесячный оборот: [УКАЖИТЕ]
- Типичные контрагенты: [УКАЖИТЕ]
- Бизнес-оправдание: [УКАЖИТЕ]

Сделай:
1. Описание подозрительного паттерна
2. Сравнение с типичным поведением счёта
3. Вероятные объяснения (легитимные и нет)
4. Необходимые дополнительные данные
5. Рекомендация (SAR/дополнительный анализ/закрыть)
6. Обоснование для файла

Результат: обоснованное заключение по алерту, экономит 2–3 часа.

4. Подготовка SAR/STR

Сообщение о подозрительной операции для регулятора — юридически значимый документ.

Пример промпта
Подготовь SAR (Suspicious Activity Report):

Данные операции (конфиденциально):
- Дата операции: [УКАЖИТЕ]
- Сумма: [УКАЖИТЕ]
- Валюта: [УКАЖИТЕ]
- Отправитель: [УКАЖИТЕ]
- Получатель: [УКАЖИТЕ]
- Основание для подозрения: [УКАЖИТЕ]
- История взаимоотношений: [УКАЖИТЕ]
- Предыдущие алерты: [УКАЖИТЕ]

Сделай:
1. Резюме подозрительной активности
2. Детальное описание транзакций (хронология)
3. Обоснование подозрения (почему считаем подозрительным)
4. Связанные лица и счета
5. Предыдущая история (если есть)
6. Действия, предпринятые компанией
7. Приложения (список документов)

Формат: в соответствии с требованиями [УКАЖИТЕ РЕГУЛЯТОРА]

Результат: готовый SAR, экономит 3–4 часа.

5. Расследование сигналов о коррупции

Структурирование материалов, подготовка заключения для комитета по этике.

Пример промпта
Проведи расследование сигнала о коррупции:

Источник сигнала: [УКАЖИТЕ: горячая линия/аудит/внутренний контроль]

Содержание сигнала:
[УКАЖИТЕ СУТЬ (без идентификации источника, если whistleblower)]

Внутренние данные:
- Должностное лицо: [УКАЖИТЕ]
- История подарков/представительских расходов: [УКАЖИТЕ]
- Контрагенты: [УКАЖИТЕ]
- Документы: [УКАЖИТЕ]

Сделай:
1. Хронология событий
2. Анализ фактов (что подтверждается, что нет)
3. Оценка нарушений (какие политики нарушены)
4. Вовлечённые лица
5. Рекомендации (дисциплинарные/административные/уголовные)
6. План корректирующих мероприятий
7. Отчёт для комитета по этике

Результат: структурированное расследование, экономит 4–5 часов.

6. Анализ обращений с горячей линии

Первичная категоризация и приоритизация whistleblower-обращений.

Пример промпта
Проанализируй обращение с горячей линии:

Обращение (анонимное):
[ВСТАВЬТЕ ТЕКСТ ОБРАЩЕНИЯ]

Сделай:
1. Категория нарушения (коррупция/финансовые махинации/нарушение ТБ/дискриминация/другое)
2. Уровень серьёзности (критический/высокий/средний/низкий)
3. Подразделения/лица, упомянутые в обращении
4. Необходимые первые шаги расследования
5. Риски (реторсанкции, утечка, репутация)
6. Рекомендация по срокам расследования
7. Приоритет относительно других дел

Важно: сохранить анонимность источника.

Результат: категоризация и план действий, экономит 1–2 часа.

7. Комплаенс-отчётность для совета директоров

Подготовка структурированного отчёта для высшего органа управления.

Пример промпта
Подготовь комплаенс-отчёт для совета директоров:

Период: [УКАЖИТЕ]

Внутренние данные:
- Количество проверок: [УКАЖИТЕ]
- Выявленные нарушения: [УКАЖИТЕ]
- SAR/STR подано: [УКАЖИТЕ]
- Санкционные алерты: [УКАЖИТЕ]
- Обращения с горячей линии: [УКАЖИТЕ]
- Расследования: [УКАЖИТЕ]
- Обучение пройдено: [УКАЖИТЕ]% сотрудников

Сделай:
1. Executive summary (1 страница)
2. Ключевые риски периода
3. Тренды (сравнение с прошлым периодом)
4. Значимые инциденты (анонимизированно)
5. Статус корректирующих мероприятий
6. Рекомендации руководству
7. План на следующий период

Тон: профессиональный, без паники, с фокусом на рисках.

Результат: готовый отчёт, экономит 3–4 часа.

Проверка знаний

Вопрос 1: Почему первичный анализ whistleblower-обращений — только on-prem?

Риск деанонимизации заявителя и компрометации всей системы горячей линии
Только если обращений меньше 10 в месяц
Можно в облаке, если обезличить текст обращения

Вопрос 2: SAR/STR — какой контур?

On-prem — юридически значимый документ с конфиденциальными данными
Внешний ИИ — регуляторы публикуют требования к SAR
Гибрид — структура в облаке, факты внутри

Вопрос 3: Почему KYC/KYB — on-prem?

Бенефициары и структура собственности — персональные данные
Публичные реестры доступны всем, значит можно в облаке
Можно в облаке, если обезличить названия
