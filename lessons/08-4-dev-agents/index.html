8.4 Разработка и агенты

15 мин · для руководителей проектов и сметчиков · когда нужна команда

В строительстве AI — это не только промпты в ChatGPT. Настоящая эффективность появляется там, где модель интегрируется с вашими системами: сметными программами, BIM, ERP, базой чертежей. Это требует команды разработки или готовых агентных решений.

В этом уроке: три корзины сложности — от задач, которые решает подрядчик, до задач, где нужна своя команда разработки или агентная система.

1. Где нужна команда разработки

Некоторые задачи в строительстве невозможно решить готовыми AI-инструментами. Нужна интеграция с внутренними системами, точные расчёты и юридическая ответственность.

Задача Почему нужна разработка Что строить
Сметы по ГЭСН/ФЕР Гранд-Смета и аналоги — закрытые форматы, нужен расчётный движок Интеграция с Гранд-Смета через API или плагин + собственный расчётный движок для проверки
Актуализация под индексы Минстроя Индексы меняются ежеквартально, ручное обновление — риск ошибки Парсер индексов Минстроя + авто-обновление сметных нормативов
Расчёт НМЦК Начальная максимальная цена контракта — юридически чувствительна, ошибка = перетендер Модуль расчёта с трассировкой источников, экспортом обоснования и аудит-логом
Извлечение спецификаций из чертежей Vision-модели дают черновик, для продакшена нужен пайплайн верификации Vision-модель (Qwen/GPT-4V) → структурирование → ручная проверка → экспорт в смету
ВОР из BIM BIM — источник истины, но ведомость объёмов работ нужна в сметной программе Коннектор BIM → извлечение ВОР → форматирование под Гранд-Смета
Сверка с ERP/складом Материалы на складе ≠ материалы в смете, нужна двусторонняя синхронизация Интеграция с ERP (1С, SAP) для сверки остатков и заказов
Коллизии Navisworks и Tekla находят коллизии, но интеграция отчётов в workflow — своя задача API Navisworks → извлечение отчётов → автоматическое назначение ответственных → дашборд
3D-рендеры Генеративные инструменты (Stable Diffusion, DALL-E) нуждаются в интеграции с BIM-моделями Pipeline: BIM → экспорт видов → generative render → сравнение с исходной геометрией
Ключевой принцип: если задача требует интеграции с двумя и более внутренними системами — это работа для команды разработки, не для «подрядчика за 2 недели».

2. Где нужны агенты

Агентные системы — это не просто LLM, а автономные процессы, которые могут ходить в системы, собирать данные, принимать решения и докладывать человеку.

Анализ рисков проекта

Агент анализирует смету, график, погоду, поставщиков и выдаёт risk-score по каждому риску с рекомендациями.

Работа агента
1. Получил смету и график
2. Проверил индексы Минстроя — актуальны
3. Проверил прогноз погоды на период фундамента — риск дождей
4. Проверил поставщиков — 2 из 5 с задержками в прошлом квартале
5. Результат: отчёт с 3 критичными рисками + рекомендации

Актуализация смет под индексы

Агентный процесс: раз в квартал агент проверяет индексы Минстроя, сравнивает с текущими сметами, предлагает корректировки.

Работа агента
1. Загрузил актуальные индексы с сайта Минстроя
2. Сравнил с 15 активными сметами
3. Нашёл 3 сметы с устаревшими индексами
4. Рассчитал дельту (смета завышена на 8%)
5. Отправил сметчику на согласование

Сверка ВОР с закупками и остатками

Агент ходит в ERP, проверяет: заказаны ли материалы по ВОР? Есть ли на складе? Когда поставка?

Работа агента
1. Получил ВОР на следующий месяц
2. Зашёл в ERP — проверил остатки: цемента не хватает
3. Проверил заказы: поставка цемента через 2 недели
4. ВОР требует цемент через 1 неделю
5. Результат: уведомление о риске срыва сроков

Проверка смет подрядчика

Агент достаёт рыночные цены из открытых источников (или вашей базы) и сверяет со сметой подрядчика.

Работа агента
1. Получил смету подрядчика на 45 млн
2. Проверил 120 позиций по базе рыночных цен
3. Нашёл 15 позиций завышенными на 12-30%
4. Сформировал отчёт с рекомендациями по переговорам

3. Пилотирование: с чего начать

Не пытайтесь автоматизировать всё сразу. Стройка — консервативная отрасль, и ошибка в смете стоит дороже, чем задержка на месяц.

Корзина 1: Быстрый эффект, низкий риск

  • Извлечение спецификаций из чертежей — Vision-модель + ручная проверка. Через 2-3 итерации точность 90%+.
  • Анализ рисков проекта — агент на основе открытых данных + ваших смет. Не трогает системы.
  • 3D-рендеры для презентаций — generative AI, не влияет на расчёты.

Корзина 2: Средний риск, нужна интеграция

  • Актуализация смет под индексы — нужен парсер Минстроя + доступ к сметам. Human-in-the-loop обязателен.
  • Сверка ВОР с закупками — агент ходит в ERP, нужны права доступа и аудит-лог.
  • Коллизии из Navisworks — интеграция с BIM, но результат — отчёт, не изменение модели.

Корзина 3: Высокий риск, нужна команда

  • Сметы по ГЭСН/ФЕР с авто-расчётом — ошибка = некорректный тендер или срыв сроков.
  • Расчёт НМЦК — юридическая ответственность, нужна трассировка и экспертиза.
  • Проверка смет подрядчика с автоматическим формированием претензий — последствия для контракта.
Human-in-the-loop обязателен для корзин 2 и 3. AI предлагает, человек подтверждает. Это не замедление — это страховка. В стройке ошибка в смете на 5% может стоить десятков миллионов.
