1.3 · Цепочки запросов: от простого к сложному

🎯 Без кода · ✅ Интерактивный квиз

🎯 Зачем это бизнесу

Сложные задачи нельзя решить одним промптом. Попытка запросить всё сразу приводит к поверхностным ответам.

Пример: Анализ рынка поставщиков включает: поиск → сравнение → оценку рисков → рекомендацию. Каждый шаг строится на предыдущем.

Экономия: Цепочка из 4 промптов = 30 минут. Ручной анализ = 4 часа.

📚 Что изучаем

Что такое цепочка запросов

Цепочка — это последовательность промптов, где каждый следующий использует результат предыдущего.

🔍 Сбор
→
📊 Структура
→
🔎 Анализ
→
✅ Решение

Пример: подбор поставщика упаковки

Шаг 1: Сбор информации
Ты — закупщик FMCG.
Найди 5 крупнейших производителей картонной упаковки в Москве.
Для каждого: название, сайт, телефон, минимальный заказ, срок изготовления.
Шаг 2: Структурирование
На основе результатов предыдущего шага создай таблицу:
| Поставщик | Сайт | Телефон | Мин. заказ | Срок | Примечания |

Отметь поставщиков с сроком > 14 дней.
Шаг 3: Анализ рисков
Проанализируй таблицу:
1. Какие риски видишь? (сроки, минималки, неизвестные игроки)
2. Кто лидер по соотношению цена/срок?
3. Что проверить дополнительно?
Шаг 4: Рекомендация
На основе анализа рекомендуй 2 поставщика для первого контакта.
Обоснуй выбор. Укажи, какие вопросы задать при звонке.

🛠️ Как применить

Правило: один промпт = одна задача

❌ Неправильно: "Найди поставщиков, сравни их, оцени риски и рекомендуй лучшего"

✅ Правильно: Разбить на 4 отдельных промпта

Почему: AI даёт поверхностный ответ, когда просят слишком много сразу.

Когда использовать цепочки

ПризнакПример
Задача требует > 3 действийАнализ рынка + сравнение + рекомендация
Каждый шаг зависит от предыдущегоНельзя сравнить, пока не собрал данные
Нужен промежуточный результатТаблица для проверки перед анализом
Один промпт даёт слишком длинный ответОтвет обрезается, теряются детали

Шаблоны цепочек для бизнеса

Цепочка "Тендер"

  1. Собрать требования к поставщику
  2. Составить список критериев оценки
  3. Собрать предложения (или найти поставщиков)
  4. Сравнить по критериям
  5. Оценить риски
  6. Сформулировать рекомендацию

Цепочка "Отчёт"

  1. Собрать данные за период
  2. Вычислить ключевые метрики
  3. Выявить тренды и аномалии
  4. Сравнить с прошлым периодом
  5. Сформулировать выводы и рекомендации

Цепочка "Договор"

  1. Проанализировать ключевые условия
  2. Найти риски для каждой стороны
  3. Сравнить с шаблоном/предыдущими договорами
  4. Сформулировать замечания
  5. Подготовить список вопросов контрагенту

👨‍💻 Как делегировать разработчику

Шаблон задачи

"Нужно автоматизировать цепочку анализа [процесс].

Текущая цепочка (ручная):
1. [шаг 1] → [шаг 2] → [шаг 3]

Требования:
1. Сохранение промежуточных результатов между шагами
2. Возможность остановиться и проверить результат после каждого шага
3. Если результат шага некорректный — возможность повторить
4. Итоговый отчёт в формате [указать]
5. Уведомление на email при завершении

Интеграции: [1С / Excel / CRM / email]
Срок: [дата]"

✅ Чеклист принятия результата

  • Каждый шаг имеет чёткую входную и выходную задачу?
  • Можно проверить результат промежуточного шага?
  • Если шаг 2 некорректен — можно перезапустить только его?
  • Итоговый результат строится на всех предыдущих шагах?
  • Нет шагов, которые можно объединить без потери качества?
  • Цепочка покрывает всю задачу от начала до конца?

🎯 Главное из урока

Сложные задачи разбивайте на цепочку: сбор → структурирование → анализ → рекомендация. Проверяйте результат каждого шага. Одна задача = один промпт. Это даёт качество, которого невозможно достичь одним запросом.
