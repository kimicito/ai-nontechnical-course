8.1 Реальность: что ИИ умеет в стройке

15 мин · для сметчиков, прорабов, инженеров · капстроительство

Главный тезис этого урока: смета — это не текст. LLM не считает по ГЭСН/ФЕР, не знает коэффициентов и не понимает ресурсный метод. Если вы скармливаете нейросети смету как текст и ждёте точного расчёта — вы обречены на галлюцинации.

Но это не значит, что ИИ бесполезен в стройке. Он полезен — но в задачах, где не нужен точный расчёт по нормативам. Разберём, где он работает сейчас, а где — пока нет.

Почему стройка сложнее закупок

В закупках есть конкретные числа: цена, срок, количество. LLM сравнивает 5 КП — и делает это хорошо. В стройке всё иначе:

Что делает стройку сложной для LLM:
  • Нормативные базы: ГЭСН, ФЕР, ТСН, ФССЦ — десятки тысяч позиций с коэффициентами
  • Ресурсный метод: стоимость = норма × цена ресурса × коэффициенты. LLM не знает текущие цены ресурсов
  • Коэффициенты: Кп (прибыль), Кнр (налоги), Кпз (погрузка), индексы пересчёта — они меняются по регионам и годам
  • Локальные сметы: каждый объект уникален, нет двух одинаковых смет
  • Сметные нормы ≠ рыночные цены: ГЭСН даёт норму труда и машин, а цены материалов — по рынку
Пример ошибки: Вы загружаете PDF-смету в ChatGPT и просите «проверь расчёт». LLM видит текст, но не видит за таблицей: какие нормы применены, какие коэффициенты, какие индексы. Он может сказать «всё верно» или найти очевидную арифметическую ошибку — но пропустит сметную ошибку на 30%.

Три корзины задач

Разделим задачи строительного подразделения на три группы — по готовности ИИ к их решению.

🟢 Корзина 1: Быстро сами — прямо сейчас

LLM справляется без доработки. ROI за дни и недели.

  • Поиск по нормативам: «В каком сборнике ГЭСН найти устройство буронабивных свай?»
  • Расшифровка сметных позиций: «Что означает позиция ФЕРм 01-01-001-01?»
  • Перевод ВОР в текст: «По ведомости объёмов работ составь технологическую карту»
  • Генерация текстовых документов: пояснительные записки, технологические карты, письма
  • Сравнение локальных смет по тексту: «Смета А и смета Б — в чём разница в подходе?»
  • Проверка логики (не расчёта!): «В смете на фундамент есть работы по кровле — это ошибка?»
  • Извлечение данных из чертежей (OCR): распознать размеры, марки, количества

🟡 Корзина 2: Нужна разработка — месяцы

Требуется интеграция с сметным софтом, базами нормативов или кастомная логика.

  • Автоматический расчёт сметы по ВОР: связать объёмы работ с ГЭСН/ФЕР, применить коэффициенты
  • Проверка сметы по нормам: автоматическая валидация: правильно ли подобраны расценки, не пропущены ли работы
  • Прогнозирование стоимости: модель на исторических данных предсказывает итоговую смету по черновому ВОР
  • Сопоставление чертежа и сметы: проверить, что все элементы на чертеже учтены в смете
  • Извлечение ВОР из BIM-модели: автоматический подсчёт объёмов из 3D-модели

🔴 Корзина 3: Нужны агенты — годы

Требуется мультиагентная система с доступом к внешним данным и автономным принятием решений.

  • Полная автоматизация сметного дела: от чертежа до утверждённой сметы без человека
  • Автоматический контроль цен ресурсов: агент мониторит рынок и обновляет смету в реальном времени
  • Прогнозирование рисков строительства: агент анализирует погоду, поставщиков, субподрядчиков и предупреждает о срывах
  • Автоматическое управление изменениями: при изменении проекта агент пересчитывает всё: смету, график, ресурсы

С чего пилотировать: корзина 1

Не начинайте с автоматического расчёта сметы — это корзина 2, нужна разработка. Начните с задач, которые дают результат за неделю.

Шаг 1: Поиск по нормативам

Пример промпта
Мне нужно найти норму на устройство монолитного железобетонного фундамента ленточного.

Вопросы:
1. В каком сборнике ГЭСН или ФЕР искать?
2. Какие расценки подходят для фундамента глубиной 1.5 м, шириной 0.4 м?
3. Какие ресурсы входят в норму (бетон, арматура, опалубка, трудозатраты)?
4. Какие коэффициенты обычно применяют к этой норме?

LLM не даст точный расчёт, но укажет направление — сэкономит 30 минут на поиске по сборникам.

Шаг 2: Расшифровка сметных позиций

Пример промпта
Вот список позиций из локальной сметы:
- ФЕРм 08-02-001-01
- ГЭСНм 81-05-001-01
- ТСН-2001-08-02-001

Расшифруй каждую позицию:
1. Полное наименование работы
2. Единица измерения
3. Основные ресурсы (материалы, труд, машины)
4. Типовая область применения

Шаг 3: Проверка логики сметы

Пример промпта
Я загрузил PDF локальной сметы на капитальный ремонт здания.

Проверь логику (НЕ считай суммы):
1. Все ли разделы сметы соответствуют видам работ в пояснительной записке?
2. Нет ли работ, которые не должны быть в этом виде ремонта?
3. Есть ли пропущенные работы, типичные для капремонта?
4. Соответствует ли последовательность работ технологии?

Выдай список подозрительных мест с пояснениями.

Шаг 4: Генерация пояснительных записок

Пример промпта
На основе локальной сметы (PDF во вложении) составь пояснительную записку:

1. Общие положения (объект, заказчик, подрядчик)
2. Основание для составления сметы
3. Перечень принятых расценок и норм
4. Особенности применения коэффициентов
5. Итоговая стоимость и сроки работ

Формат: официальный, по ГОСТ Р 21.1101-2013.

Что НЕ делать с LLM в стройке

Красные флаги — задачи, где LLM обманет:
  • Считать смету «с нуля» по тексту: LLM не знает текущие цены ресурсов, индексы и коэффициенты
  • Проверять арифметику сложных смет: может пропустить ошибку в коэффициенте, но «подтвердить» сумму
  • Подбирать нормы по описанию работ: LLM может предложить похожую, но неправильную расценку
  • Использовать для обоснования сметы в суде или экспертизе: выводы LLM не имеют юридической силы
Правило безопасности: LLM — помощник сметчика, не замена сметчику. Он ускоряет рутину (поиск, текст, проверка логики), но точный расчёт — всё ещё за человеком и сметным ПО (ГРАНД-Смета, А0, Смета.ру).

Проверка знаний

Вопрос 1: Почему LLM не может точно посчитать смету по ГЭСН/ФЕР?

Потому что ГЭСН запрещает использование ИИ
Потому что LLM не знает текущие цены ресурсов, коэффициенты и индексы
Потому что сметы слишком длинные для контекста

Вопрос 2: К какой корзине относится задача «проверить, нет ли в смете работ, не подходящих для капремонта»?

Корзина 1 — быстро сами
Корзина 2 — нужна разработка
Корзина 3 — нужны агенты
