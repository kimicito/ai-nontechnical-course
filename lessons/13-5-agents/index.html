13.5 Агентские сценарии для Логистики

⏱ 12 мин · отслеживание грузов, маршруты, претензии
Чем агент отличается от языковой модели?
LLM отвечает на вопросы. Агент работает постоянно: отслеживает грузы, анализирует задержки, предупреждает — без команды «где мой груз?». Подробнее — Уровень 4.

Сценарий 1: Агент отслеживания грузов (n8n + TMS API)

Задача: отслеживать статусы грузов, предупреждать о задержках, обновлять клиентов.

[Cron: каждые 30 минут]
↓
[API TMS / перевозчика: статусы активных рейсов]
↓
[Агент: анализ отклонений]
├─→ Груз в пути дольше нормы: предупреждение
├─→ Статус не обновлялся >24ч: запрос у перевозчика
└─→ Груз прибыл: уведомление клиенту
↓
[Автоматические уведомления]
├─ Клиенту: «Груз прибыл в пункт Х, ожидайте доставку ЗАВТРА»
├─ Логисту: «Рейс N задерживается на 2 дня, причина: [из API]»
└─ Складу: «Планируйте приёмку на [дата]»
↓
[Ежедневный дашборд: все рейсы, статусы, риски]
Результат: клиенты узнают о задержках до того, как начнут звонить. Экономия 20–30 звонков в день.

Сценарий 2: Агент оптимизации маршрутов (OpenClaw-агент)

Задача: анализировать исторические данные по маршрутам, предлагать альтернативы, оценивать стоимость.

[Запрос: «Оптимизировать маршрут Москва → Новосибирск»]
↓
[Агент: сбор данных]
├─→ Исторические маршруты (время, стоимость, инциденты)
├─→ Текущие тарифы перевозчиков
├─→ Пограничные условия (очереди, погода)
└─→ Требования клиента (срок, температура)
↓
[On-prem LLM: сравнительный анализ]
├─→ Вариант А: ЖД, срок, стоимость, риски
├─→ Вариант Б: Авто, срок, стоимость, риски
└─→ Вариант В: Мультимодальный, срок, стоимость, риски
↓
[Рекомендация с обоснованием]
├─ Оптимальный по цене
├─ Оптимальный по сроку
└─ Оптимальный по риску
↓
[Отчёт логисту]

Сценарий 3: Агент претензий перевозчикам (гибридный)

Задача: собирать данные по просрочкам и повреждениям, готовить претензии, отслеживать сроки ответа.

[Триггер: статус рейса = «доставлен»]
↓
[Агент: сбор данных]
├─→ Плановая vs фактическая дата доставки
├─→ Акты осмотра (повреждения)
├─→ Трекинг: фактический маршрут
└─→ Переписка с перевозчиком
↓
[On-prem LLM: оценка оснований для претензии]
├─→ Задержка > допустимой: расчёт штрафа
├─→ Повреждение: оценка по акту
└─→ Недостача: сверка по накладной
↓
[Автоматическая генерация претензии]
├─→ Ссылки на договор, пункты
├─→ Расчёт суммы требования
└─→ Срок для ответа
↓
[Отправка перевозчику + создание задачи в СЭД]
↓
[Мониторинг: ответ не получен в срок → эскалация]
Важно: претензии содержат коммерческую информацию (суммы, условия договоров). Вся обработка — on-prem.
