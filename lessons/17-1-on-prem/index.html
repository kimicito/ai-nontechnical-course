Уровень 17 — Персонал

17.1 Внутренние задачи HR: on-prem

⏱ 20 мин · задачи, где фигурирует конкретный человек
Главный принцип урока
HR работает с персональными данными почти в каждой задаче. Правило: если в тексте есть ФИО, оклад, оценка или личная ситуация конкретного человека — только on-prem.

Задача 1: Скрининг резюме и оценка кандидатов

Почему on-prem: резюме содержат персональные данные (ФИО, контакты, предыдущие места работы, зарплата). Даже анализ соответствия требованиям — обработка персональных данных.

Что делает AI

  • Сравнивает резюме с требованиями вакансии
  • Выделяет сильные и слабые стороны кандидата
  • Формирует краткую аннотацию для HR
Промпт:
Требования вакансии: [текст требований].
Резюме кандидата: [текст резюме].

Проанализируй соответствие по каждому требованию. Выдели сильные стороны, риски, вопросы для интервью. Составь краткую аннотацию (макс. 150 слов).
Результат: экономия 10–15 минут на каждое резюме. Команда из 3 HR может обрабатывать в 2 раза больше кандидатов.

Задача 2: Подготовка кадровых приказов

Почему on-prem: приказы содержат ФИО, оклады, должности, даты — полный набор персональных данных. Юридически значимые документы.

Что делает AI

  • Генерирует текст приказа по шаблону
  • Проверяет соответствие ТК РФ (или локальному законодательству)
  • Формирует список приложений
Промпт:
Составь приказ о приёме на работу. Данные: ФИО [имя], должность [название], отдел [название], оклад [сумма], дата приёма [дата], испытательный срок [срок]. Проверь соответствие ТК РФ. Укажи необходимые приложения.

Задача 3: Ведение и структурирование личных дел

Почему on-prem: личные дела — персональные данные в полном смысле. Даже текстовая часть содержит биографические сведения.

Промпт:
Структурируй следующие документы личного дела [ФИО] в стандартизированный формат: [список документов]. Для каждого документа укажи дату, тип, соответствие требованиям кадрового делопроизводства.

Задача 4: Кадровая отчётность (текучесть, штатное расписание)

Почему on-prem: аналитика по подразделениям, фамилии уволившихся, причины ухода — управленческая чувствительная информация.

Промпт:
Данные по текучести за [период]: [таблица с данными]. Проанализируй: тренды по подразделениям, средний срок работы, основные причины ухода (по категориям). Составь сводку для руководства с рекомендациями.

Задача 5: Оценка персонала (performance review)

Почему on-prem: оценка на реальных сотрудниках — крайне чувствительная информация. Влияет на карьеру, вознаграждение, мотивацию конкретных людей.

Что делает AI

  • Структурирует обратную связь из 360-оценок
  • Формирует сводку по компетенциям
  • Подготавливает формулировки для review-встречи
Промпт:
Результаты 360-оценки сотрудника [ФИО] за [период]: [данные оценок]. Структурируй обратную связь по компетенциям. Выдели сильные стороны, зоны роста, рекомендации. Подготовь формулировки для review-встречи.

Задача 6: Индивидуальные планы развития (ИПР)

Почему on-prem: ИПР привязан к конкретному сотруднику, его оценке, карьерным целям.

Промпт:
Составь ИПР для сотрудника [ФИО] на [период]. Данные: текущая должность [название], результаты оценки [данные], цели сотрудника [цели]. Включи: развиваемые компетенции, мероприятия, сроки, критерии успеха, поддержку руководителя.

Задача 7: Программы кадрового резерва и преемственности

Почему on-prem: «кто на кого претендует» — политически чувствительная информация внутри компании. Утечка может вызвать конфликты и отток ключевых сотрудников.

Промпт:
На основе матрицы потенциала и оценки ключевых сотрудников: [данные обезличенные — без ФИО, только ID], сформируй предложения по кадровому резерву на критические позиции. Укажи: позиции, требуемые компетенции, сроки подготовки, риски отсутствия преемника.

Задача 8: Расчёт и обоснование ФОТ, грейдирование

Почему on-prem: реальные оклады, грейды конкретных должностей — конфиденциальная информация. AI обрабатывает текстовую часть обоснования, а не математический расчёт.

Промпт:
Обоснуй предложение по грейдированию должности [название]. Данные: текущий грейд [номер], рыночная медиана [диапазон], внутренняя справедливость (соотношение с похожими ролями). Сформулируй аргументы для руководства.

Задача 9: Анализ причин увольнений (exit-интервью)

Почему on-prem: exit-интервью могут содержать критику руководства, личные причины, жалобы — всё это персональные данные + риск деанонимизации.

Промпт:
Проведи анализ причин увольнений на основе exit-интервью за [период]: [тексты интервью, обезличенные]. Сгруппируй причины по категориям. Выдели системные проблемы и рекомендации по их устранению.

Задача 10: Дисциплинарные взыскания

Почему on-prem: юридически значимые документы с персональными данными. Требуют точности формулировок.

Промпт:
Составь проект приказа о дисциплинарном взыскании. Данные: ФИО [имя], должность [название], нарушение [описание], дата [дата], свидетели [список]. Проверь соответствие ТК РФ, статьи 192, 193. Укажи правовые основания.

Задача 11: Трудовые споры, позиция работодателя

Почему on-prem: конкретный сотрудник, его трудовой спор, стратегия защиты компании — адвокатская тайна + коммерческая тайна.

Промпт (совместно с юристом):
Ситуация: сотрудник [ФИО] обратился с требованием [суть требования]. Факты: [контекст]. Подготовь позицию работодателя: правовое обоснование, риски, возможные сценарии разрешения. Отметь слабые места позиции.

Задача 12: Жалобы на харассмент/дискриминацию

Почему on-prem: максимальная чувствительность. Риск деанонимизации заявителя, вторичная травматизация, репутационные риски для компании.

Промпт:
Составь план первичной обработки жалобы на [тип: харассмент/дискриминацию]. Включи: этапы расследования, необходимые документы, требования к конфиденциальности, сроки, форму итогового документа. Укажи, что требует привлечения внешнего юриста.

Задача 13: Отчёты для руководства по HR-метрикам

Почему on-prem: управленческая информация: ФОТ, текучесть по подразделениям, производительность труда — стратегические данные.

Промпт:
На основе HR-метрик за [период]: [данные], составь аналитическую сводку для руководства. Включи: ключевые тренды, отклонения от плана, риски, рекомендации. Формат: 1 страница, структурировано, с выводами.

Задача 14: Reference check конкретных кандидатов

Почему on-prem: reference check включает персональные данные не только кандидата, но и третьих лиц (рекомендателей, бывших руководителей).

Промпт:
Результаты reference check кандидата [ФИО]: [записи разговоров/письма рекомендаций]. Структурируй информацию по категориям: профессиональные качества, коммуникация, надёжность, зоны роста. Сформируй сводку для принятия решения.

Проверка знаний

Вопрос 1: Почему даже подготовка простой справки по запросу сотрудника — on-prem?

Содержит персональные данные (ФИО, должность, оклад)
Потому что справка — внутренний документ
Потому что справки редко запрашивают

Вопрос 2: Какая HR-задача является наиболее чувствительной из перечисленных?

Оценка персонала (performance review)
Жалобы на харассмент/дискриминацию
Кадровое резервирование и преемственность

Вопрос 3: Почему анализ рыночных зарплат (бенчмаркинг) — задача для внешнего ИИ, а не on-prem?

Анализируются публичные обзоры зарплат, не конкретные оклады сотрудников
Потому что методология универсальна
Потому что внешний ИИ точнее в расчётах
