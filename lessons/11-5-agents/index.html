11.5 Агентские сценарии для Качества

⏱ 12 мин · агенты для контроля качества и претензий
Чем агент отличается от языковой модели?
LLM анализирует по запросу. Агент работает постоянно: получает данные, классифицирует, маршрутизирует, отслеживает сроки — без участия человека. Подробнее — Уровень 4.

Сценарий 1: Агент обработки претензий (n8n + LLM)

Задача: при поступлении претензии — классифицировать, назначить ответственного, отслеживать сроки.

[Email / СЭД: новая претензия от клиента]
↓
[n8n: извлечение вложений и текста]
↓
[LLM: классификация]
├─→ Тип: качество продукта / логистика / документы / цена
├─→ Критичность: критическая / серьёзная / стандартная
└─→ Потенциальная сумма убытка
↓
[Назначение ответственного по справочнику]
↓
[Создание задачи в СЭД с дедлайном]
├─ Критическая: 24 часа
├─ Серьёзная: 72 часа
└─ Стандартная: 5 дней
↓
[Уведомление ответственному + руководителю при критичности]
↓
[Ежедневный мониторинг: просроченные → эскалация]
Результат: время реакции на претензию снижается с 1–2 дней до часов. Ни одна претензия не теряется.

Сценарий 2: Агент root-cause анализа (OpenClaw-агент)

Задача: при поступлении данных о дефекте — автоматически провести анализ причин, построить диаграмму Исикавы, предложить решения.

[Ввод: описание дефекта + данные (фото, параметры процесса)]
↓
[On-prem агент: сбор исторических данных]
├─→ Аналогичные дефекты за период
├─→ Изменения в процессе (сырьё, оборудование, персонал)
└─→ Корреляции (температура, давление, скорость)
↓
[LLM: построение диаграммы Исикавы]
├─→ 6M: Man, Machine, Material, Method, Measurement, Environment
├─→ Вероятные корневые причины с обоснованием
└─→ Ссылки на похожие кейсы
↓
[Предложение корректирующих действий]
├─ Краткосрочные (устранение последствий)
└─ Долгосрочные (предотвращение повторения)
↓
[Отчёт инженеру по качеству]

Сценарий 3: Агент подготовки 8D-отчётов (гибридный)

Задача: собрать данные по всем этапам 8D, сформировать отчёт, отследить выполнение действий.

[Триггер: создание карточки 8D в системе]
↓
[Агент: сбор данных по этапам]
D1: Команда → список ответственных
D2: Описание проблемы → извлечение из заявки
D3: Временные меры → проверка выполнения
D4: Root cause → из агента root-cause
D5–D7: Корректирующие действия → отслеживание
D8: Поздравление команды → формирование
↓
[LLM: формирование текста отчёта]
↓
[PDF + уведомление команде]
↓
[Мониторинг: D5–D7 не выполнены в срок → напоминание]
