0.3 · Облако vs On-premise: GENAi для конфиденциальных данных

🎯 Без кода · ✅ Квиз в конце

🎯 Зачем это бизнесу

60% компаний не используют AI из-за страха утечки данных. Но между "отправлять всё в ChatGPT" и "не использовать AI" есть третий путь — on-premise размещение.

⚠️ Реальный кейс: Менеджер закупок загрузил список поставщиков с ценами в ChatGPT. Через 3 месяца конкурент знал их стратегию ценообразования. Данные утекли через обучение модели.

📚 Облако vs On-premise: простыми словами

☁️ Облако (Cloud)

Что это: Вы отправляете данные на серверы компании (OpenAI, Anthropic, Google).

✅ Дешево, быстро, не нужен IT-отдел

❌ Данные уходят за периметр компании

🏢 On-premise (Свой сервер)

Что это: AI-модель работает на сервере внутри компании, данные не покидают офис.

✅ Полный контроль данных, конфиденциальность

❌ Дороже, нужен сервер и админ

Аналогия для переговоров

💡 АНАЛОГИЯ

Облако = аренда офиса в бизнес-центре. Удобно, но консьерж видит всех ваших посетителей, а соседи слышат через стенку.

On-premise = свой дом. Вы хозяин, но сами чините краны и платите за ремонт крыши.

🛠️ GENAi: золотая середина

GENAi — это бесплатный интерфейс (как ChatGPT), который работает с моделями на вашем сервере. Сотрудники видят привычный чат, но данные остаются внутри компании.

Когда выбирать GENAi:

  • Работа с коммерческой тайной (цены, поставщики, контракты)
  • Персональные данные сотрудников или клиентов
  • Финансовая отчётность до публикации
  • Стратегические планы и прогнозы
  • Регуляторное требование хранить данные в стране (РФ, КНР, ЕС)

Матрица рисков

🔴 ВЫСОКИЙ

Контракты, цены, M&A

🟡 СРЕДНИЙ

Внутренние отчёты, HR

🟢 НИЗКИЙ

Маркетинг, публичные данные

👨‍💻 Как поставить задачу разработчику

Шаблон для IT-директора: 
"Нужно развернуть локальный LLM для отдела [НАЗВАНИЕ].

Требования:
- Количество пользователей: [N] человек
- Документы: до [N] страниц за раз
- Конфиденциальность: КРИТИЧНО (коммерческая тайна)
- Языки: RU, EN
- Бюджет железа: [сумма]
- Срок: [N] дней

Прошу предложить:
1. Вариант модели (Qwen / Llama / Mistral)
2. Конфигурацию сервера
3. Стоимость лицензий
4. План резервного копирования
5. Инструкцию для пользователей"

✅ Чеклист принятия результата

  • Данные не покидают сервер компании? (проверить firewall)
  • Есть резервное копирование? (что если сервер сгорит)
  • Пользователи могут работать без IT-шника? (интерфейс понятен)
  • Скорость ответа < 10 секунд? (проверить на пиковой нагрузке)
  • Есть процесс обновления модели? (новые версии выходят каждый месяц)

🧰 Инструменты

  • GENAi (openwebui.com) — бесплатный интерфейс, ставится за 30 минут
  • Ollama (ollama.com) — запуск локальных моделей (Llama, Mistral, Qwen)
  • Qwen (Alibaba) — open-source модель для коммерческого использования
  • vLLM — ускорение推理 для on-premise (потребует разработчика)

📝 Проверка понимания

Вопрос 1: Можно ли использовать ChatGPT для анализа договоров с поставщиками?

A) Да, ChatGPT безопасен для любых данных B) Нет, это коммерческая тайна. Нужен on-premise (GENAi + Qwen/Llama) C) Да, если удалить историю после

Вопрос 2: Что такое GENAi?

A) Платный чат-бот от Google B) Бесплатный интерфейс для локальных моделей. Данные не уходят в интернет C) Облачный сервис для хранения данных

Вопрос 3: Какой бюджет нужен для on-premise LLM?

A) Сервер ~$2000-5000 (или GPU ~$100-500/мес) + админ 20% времени B) Бесплатно, достаточно ноутбука C) Только $20/мес как ChatGPT Plus

🎯 Главное из урока

Для публичных данных — облачные модели (ChatGPT, Claude). Для конфиденциальных — on-premise (GENAi + Qwen/Llama). Нет универсального решения — выбирайте по классу данных. И никогда не загружайте коммерческую тайну в публичные чат-боты.
