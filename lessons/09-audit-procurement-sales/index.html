9.2 · Аудит закупок и продаж

🎯 Без кода · ✅ Интерактивный квиз

🎯 Зачем это бизнесу

Закупки и продажи — точки максимального риска: переплаты, скидки без согласования, фиктивные поставщики, необоснованные отступные. Ручная проверка каждой сделки невозможна при объёмах.

AI-аудит закупок и продаж — автоматический анализ всех сделок на соответствие политикам, поиск аномалий и выявление рисков. Менеджер получает рейтинг сделок: зелёные, жёлтые, красные.

Пример: AI проанализировал 200 закупок за квартал и выявил 8 сделок с рисками: 3 — единственщики без обоснования, 2 — изменение реквизитов поставщика, 3 — скидки выше лимита.

📚 Что изучаем

Закупки: что проверяет AI

  • Наличие 3 КП: есть ли три коммерческих предложения, соответствуют ли требованиям закона/политики
  • Лимиты: не превышен ли лимит на закупку, правильно ли выбрана процедура (конкурс, аукцион, единственщик)
  • Обоснование единственщика: есть ли меморандум, соответствует ли критериям (авторский надзор, технологическая уникальность, срочность)
  • Связанные лица: проверка поставщиков на связанность (учредители, адреса, телефоны)
  • История поставщика: новый контрагент, частые смены реквизитов, массовый адрес

Продажи: что проверяет AI

  • Сверка отгрузки и оплаты: отгружено — оплачено, есть ли просрочки
  • Скидки и отступные: не превышают ли лимит, есть ли согласование, кто одобрил
  • Возвраты: аномальная частота возвратов по клиенту, причины (брак, передумал, ошибка менеджера)
  • Дебиторка: просроченные платежи, динамика долга, клиенты с ростом задолженности
  • Ценовые аномалии: резкое изменение цен, скидки в выходные, нетипичные условия

Сигналы рисков: красные, жёлтые, зелёные

🛠️ Как применить

Сценарий: ежемесячный аудит закупок

Промпт для AI
Проанализируй реестр закупок за месяц (файл).

Для каждой сделки проверь:
1. Процедура: конкурс / аукцион / единственщик — соответствует сумме?
2. Единственщик: есть ли обоснование (меморандум)?
3. КП: есть ли 3 предложения (если не единственщик)?
4. Поставщик: проверенный или новый? есть ли в чёрном списке?
5. Связанность: совпадают ли учредители/адреса с другими поставщиками?
6. Сумма: круглая? резко отличается от рыночной?

Выведи:
- Топ-10 рисковых сделок (с обоснованием)
- Сводка: сколько красных, жёлтых, зелёных
- Рекомендации: что проверить вручную

Результат: Excel-таблица с рейтингом каждой сделки. Руководитель видит, на чём сосредоточить внимание аудитора.

Сценарий: мониторинг продаж на аномалии

Промпт для AI
Проанализируй реестр продаж за квартал.

Для каждой сделки проверь:
1. Скидка: не превышает ли лимит? есть ли одобрение?
2. Возврат: частота по клиенту, причины, сумма
3. Дебиторка: сроки оплаты, просрочки, тренд
4. Отступные: обоснованы? согласованы?
5. Цена: не ниже себестоимости? аномально низкая?

Выведи:
- Клиенты с аномальными возвратами
- Менеджеры с частыми скидками выше лимита
- Сделки с ценой ниже себестоимости
- Рекомендации по дебиторке

Результат: Менеджер по продажам получает список клиентов, требующих внимания, и сделок, которые нужно пересмотреть.

👨‍💻 Как делегировать разработчику

Шаблон задачи

"Нужен AI-аудит закупок и продаж.

Закупки — проверяем:
- 3 КП, процедура, лимиты
- Обоснование единственщика
- Связанные лица (по учредителям, адресам)
- История поставщика (новый, чёрный список, смена реквизитов)
- Аномалии сумм

Продажи — проверяем:
- Сверка отгрузки и оплаты
- Скидки и отступные (лимиты, одобрения)
- Возвраты (частота, причины)
- Дебиторка (просрочки, тренды)
- Ценовые аномалии

Выход:
- Дашборд с рейтингом сделок (красный/жёлтый/зелёный)
- Excel-отчёт с топ-рисками
- Алерты в Telegram при красных сигналах

Интеграция: 1С, ЭДО, банк, Telegram
Срок: [дата]"

✅ Чеклист принятия результата

  • Все сделки имеют рейтинг (красный/жёлтый/зелёный)?
  • Проверка 3 КП работает корректно?
  • Связанные лица находятся (не ложные срабатывания)?
  • Скидки сравниваются с утверждёнными лимитами?
  • Дебиторка обновляется в реальном времени?
  • Красные сигналы приходят мгновенно (а не через неделю)?
  • Отчёт понятен директору (не только аудитору)?

🎯 Главное из урока

AI превращает аудит из «проверь всё» в «проверь красное». Автоматический рейтинг сделок, поиск связанных лиц, проверка лимитов — это не замена аудитору, а его инструмент. Менеджер тратит время на решения, а не на просмотр тысяч строк.
