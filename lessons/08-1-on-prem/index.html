Уровень 8 — Дирекция по качеству

8.1 Внутренние задачи: on-prem

20 мин · для специалистов по качеству · закрытый контур

В этом уроке — задачи Дирекции по качеству, которые выполняются с реальными производственными данными. Дефекты, рецептура, параметры процессов, клиентские претензии — всё это производственная тайна и коммерчески чувствительная информация.

Решение: on-prem LLM в закрытом контуре компании. Данные не покидают периметр.

Зачем on-prem для дирекции по качеству

Что нельзя отправлять в облако:
  • Конкретные параметры технологических процессов (температуры, скорости, составы)
  • Рецептура сплавов и химический состав партий
  • Данные лабораторных испытаний с номерами партий
  • Реальные претензии клиентов с объёмами и суммами
  • Исторические данные о дефектах по конкретным линиям
  • Контрактные условия с поставщиками сырья
Что можно сделать в on-prem:
  • Root-cause анализ дефектов по фактическим параметрам процесса
  • Подготовка 8D-отчётов и ответов на рекламации
  • Анализ повторяемости претензий по линиям/партиям/клиентам
  • Внутренний аудит СМК и карта слабых мест
  • Аудит поставщиков сырья по внутренним данным
  • Разработка ТУ с реальными техническими параметрами
  • R&D новых сплавов с экспериментальными данными
  • Cost of quality и KPI для руководства

1. Root-cause анализ дефектов (Ishikawa, 5 Why)

Дефект на линии. Нужно понять корневую причину. LLM анализирует фактические параметры процесса и строит дерево причин.

Пример промпта
На линии №3 произошёл дефект "раковины на поверхности слитка" в партии П-2026-1847.

Фактические параметры процесса:
- Температура заливки: 1480°C (норма 1450-1470°C)
- Скорость охлаждения: 12°C/мин (норма 8-10°C/мин)
- Состав шлака: Al2O3 45%, SiO2 32% (норма Al2O3 40-42%, SiO2 28-30%)
- Время между заливкой и началом прокатки: 4.5 часа (норма 3-4 часа)
- Текущий поставщик флюса: ООО "МеталлургХим"

Проведи root-cause анализ:
1. Построй Ishikawa-диаграмму (6M: Man, Machine, Material, Method, Milieu, Measurement)
2. Примени 5 Why к главной ветке
3. Оцени вероятность каждой причины (высокая/средняя/низкая)
4. Дай рекомендации по устранению
Результат: структурированный анализ с деревом причин, приоритизацией и конкретными рекомендациями. Экономия 2-4 часа инженера.

2. Обработка рекламаций и 8D-отчёты

Клиент прислал претензию. Нужно подготовить 8D-ответ с расследованием, корректирующими действиями и превентивными мерами.

Пример промпта
Клиент "АвтоПром ООО" прислал претензию на партию АП-2026-0523:
"В 15% слитков выявлено превышение допустимого содержания серы (факт 0.045%, ТУ ≤ 0.035%)"

Внутренние данные:
- Партия произведена на линии №2, 12-15 июня 2026
- Поставщик чугуна: ООО "УралСырье" (партия УС-2026-4412)
- Лабораторные данные: сера в чугуне 0.038% (по сертификату поставщика 0.025%)
- Данные десульфурации: расход флюса 12 кг/т (норма 15-18 кг/т)
- Технолог: Иванов П.С. (смена 2, опыт 2 года)

Подготовь 8D-отчёт:
D0: Подготовка (проверка критичности)
D1: Команда (роли)
D2: Описание проблемы (5W2H)
D3: Временное сдерживающее действие
D4: Root-cause (Ishikawa + 5 Why)
D5: Корректирующее действие
D6: Проверка эффективности (критерии)
D7: Превентивные действия
D8: Поздравление команды и закрытие

3. Анализ повторяемости претензий

Репутационно и коммерчески чувствительно. Нужно понять: по каким линиям/клиентам/партиям претензии повторяются.

Пример промпта
У меня реестр претензий за 2025-2026 годы (файл Претензии.xlsx).
Колонки: дата, клиент, линия, тип дефекта, партия, сумма убытка, статус.

Проанализируй:
1. Топ-3 клиента по частоте претензий (количество)
2. Топ-3 типа дефекта по сумме убытка
3. Линии с максимальной повторяемостью (какие линии "болеют")
4. Сезонность претензий (месяцы пиков)
5. Динамика: растёт/падает количество претензий YoY
6. Корреляция: тип дефекта vs линия vs поставщик сырья
7. Рекомендации: на что сфокусировать ресурсы качества

4. Внутренний аудит СМК

Карта слабых мест системы менеджмента качества. LLM анализирует результаты внутренних аудитов и находит системные проблемы.

Пример промпта
Результаты внутренних аудитов СМК за 2025-2026 (файл Аудиты.xlsx):
Колонки: дата аудита, подразделение, пункт ISO 9001, найденное несоответствие, оценка риска, срок устранения, факт устранения, просрочено (да/нет).

Сделай:
1. Карта слабых мест: какие пункты ISO чаще всего нарушаются
2. Подразделения с наибольшим количеством несоответствий
3. Процент просроченных мероприятий по подразделениям
4. Тренд: улучшается или ухудшается ситуация
5. Системные проблемы (повторяющиеся несоответствия в разных подразделениях)
6. Рекомендации по усилению СМК

5. Аудит поставщиков сырья

Коммерческая информация о контрагентах. Анализ истории поставок, качества, своевременности.

Пример промпта
Данные по поставщикам сырья за 2 года (файл Поставщики.xlsx):
Колонки: поставщик, материал, количество поставок, количество отклонений по качеству, процент брака, среднее отклонение по химсоставу, просрочки поставки, цена за тонну, сертификация ISO.

Оцени каждого поставщика:
1. Общий скоринг качества (1-10)
2. Скоринг надёжности (1-10)
3. Скоринг соотношения цена/качество (1-10)
4. Риск-статус: зелёный/жёлтый/красный
5. Рекомендация: продолжать, усилить контроль, заменить
6. Для "красных" — конкретные риски и обоснование

6. Разработка ТУ на новую продукцию

Рецептура, режимы, ноу-хау — всё это конфиденциально. On-prem LLM помогает структурировать ТУ.

Пример промпта
Разрабатываем ТУ на новый сплав "СуперАлюм-3000" для авиапрома.

Экспериментальные данные (5 партий):
- Химсостав: Al 94.2%, Mg 2.8%, Si 1.5%, Cu 0.8%, Fe 0.4%, остальное
- Механические свойства: σв = 380-420 МПа, δ = 12-15%, HB = 110-120
- Температура литья: 720-740°C (оптимум 730°C)
- Скорость кристаллизации: 15-20°C/мин
- Брак по раковинам: 3% (при 730°C), 8% (при 720°C), 5% (при 740°C)

Подготовь структуру ТУ:
1. Область применения
2. Технические требования (химсостав, мехсвойства, допуски)
3. Методы контроля (испытания, частота, оборудование)
4. Упаковка и маркировка
5. Технологические указания (режимы, но без точных параметров процесса — вставить [УТОЧНИТЬ ПРОИЗВОДСТВО])
6. Критерии приёмки

Сохрани в формате, готовом для согласования с производством.

7. R&D новых сплавов

Сопоставление экспериментальных данных с литературными. Помогает принять решение о масштабировании.

Пример промпта
Эксперимент: сплав Al-Mg-Sc (0.3% Sc) для повышения прочности.

Наши данные (3 партии):
- σв = 420, 435, 428 МПа (среднее 428)
- δ = 10, 11, 9% (среднее 10%)
- Твердость: 135, 142, 138 HB
- Стоимость добавки Sc: +15% к цене сплава
- Проблема: склонность к растрескиванию при сварке (2 из 3 партий)

Сравни с литературными данными (если известны) и дай:
1. Соответствие ли наши результаты ожидаемым для Al-Mg-Sc
2. Окупается ли прирост прочности (+15% vs базовый Al-Mg) дополнительной стоимостью
3. Риски: свариваемость, коррозия, стабильность свойств
4. Рекомендация: продолжить R&D / промышленные испытания / отказаться

8. Подготовка сертификатов качества

Для реальных отгрузок. LLM генерирует сертификат по данным партии.

Пример промпта
Подготовь сертификат качества для партии П-2026-2105, отгрузка клиенту "АвиаМаш ООО".

Данные партии:
- Номер: П-2026-2105
- Дата производства: 2026-07-10
- Линия: №3
- Сплав: АМг6
- Химсостав: Al 93.8%, Mg 6.0%, Si 0.3%, Cu 0.2%, Fe 0.15%, Mn 0.5%
- Мехсвойства: σв = 325 МПа, δ = 14%, HB = 95
- ГОСТ: ГОСТ 22233-2018
- ТУ: ТУ 1-2-123-2025
- Результаты УЗК: дефекты не обнаружены (класс 1)
- Вес партии: 12.5 тонн
- Количество слитков: 25 шт

Сформируй сертификат в формате ГОСТ. Включи все обязательные поля.

9. Cost of Quality и KPI

Отчёты для совета директоров. Сколько стоит качество: превентивные затраты, оценка, внутренние отказы, внешние отказы.

Пример промпта
Данные по качеству за 2026 год (файл CoQ.xlsx):
Колонки: месяц, превентивные затраты (обучение, плановое ТО), затраты на оценку (лаборатория, аудиты), внутренние отказы (брак, передел), внешние отказы (претензии, возвраты), объём производства (тонны).

Подготовь отчёт для совета директоров:
1. Cost of Quality (COQ) по категориям: превентивные, оценка, внутренние отказы, внешние отказы
2. COQ как % от объёма производства (тренд по месяцам)
3. Соотношение "хорошие" затраты (превентивные + оценка) vs "плохие" (отказы)
4. Топ-3 источника потерь по сумме
5. Динамика: улучшение vs ухудшение vs прошлый год
6. KPI качества: процент брака, PPM (parts per million), стоимость претензий на тонну
7. Рекомендации: куда направить инвестиции в качество

10. Скоринг качества поставщиков (текстовый)

LLM-рассуждение над текстовым досье поставщика (не числовая ML-модель).

Пример промпта
Оцени поставщика "МеталлургХим" на основе текстового досье:

- Работаем с 2019 года
- Поставляет флюс для десульфурации
- За 2 года: 2 претензии по качеству (нестабильный химсостав, отклонения до 15%)
- Просрочка поставок: 3 из 24 (средняя задержка 5 дней)
- Предоставил сертификат ISO 9001 (2023), но аудит не проводился
- Цена ниже рынка на 12%
- Финансовое состояние: неизвестно (не публичная компания)
- Конкуренты: ООО "ФлюсПром" (цена +8%, но 0 претензий за 3 года)

Оцени по шкале 1-10:
1. Качество продукции
2. Надёжность поставок
3. Финансовая стабильность
4. Потенциал улучшения

Дай рекомендацию: продолжать, усилить контроль, искать замену.

11. Интерпретация лабораторных испытаний

Рецептура и производственная тайна. LLM помогает интерпретировать результаты.

Пример промпта
Результаты лабораторных испытаний партии П-2026-2107:

Химический состав:
- Al: 94.5% (ТУ: 94.0-95.0%) ✓
- Mg: 2.8% (ТУ: 2.5-3.0%) ✓
- Si: 1.2% (ТУ: 0.8-1.5%) ✓
- Cu: 0.9% (ТУ: 0.5-1.0%) ⚠️ — близко к верхней границе
- Fe: 0.35% (ТУ: ≤0.4%) ✓
- Ti: 0.08% (ТУ: 0.05-0.12%) ✓
- Прочие: 0.15% (ТУ: ≤0.3%) ✓

Механические свойства:
- σв: 385 МПа (ТУ: ≥350) ✓
- δ: 13% (ТУ: ≥10%) ✓
- HB: 105 (ТУ: 95-115) ✓

УЗК: класс 2 (ТУ: класс 1-2) ✓

Интерпретируй:
1. Все показатели в норме? Какие требуют внимания?
2. Критичность Cu на верхней границе: влияет ли на свариваемость?
3. Рекомендации по партии: отгрузка, дополнительные испытания, передел?
4. Какие параметры стоит мониторить в следующих партиях

Вопрос 1: Почему root-cause анализ дефектов нельзя делать в облачном ChatGPT?

Потому что ChatGPT не знает метод Ishikawa
Потому что параметры процесса и рецептура — производственная тайна
Потому что ChatGPT работает медленно
