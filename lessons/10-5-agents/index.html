10.5 Агентские сценарии для PR

⏱ 12 мин · медиа-мониторинг, кризис-коммуникации, ESG
Чем агент отличается от языковой модели?
LLM пишет текст по запросу. Агент работает 24/7: сканирует сотни источников, классифицирует, алертит, готовит черновики — без команды «проверь, что пишут». Подробнее — Уровень 4.

Сценарий 1: Агент медиа-мониторинга (n8n + RSS/API)

Задача: отслеживать упоминания компании, конкурентов, отрасли; анализировать тональность; формировать дайджест.

[Cron: каждые 2 часа]
↓
[Параллельный опрос источников]
├─→ RSS СМИ (Коммерсант, Ведомости, РБК)
├─→ Telegram-каналы отрасли
├─→ Google News API
└─→ YouTube (упоминания в видео)
↓
[LLM: фильтрация по релевантности]
├─→ Прямые упоминания компании
├─→ Упоминания конкурентов
├─→ Регуляторные изменения
└─→ Тренды отрасли
↓
[LLM: анализ тональности (позитив / нейтрал / негатив)]
↓
[Формирование дайджеста]
├─ Топ-5 новостей
├─ Тональность
└─ Рекомендации по реакции
↓
[Email PR-директору + пост в корп. Slack]
Результат: PR получает дайджест вместо ручного просмотра 50+ источников. Экономия 3–4 часа в день.

Сценарий 2: Агент кризис-коммуникаций (OpenClaw-агент)

Задача: при обнаружении негативной волны — быстро оценить масштаб, подготовить предложения по ответу, маршрутизировать на согласование.

[Триггер: >10 негативных упоминаний за час ИЛИ упоминание в федеральном СМИ]
↓
[Агент: экстренный сбор]
├─→ Где вспышка (площадки, география)
├─→ Ключевые сообщения / мемы
├─→ Инфлюенсеры, разгоняющие тему
└─→ Связь с реальными событиями
↓
[On-prem LLM: анализ риска]
├─→ Уровень угрозы (низкий / средний / критический)
├─→ Сценарии развития
└─→ Рекомендации по ответу (молчание / опровержение / извинение / действие)
↓
[Черновики ответов]
├─ Для СМИ
├─ Для соцсетей
└─ Внутренняя памятка сотрудникам
↓
[Уведомление PR-директору: «Кризис: уровень Х, предложения прилагаются»]

Сценарий 3: Агент ESG-отчётности (гибридный)

Задача: собирать данные из разных систем, формировать черновик ESG-отчёта по стандартам GRI/SASB.

[Запрос: «Подготовить ESG-отчёт за год»]
↓
[Агент: сбор данных из внутренних систем]
├─→ ERP: энергопотребление, выбросы
├─→ HR: гендерное равенство, обучение
├─→ Закупки: цепочка поставок, локализация
└─→ Производство: безопасность, отходы
↓
[Внешний LLM: анализ по стандарту GRI]
├─→ Какие индикаторы покрыты
├─→ Какие данные отсутствуют
└─→ Сравнение с прошлым годом
↓
[On-prem LLM: формирование текста отчёта]
↓
[Черновик PDF + список доработок]
Важно: внутренние данные (зарплаты, травмы, выбросы) — on-prem. Анализ по стандарту GRI можно делать внешним ИИ на обезличенных данных.
