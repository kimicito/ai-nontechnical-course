8.5 Агентские сценарии для Стройки

⏱ 12 мин · автоматизация строительных процессов агентами
Чем агент отличается от языковой модели?
LLM отвечает на одиночные запросы. Агент работает циклически: получает данные, анализирует, принимает решения, выполняет действия — без участия человека в каждом шаге. Подробнее — Уровень 4.

Сценарий 1: Агент отслеживания дефектов (n8n + Vision API)

Задача: получать фото с объекта, классифицировать дефекты, назначать ответственных, отслеживать устранение.

[Telegram-бот на объекте: фото + описание дефекта]
↓
[n8n: приём сообщения]
↓
[Vision LLM: классификация дефекта (трещина, неровность, брак материала)]
↓
[LLM: определение критичности (критический / серьёзный / косметический)]
↓
[Notion/Битрикс24: создание задачи с фото, описанием, дедлайном]
↓
[Назначение ответственного по справочнику: тип дефекта → отдел]
↓
[Уведомление ответственному в Telegram]
Результат: дефект фиксируется за 2 минуты вместо 2 часов (фото → бумажный акт → внесение в журнал → маршрутизация).

Сценарий 2: Агент проверки смет (OpenClaw-агент)

Задача: загрузить смету, проверить на соответствие нормативам (ФЕР/ГЭСН), найти ошибки, подготовить замечания.

[Загрузка файла сметы (Excel, PDF) в систему]
↓
[On-prem агент: извлечение позиций]
├─→ Проверка: соответствие ФЕР/ГЭСН (внутренняя база)
├─→ Проверка: расчёт объёмов
├─→ Проверка: коэффициенты и поправки
└─→ Проверка: расценки по региону
↓
[LLM: формирование протокола проверки]
├─ Найденные ошибки
├─ Рекомендации по корректировке
└─ Ссылки на нормативные документы
↓
[PDF-протокол + уведомление сметчику]

Сценарий 3: Агент контроля сроков (гибридный)

Задача: мониторить график работ, анализировать риски просрочки, предупреждать ответственных.

[Интеграция с Project / MS Project / Primavera]
↓
[Агент: ежедневное чтение статусов]
├─→ Сравнение план/факт
├─→ Анализ критического пути
└─→ Выявление задач с риском просрочки (>3 дней отклонение)
↓
[LLM: формулировка предупреждения]
├─ Какая работа рискует
├─ Почему (причины по данным)
└─ Что делать (варианты)
↓
[Email руководителю проекта + обновление дашборда]
Важно: данные графика работ — внутренняя информация. Агент работает полностью on-prem или через DMZ с фильтрацией.
