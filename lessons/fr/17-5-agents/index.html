17.5 Scénarios agents pour les RH

⏱ 20 min · agents pour l'intégration, le recrutement, l'évaluation 360°
Agent vs LLM
Un LLM répond à une question isolée. Un agent effectue une séquence d'actions : surveille, analyse, alerte, crée des tâches. Pour les scénarios agents RH, voir Niveau 4 — Qu'est-ce qu'un agent.

Scénario 1 : Agent d'intégration (onboarding) (n8n/no-code)

Problème : les nouveaux salariés reçoivent des informations éparses et manquent des étapes importantes d'intégration.

Architecture de l'agent

Flux de données
1. Déclencheur : nouveau salarié ajouté dans le système RH
2. Vérification : le salarié a-t-il reçu tous les matériels d'intégration ?
3. Action (si non) : envoi automatique des matériels manquants
4. Rappel (jour 3) : demande de feedback sur la première semaine
5. Rappel (jour 7) : vérification de l'accès aux systèmes
6. Rappel (jour 30) : entretien de suivi programmé
7. Notification : alerte au manager si des étapes sont manquées

Données sensibles : les données du salarié (nom, poste, département) — on-prem. Les matériels d'intégration généraux — peuvent être dans le cloud.

Scénario 2 : Agent de recrutement (full AI agent)

Problème : le flux de CV est élevé, la sélection manuelle prend trop de temps.

Architecture de l'agent

Flux de données
1. Déclencheur : nouveau CV dans le système
2. Extraction : texte du CV (OCR si nécessaire)
3. Analyse LLM : comparaison avec les exigences du poste
4. Évaluation : score de conformité (0–100%)
5. Filtrage : CV au-dessus du seuil → entretien
6. Action : création d'une tâche pour le recruteur
7. Notification : alerte avec le résumé du candidat

Données sensibles : tous les CV contiennent des données personnelles — uniquement on-prem.

Scénario 3 : Agent d'évaluation 360° (hybride)

Problème : la collecte et l'analyse des évaluations 360° prennent beaucoup de temps et sont sujettes à des biais.

Architecture de l'agent

Flux de données
1. Déclencheur : début de la période d'évaluation
2. Action (cloud) : envoi des questionnaires types aux répondants
3. Collecte (on-prem) : stockage des réponses dans le système interne
4. Analyse LLM (on-prem) :
   - Agrégation des scores par compétence
   - Analyse des commentaires ouverts
   - Identification des tendances
5. Action : génération d'un rapport pour le manager
6. Notification : rapport prêt pour l'entretien d'évaluation

Sécurité : les questionnaires types — peuvent être dans le cloud. Les réponses et l'analyse — uniquement on-prem (données personnelles).

Vérification des connaissances

Question 1 : Pourquoi l'agent de recrutement — uniquement on-prem ?

Parce que les CV contiennent des données personnelles
On peut anonymiser les CV pour le cloud
Parce que le recrutement nécessite une précision maximale

Question 2 : L'agent d'intégration — quel contour pour les matériels ?

Cloud — matériels généraux sur l'entreprise
On-prem — tous les matériels contiennent des données internes
Hybride — matériels à l'extérieur, données salarié à l'intérieur
