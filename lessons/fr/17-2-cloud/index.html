Niveau 17 — Ressources humaines

17.2 Tâches externes RH : IA externe

⏱ 15 min · méthodologie universelle sans données personnelles
Principe principal de la leçon
Dans le contour externe, on peut transférer en toute sécurité uniquement ce qui n'est pas personnalisé : méthodologie, modèles, analytique publique du marché du travail.

Tâche 1 : Rédaction d'offres d'emploi et descriptions de postes

Pourquoi l'IA externe : contenu public par définition. L'offre est publiée sur des sites d'emploi, dans les réseaux sociaux, sur le site de l'entreprise.

Ce que fait l'IA

  • Formule une description attractive sur la base des exigences
  • Adapte au ton de l'entreprise
  • Ajoute des accents ESG/DEI si nécessaire
Prompt :
Rédige une description d'offre d'emploi pour le poste [titre] dans [secteur]. Exigences clés : [liste]. Niveau : [junior/middle/senior]. Format : attractif, avec accent sur le développement et la culture d'entreprise. Ajoute les sections "Ce que vous ferez", "Ce que nous attendons", "Ce que nous proposons".
Résultat : économie de 20–30 minutes par offre. Les RH peuvent publier 2–3 fois plus d'offres.

Tâche 2 : Préparation des questions d'entretien (méthodologie)

Pourquoi l'IA externe : méthodologie générale pour un type de poste — sans lien à un candidat réel. Contenu réutilisable.

Prompt :
Développe la structure d'un entretien comportemental pour le poste [titre] dans [secteur]. Niveau : [junior/middle/senior]. Inclus : questions d'ouverture, comportementales (STAR), cas pratiques, questions sur la culture, questions de vérification des compétences [liste des compétences].

Tâche 3 : Développement de programmes d'évaluation type (performance review, 360°)

Pourquoi l'IA externe : méthodologie d'évaluation — structure universelle. Les évaluations réelles de salariés spécifiques — on-prem.

Prompt :
Développe la structure d'un programme d'évaluation 360° pour [niveau : manager/spécialiste]. Inclus : compétences à évaluer, échelle, catégories de répondants (manager, collègues, subordonnés), format des retours, délais de réalisation.

Tâche 4 : Développement de programmes de formation et matériels

Pourquoi l'IA externe : matériels de formation de contenu général — partie publique. Peuvent être publiés sur le portail corporate.

Prompt :
Développe un programme de formation sur le thème [titre] pour les salariés de niveau [junior/middle/senior]. Format : structure modulaire, 4–6 séances de 2 heures. Pour chaque module : objectifs, contenu, méthodes, évaluation. Inclus des exercices pratiques.

Tâche 5 : Préparation de tests et matériels d'évaluation

Pourquoi l'IA externe : tests de type général — non liés à un salarié spécifique. Peuvent être types pour tous les candidats à un poste.

Prompt :
Rédige un test de connaissances sur [sujet/compétence] pour les candidats au poste [titre]. 10 questions : 5 théoriques, 3 cas pratiques, 2 de logique. Indique les bonnes réponses et les critères d'évaluation.

Tâche 6 : Matériels d'intégration (onboarding) de caractère général

Pourquoi l'IA externe : matériels sur l'entreprise, la culture, les processus — partie publique. Ne contiennent pas de données personnelles.

Prompt :
Rédige un guide d'intégration pour un nouveau salarié. Sections : premier jour, première semaine, premier mois. Inclus : rencontre avec l'équipe, processus clés, outils, culture d'entreprise, points de contact RH. Format : amical, structuré.

Tâche 7 : Benchmarking des salaires du marché

Pourquoi l'IA externe : analyse d'études de salaires publiques (Indeed, Glassdoor, études sectorielles) — sources ouvertes. Ne contient pas les salaires réels des salariés.

Prompt :
Sur la base de données ouvertes (Indeed, Glassdoor, études sectorielles) pour [période], analyse les salaires du marché pour le poste [titre] dans [région/secteur]. Indique : médiane, P25, P75, P90. Compare avec la période précédente. Identifie les tendances.

Tâche 8 : Matrices de compétences pour des postes types

Pourquoi l'IA externe : compétences types — méthodologie universelle. Le lien à un salarié spécifique — on-prem.

Prompt :
Développe une matrice de compétences pour le poste [titre]. Inclus : compétences techniques, soft skills, qualités de leadership (pour les managers). Pour chacune : description, niveaux de développement (1–5), indicateurs de comportement. Format : tableau.

Tâche 9 : Benchmarking des pratiques RH du secteur

Pourquoi l'IA externe : analyse de rapports publics, cas, études — données ouvertes.

Prompt :
Effectue un benchmarking des pratiques RH dans [secteur] selon les directions : recrutement, formation, motivation, engagement. Utilise les rapports publics Deloitte, McKinsey, études sectorielles. Identifie les meilleures pratiques et les tendances.

Tâche 10 : Stratégie RH de forme générale (workforce planning)

Pourquoi l'IA externe : méthodologie de planification des effectifs — universelle. Les chiffres réels de l'effectif — on-prem.

Prompt :
Développe une stratégie RH pour une entreprise dans [secteur] pour [période]. Inclus : analyse de l'état actuel (problèmes types), objectifs, initiatives clés (recrutement, développement, rétention), métriques de succès, risques. Format : plan structuré.

Vérification des connaissances

Question 1 : Pourquoi les matériels d'intégration peuvent être préparés dans le cloud ?

Ils décrivent l'entreprise et les processus, pas un salarié spécifique
Parce qu'ils sont au format PDF
Parce que l'intégration est une tâche rapide

Question 2 : Pourquoi la matrice de compétences peut être préparée dans le cloud ?

C'est une méthodologie universelle, non liée à un salarié spécifique
Parce que la matrice indique les niveaux 1–5
Parce que les compétences sont un concept public

Question 3 : Quelle tâche de la liste NE convient PAS au cloud ?

Développement de la structure de l'évaluation 360°
Analyse des résultats de l'évaluation 360° de salariés réels
Benchmarking des pratiques RH du secteur
