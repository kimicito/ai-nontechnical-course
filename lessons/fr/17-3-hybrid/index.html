Niveau 17 — Ressources humaines

17.3 Tâches hybrides RH

⏱ 15 min · type/anonymisé à l'extérieur, personnel à l'intérieur
Principe de l'hybride pour les RH
La structure générale, la méthodologie, les formulations types — à l'extérieur. Les noms spécifiques, salaires, évaluations, faits sur le salarié — à l'intérieur.

Tâche 1 : Vérification des références — structure des questions à l'extérieur, évaluation avec nom à l'intérieur

Hybride : la structure type de la vérification des références et les critères d'évaluation — universels. L'évaluation finale du candidat spécifique sur la base des réponses réelles des recommandeurs — on-prem.

Étape 1 — IA externe (structure)

Prompt :
Développe la structure de la vérification des références pour le poste [titre] de niveau [junior/middle/senior]. Inclus : cercle des recommandeurs (manager, collègues, clients), questions par compétences [liste], échelle d'évaluation, format d'enregistrement des résultats. Format : modèle prêt pour les RH.

Étape 2 — On-prem (évaluation du candidat spécifique)

Prompt :
Résultats de la vérification des références du candidat [nom] selon la structure ci-dessus : [enregistrements]. Effectue l'évaluation pour chaque compétence. Forme une conclusion finale avec recommandation (embauche/refus/entretien complémentaire).
Économie : 30–40 minutes pour la préparation de la structure (faite 1 fois par type de poste). Évaluation d'un candidat spécifique — 10 minutes au lieu de 30.

Tâche 2 : Plan de développement individuel — modèle à l'extérieur, lien au salarié à l'intérieur

Hybride : le modèle général du PDI, les parcours de développement types — universels. Le lien aux résultats d'évaluation du salarié spécifique — on-prem.

Étape 1 — IA externe (modèle)

Prompt :
Développe un modèle de PDI pour [niveau : spécialiste/manager] pour [période]. Inclus : structure (objectifs, compétences, activités, délais, critères), parcours de développement types (techniques, managériales, soft skills). Format : modèle à compléter.

Étape 2 — On-prem (personnalisation)

Prompt :
Complète le modèle de PDI pour le salarié [nom]. Données : poste [titre], résultats d'évaluation [données], objectifs du salarié [objectifs]. Rédige un plan concret avec des critères mesurables.

Tâche 3 : Enquêtes d'engagement — questions à l'extérieur, analyse des réponses à l'intérieur

Hybride : les formulations des questions — types, peuvent être publiques. L'analyse des réponses réelles des salariés — données personnelles + sensibilité de gestion.

Étape 1 — IA externe (questions)

Prompt :
Développe une enquête d'engagement des salariés. 15 questions : 10 fermées (échelle 1–5), 5 ouvertes. Thèmes : satisfaction au travail, relations avec le manager, développement, équilibre travail/vie personnelle, recommandation de l'entreprise comme employeur. Format : questionnaire prêt.

Étape 2 — On-prem (analyse)

Prompt :
Analyse les résultats de l'enquête d'engagement pour [période]. Données : [résultats agrégés, sans noms par réponse individuelle]. Identifie : indice général, zones problématiques par départements, tendances vs période précédente, recommandations.
Important : l'analyse des réponses ouvertes, où les salariés peuvent indiquer des noms ou décrire des situations concrètes — uniquement on-prem.

Tâche 4 : Retour d'information sur l'évaluation — principes à l'extérieur, faits sur le salarié à l'intérieur

Hybride : les principes généraux de formulation d'un retour constructif — universels. Le contenu avec des faits réels sur le salarié spécifique — on-prem.

Étape 1 — IA externe (méthodologie)

Prompt :
Développe un guide pour le manager sur la conduite d'un entretien d'évaluation. Inclus : structure de la conversation, exemples de retour constructif (modèle SBI : Situation-Behavior-Impact), exemples de formulations pour différents scénarios (dépassement des attentes, conformité, zone de développement). Format : aide-mémoire du manager.

Étape 2 — On-prem (préparation d'un entretien spécifique)

Prompt :
Prépare les talking points pour l'entretien d'évaluation avec le salarié [nom]. Données : poste [titre], résultats d'évaluation [données], objectifs de la période [objectifs]. Utilise le modèle SBI. Formule le retour pour chaque compétence.

Tâche 5 : Offres et contrats de travail — modèle à l'extérieur, conditions réelles à l'intérieur

Hybride : le modèle type d'offre/contrat — universel. Les conditions réelles (salaire, nom, date) — on-prem.

Étape 1 — IA externe (modèle)

Prompt :
Rédige un modèle type d'offre pour le poste [titre] dans [secteur]. Inclus : structure, sections obligatoires (poste, salaire, période d'essai, avantages sociaux), formulations sur la confidentialité, conditions de temps partiel (si applicable). Format : modèle avec marqueurs [NOM], [SALAIRE].

Étape 2 — On-prem (remplissage)

Prompt :
Remplis le modèle d'offre pour le candidat [nom]. Données : poste [titre], salaire [montant], date d'entrée [date], période d'essai [durée]. Vérifie la conformité au droit du travail.

Vérification des connaissances

Question 1 : Pourquoi l'enquête d'engagement — hybride, et pas entièrement on-prem ?

Questions types (peuvent être dans le cloud), analyse des réponses — on-prem
Parce que l'enquête se déroule en ligne
Parce que l'enquête est anonyme

Question 2 : Quelle étape de la vérification des références — on-prem ?

Développement de la structure des questions
Évaluation du candidat spécifique sur la base des réponses réelles
Choix de l'échelle d'évaluation (1–5)

Question 3 : Pourquoi le PDI — hybride ?

Parce que les objectifs sont personnels
Le modèle est universel (cloud), le lien au salarié spécifique — on-prem
Parce que les délais sont différents pour tout le monde
