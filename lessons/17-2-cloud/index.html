Уровень 17 — Персонал

17.2 Внешние задачи HR: облако

⏱ 15 мин · универсальная методология без персональных данных
Главный принцип урока
Во внешний контур безопасно выносить только то, что не персонализировано: методология, шаблоны, публичная аналитика рынка труда.

Задача 1: Составление вакансий и описаний должностей

Почему облако: публичный контент по определению. Вакансия публикуется на job-boards, в соцсетях, на сайте компании.

Что делает AI

  • Формулирует привлекательное описание на основе требований
  • Адаптирует под tone of voice компании
  • Добавляет ESG/DEI-акценты при необходимости
Промпт:
Составь описание вакансии на позицию [название] для [отрасль]. Ключевые требования: [список]. Уровень: [junior/middle/senior]. Формат: привлекательный, с акцентом на развитие и культуру компании. Добави раздел "Что ты будешь делать", "Что мы ждём", "Что мы предлагаем".
Результат: экономия 20–30 минут на каждую вакансию. HR может публиковать в 2–3 раза больше вакансий.

Задача 2: Подготовка вопросов для интервью (методология)

Почему облако: общая методология под тип позиции — без привязки к реальному кандидату. Повторно используемый контент.

Промпт:
Разработай структуру поведенческого интервью на позицию [название] в [отрасль]. Уровень: [junior/middle/senior]. Включи: открывающие вопросы, поведенческие (STAR), кейс-вопросы, вопросы на культуру, вопросы для проверки компетенций [список компетенций].

Задача 3: Разработка типовых программ оценки (performance review, 360)

Почему облако: методология оценки — универсальная структура. Реальные оценки конкретных сотрудников — on-prem.

Промпт:
Разработай структуру программы 360-оценки для [уровень: руководитель/специалист]. Включи: компетенции для оценки, шкалу, категории респондентов (руководитель, коллеги, подчинённые), формат обратной связи, сроки проведения.

Задача 4: Разработка учебных программ и материалов

Почему облако: обучающие материалы общего содержания — публичная часть. Могут публиковаться на корпоративном портале.

Промпт:
Разработай учебную программу по теме [название] для сотрудников уровня [junior/middle/senior]. Формат: модульная структура, 4–6 занятий по 2 часа. Для каждого модуля: цели, содержание, методы, оценка. Включи практические упражнения.

Задача 5: Подготовка тестов и оценочных материалов

Почему облако: тесты общего типа — не привязаны к конкретному сотруднику. Могут быть типовыми для всех кандидатов на позицию.

Промпт:
Составь тест на знание [предмет/навык] для кандидатов на позицию [название]. 10 вопросов: 5 теоретических, 3 кейсовых, 2 на логику. Укажи правильные ответы и критерии оценки.

Задача 6: Онбординг-материалы общего характера

Почему облако: материалы о компании, культуре, процессах — публичная часть. Не содержат персональных данных.

Промпт:
Составь онбординг-гайд для нового сотрудника. Разделы: первый день, первая неделя, первый месяц. Включи: знакомство с командой, ключевые процессы, инструменты, культура компании, точки контакта HR. Формат: дружелюбный, структурированный.

Задача 7: Бенчмаркинг рыночных зарплат

Почему облако: анализ публичных обзоров зарплат (HeadHunter, Glassdoor, отраслевые исследования) — открытые источники. Не содержит реальных окладов сотрудников.

Промпт:
На основе открытых данных (HeadHunter, Glassdoor, отраслевые обзоры) за [период], проанализируй рыночные зарплаты на позицию [название] в [регион/отрасль]. Укажи: медиану, P25, P75, P90. Сравни с прошлым периодом. Выдели тренды.

Задача 8: Матрицы компетенций под типовые должности

Почему облако: типовые компетенции — универсальная методология. Привязка к конкретному сотруднику — on-prem.

Промпт:
Разработай матрицу компетенций для позиции [название]. Включи: технические компетенции, soft skills, лидерские качества (для руководителей). Для каждой: описание, уровни развития (1–5), индикаторы поведения. Формат: таблица.

Задача 9: Бенчмаркинг HR-практик отрасли

Почему облако: анализ публичных отчётов, кейсов, исследований — открытые данные.

Промпт:
Проведи бенчмаркинг HR-практик в [отрасль] по направлениям: подбор, обучение, мотивация, вовлечённость. Используй публичные отчёты Deloitte, McKinsey, HH.ru. Выдели лучшие практики и тренды.

Задача 10: HR-стратегия в общем виде (workforce planning)

Почему облако: методология планирования кадров — универсальная. Реальные цифры штата — on-prem.

Промпт:
Разработай HR-стратегию для компании в [отрасль] на [период]. Включи: анализ текущего состояния (типовые проблемы), цели, ключевые инициативы (подбор, развитие, удержание), метрики успеха, риски. Формат: структурированный план.

Проверка знаний

Вопрос 1: Почему онбординг-материалы можно готовить в облаке?

Они описывают компанию и процессы, а не конкретного сотрудника
Потому что они в формате PDF
Потому что онбординг — быстрая задача

Вопрос 2: Почему матрицу компетенций можно готовить в облаке?

Это универсальная методология, не привязана к конкретному сотруднику
Потому что в матрице указаны уровни 1–5
Потому что компетенции — общедоступное понятие

Вопрос 3: Какая задача из списка НЕ подходит для облака?

Разработка структуры 360-оценки
Анализ результатов 360-оценки реальных сотрудников
Бенчмаркинг HR-практик отрасли
