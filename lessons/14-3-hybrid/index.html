Уровень 14 — Продажи и маркетинг

14.3 Гибридные задачи

⏱ 20 мин · для продавцов и маркетологов
Гибридный подход
Общую структуру и методологию готовим во внешнем ИИ.
Реальные данные (цены, клиенты, условия) добавляем в on-prem.

Принцип гибридного подхода для продаж и маркетинга

Ключевое правило: чем чувствительнее данные (конкретные цены, условия сделки, персональные данные клиента) — тем жёстче on-prem. Общая методология (структура документа, типовые сценарии) спокойно уходит во внешний контур.

Правила обезличивания для облака:
Что обязательно убратьЧто можно оставить
Конкретные цены и скидкиМетодология ценообразования
Названия клиентов и контактовТип клиента (крупный, средний, ключевой)
Индивидуальные условия оплатыТиповые формы оплаты
Объёмы по конкретным сделкамМетодология расчёта объёма
Финансовые показатели компанииОтраслевые бенчмарки
Конкретные даты сделокСезонность, типичные сроки

1. Коммерческие предложения (гибрид)

Облако: общий шаблон и структура КП.
On-prem: финальные цены, условия, персонализация под клиента.

Шаг 1: Облако — шаблон КП
Подготовь структуру коммерческого предложения для B2B-продаж:

Структура КП:
1. Титул (логотип, контакты, дата)
2. О компании (2-3 предложения)
3. Понимание потребности клиента
4. Продукт/решение (описание, спецификация)
5. Условия поставки [УТОЧНИТЬ: конкретные условия]
6. Цена [УТОЧНИТЬ: расчёт под клиента]
7. Преимущества и гарантии
8. Порядок работы (этапы, сроки)
9. Приложения (спецификация, сертификаты)

Для каждого раздела: что писать, типичные ошибки, примеры формулировок.

[УТОЧНИТЬ: отрасль клиента, тип продукта, регион]
Шаг 2: On-prem — заполнение под клиента
Заполни КП для клиента [УКАЖИТЕ НАЗВАНИЕ]:

Данные клиента:
- Отрасль: [УКАЖИТЕ]
- Объём потребления: [УКАЖИТЕ]
- История: клиент с [УКАЖИТЕ ГОДА], оборот [УКАЖИТЕ]
- Текущие условия: [УКАЖИТЕ]
- Потребность: [УКАЖИТЕ]

Наши данные:
- Цена: [УКАЖИТЕ] (базовая), скидка [УКАЖИТЕ]% при объёме [УКАЖИТЕ]
- Условия оплаты: [УКАЖИТЕ]
- Срок поставки: [УКАЖИТЕ]
- Гарантии: [УКАЖИТЕ]

Составь финальное КП с персонализацией под клиента.

Экономия: шаблон в облаке (1.5 часа) + заполненное КП в on-prem (1 час).

2. Подготовка к переговорам (гибрид)

Облако: общая методология переговоров, типовые сценарии.
On-prem: реальные цифры, конкретный клиент, BATNA.

Шаг 1: Облако — методология
Подготовь методологию переговоров для B2B-продаж:

Этапы переговоров:
1. Разведка (какие вопросы задать, чтобы понять потребности)
2. Презентация решения (как связать продукт с потребностью)
3. Работа с возражениями (типичные возражения и ответы)
4. Обсуждение условий (цена, скидки, оплата, логистика)
5. Закрытие (какие сигналы говорят о готовности)

Методология:
- BATNA: как определить и использовать
- ZOPA: как расширить зону соглашения
- Рычаги: что можем предложить (кроме цены)
- Типичные ловушки (преждевременное скидование, согласие без обмена)

[УТОЧНИТЬ: отрасль, тип продукта, средний чек]
Шаг 2: On-prem — конкретная сделка
Примени методологию к переговорам с [УКАЖИТЕ КЛИЕНТА]:

Контекст:
- Текущий контракт: [УКАЖИТЕ], истекает [УКАЖИТЕ ДАТУ]
- Объём: [УКАЖИТЕ] ед./год
- Жалобы: [УКАЖИТЕ]
- Конкурент: предлагает [УКАЖИТЕ УСЛОВИЯ]
- Наша цель: [УКАЖИТЕ]

Составь:
1. BATNA (если не договоримся)
2. ZOPA (приемлемый диапазон)
3. Рычаги (что можем предложить)
4. Сценарии: оптимистичный, реалистичный, пессимистичный
5. Ответы на ожидаемые возражения

Экономия: методология в облаке (1.5 часа) + стратегия в on-prem (1.5 часа).

3. Email-рассылки лидам (гибрид)

Облако: общий шаблон письма, A/B-варианты.
On-prem: персонализация с реальными данными клиента.

Шаг 1: Облако — шаблон письма
Напиши шаблон письма для холодной рассылки B2B:

Параметры:
- Продукт: [УКАЖИТЕ ТИП]
- ЦА: [УКАЖИТЕ ДОЛЖНОСТЬ], [УКАЖИТЕ РАЗМЕР КОМПАНИИ]
- Цель: получить встречу/демо

Варианты:
1. Короткое (3 предложения, мобильный формат)
2. Среднее (параграф + буллеты + CTA)
3. Со ссылкой на кейс (социальное доказательство)

Для каждого:
- Тема письма (3 варианта)
- Хук (что привлечёт внимание)
- CTA (конкретное действие)
- Подпись

[УТОЧНИТЬ: отрасль, ключевая боль ЦА]
Шаг 2: On-prem — персонализация
Персонализируй письмо для [УКАЖИТЕ ФИО], [УКАЖИТЕ ДОЛЖНОСТЬ], [УКАЖИТЕ КОМПАНИЮ]:

Данные клиента:
- Отрасль: [УКАЖИТЕ]
- Недавнее событие: [УКАЖИТЕ: расширение, новый продукт, публикация]
- Возможная потребность: [УКАЖИТЕ]
- История взаимодействия: [УКАЖИТЕ]

Шаблон: [ВСТАВЬТЕ ШАБЛОН ИЗ ОБЛАКА]

Адаптируй:
1. Упомяни конкретное событие/контекст
2. Свяжи продукт с их потребностью
3. Добавь релевантный кейс
4. Предложи конкретное время встречи

Экономия: шаблон в облаке (1 час) + персонализация в on-prem (30 мин).

4. IR-материалы (гибрид)

Облако: общая структура презентации, типовые слайды.
On-prem: финальные цифры до официальной публикации.

Шаг 1: Облако — структура презентации
Подготовь структуру IR-презентации для B2B-компании:

Слайды:
1. Титул (компания, период)
2. Executive summary (3 ключевых сообщения)
3. Рыночная позиция (TAM/SAM/SOM, тренды)
4. Продуктовая линейка (категории, дифференциация)
5. Финансовые результаты [УТОЧНИТЬ: конкретные цифры]
6. Клиентская база (диверсификация, retention)
7. Стратегия роста (органический, M&A, новые рынки)
8. ESG и устойчивое развитие
9. Риски и mitigations
10. Финансовый прогноз [УТОЧНИТЬ: конкретные цифры]
11. Инвестиционная тезис (почему стоит инвестировать)
12. Контакты IR-отдела

Для каждого слайда: ключевое сообщение, типичные вопросы аналитиков, как отвечать.
Шаг 2: On-prem — финальные данные
Заполни IR-презентацию финальными данными:

Предварительные данные (не публичные):
- Выручка H1: [УКАЖИТЕ]
- EBITDA: [УКАЖИТЕ]
- Маржа: [УКАЖИТЕ]%
- Новые клиенты: [УКАЖИТЕ]
- Потерянные сделки: [УКАЖИТЕ]
- Прогноз годовой: [УКАЖИТЕ]

Составь:
1. Текст для каждого слайда (с цифрами)
2. Спикер-ноуты (что говорить вслух)
3. Ответы на ожидаемые вопросы
4. Appendix (детальная разбивка для аналитиков)

Экономия: структура в облаке (2 часа) + заполнение в on-prem (2 часа).

Проверка знаний

Вопрос 1: Какой принцип гибрида для коммерческих предложений?

Общий шаблон в облаке, финальные цены и условия в on-prem
Всё КП можно делать в облаке, если обезличить название клиента
Всё КП — только в on-prem, так как содержит коммерческую тайну

Вопрос 2: Email-рассылка лидам — какой шаг в on-prem?

Персонализация: имя, компания, контекст, конкретное предложение
Написание общего шаблона письма
A/B-тестирование вариантов письма

Вопрос 3: Что из списка обязательно убрать перед облаком?

Конкретные цены, названия клиентов, объёмы сделок
Тип продукта (например, "программное обеспечение")
Отрасль клиента (например, "производство")
