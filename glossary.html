📖 Терминология

Простыми словами о сложных понятиях из мира ИИ. Если в курсе встретили незнакомое слово — загляните сюда.

🔤 Основные термины

API

«Розетка для данных» — способ подключения программ друг к другу

Если сайт А хочет получить данные из программы Б, он использует API. Например, когда калькулятор курса получает актуальный курс доллара с сайта ЦБ.

Open-source

Программа с открытым исходным кодом — можно скачать и использовать бесплатно

Разработчики выложили код в открытый доступ. Любой может скачать, изучить, изменить. Пример: n8n, LibreOffice. Но: за поддержку и обучение может взиматься плата (как Red Hat).

Self-hosted

«На своём сервере» — программа работает внутри компании, а не в облаке

Данные не покидают офис. Нужен сервер и ИТ-специалист для настройки. Лицензия бесплатна, но оборудование и обслуживание — нет.

VPN

Виртуальная частная сеть — безопасный туннель для удалённой работы

Позволяет сотруднику из дома безопасно подключиться к корпоративной сети. Для on-premise ИИ в офисе нужна локальная сеть, для удалёнки — VPN.

Firewall

Сетевой экран — защита от нежелательных подключений

Как охранник у входа: пропускает только тех, кто имеет право. Настраивает ИТ-отдел.

Cron / Расписание

Запуск задачи по расписанию: каждый день, каждый час и т.д.

Например: «Каждый понедельник в 9:00 проверять цены поставщиков». В n8n это называется Schedule, в программировании — cron.

JSON

Формат данных для программ — «таблица в текстовом виде»

AI часто выдаёт ответы в формате JSON, который потом конвертируется в Excel/таблицу. Для человека выглядит как текст со скобками.

CSV

Табличный формат — открывается в Excel

Простой текстовый формат, где данные разделены запятыми. Любая таблица может быть сохранена как CSV.

LLM (Large Language Model)

Большая языковая модель — «мозг» ИИ, который понимает и генерирует текст

ChatGPT, Claude, Kimi — все это LLM. Обучены на огромном количестве текстов, чтобы понимать вопросы и давать осмысленные ответы.

Токен

Единица измерения текста для ИИ — примерно ¾ слова

ИИ считает не слова, а токены. 1000 токенов ≈ 750 слов. От количества токенов зависит стоимость запроса.

RAG (Retrieval-Augmented Generation)

Поиск + генерация — ИИ сначала находит факты, потом отвечает

Вместо того чтобы полагаться только на память, ИИ ищет информацию в ваших документах и даёт ответ на их основе. Снижает риск выдумок.

Векторная база данных

Хранилище, где ИИ ищет по смыслу, а не по точным словам

Если вы ищете «дорогой поставщик», найдётся и «высокая цена», и «недёшево». Понимает контекст, а не только формулировку.

Промпт

Запрос или инструкция для ИИ — то, что вы пишете в чат

Чем точнее промпт, тем лучше ответ. Хороший промпт содержит: роль, задачу, формат, ограничения, примеры.

Агент ИИ

Программа, которая сама выполняет задачи по заданной инструкции

Не просто отвечает на вопросы, а действует: ищет, сравнивает, отправляет письма, заполняет документы.

Галлюцинация

Когда ИИ уверенно выдаёт выдуманную информацию

Может придумать несуществующий закон, фальшивую цитату или ложный факт. Всегда проверяйте критичные данные.

Файн-тюнинг (Fine-tuning)

Дообучение модели на ваших данных для специфических задач

Берётся готовая модель и «подстраивается» под ваши документы, терминологию, стиль. Дороже обычного использования, но точнее.

Граничные случаи (Edge cases)

Крайние, нестандартные ситуации — когда всё идёт не по плану

Например: бюджет = 0, файл пустой, поставщик не отвечает. Хорошая автоматизация учитывает и такие сценарии.

MCP (Model Context Protocol)

Стандарт подключения ИИ к внешним системам

Позволяет агенту ИИ обращаться к вашим базам данных, CRM, почте. Как универсальный разъём для разных устройств.

On-premise

«На территории» — всё работает внутри компании, не в облаке

Данные остаются в ваших стенах. Безопасно, но дороже в обслуживании. Подходит для конфиденциальной информации.

Workflow (воркфлоу)

Последовательность шагов автоматизации

Например: получить письмо → извлечь данные → записать в таблицу → отправить уведомление. В n8n и Zapier создаётся визуально.

Webhook

«Сигнал» от одной программы к другой — мгновенное уведомление

Когда происходит событие (например, новый заказ), система А мгновенно сообщает системе Б. Без постоянного опроса.

⚙️ Параметры, автоматизация и аудит

Контекстное окно

«Память модели» — сколько текста ИИ помнит за один раз

Если документ длиннее окна, модель забывает начало. Для анализа договора в 50 страниц нужен LLM с большим окном (200K+ токенов) или техника чанкинга.

Температура

«Уровень креативности» ИИ — от строгого до фантазийного

0.1 — предсказуемый, почти детерминированный (для договоров, отчётов). 0.7 — креативный, генерирует варианты (для мозгового штурма, маркетинга). 1.0+ — может начать выдумывать.

Эмбеддинг

«Числовой отпечаток смысла» — как ИИ понимает, что два текста похожи

Каждому куску текста присваивается набор чисел (вектор). ИИ находит релевантные документы по «математической близости», даже если слова совершенно другие.

Чанкинг

«Разрезание на куски» — разбиение длинного документа на части

Договор в 100 страниц режется на абзацы по 500 токенов, чтобы каждый кусок поместился в память модели. Потом ИИ ищет нужные куски и собирает ответ.

Парсинг

«Извлечение структуры из хаоса» — превращение неструктурированного в таблицу

Из PDF-каталога поставщика вытащить цены, артикулы, единицы измерения в Excel. Или из email-переписки — даты, суммы, адреса.

SaaS

Программа по подписке через интернет — не ставишь, платишь и пользуешься

ChatGPT, n8n Cloud, Zapier — всё SaaS. Заходишь через браузер, данные на серверах провайдера. Удобно, но для конфиденциальных документов нужен on-premise аналог.

Скоринг

Оценка по баллам — объективный отбор поставщиков или проверка качества

Поставщик набирает 85/100 — проходит, 60/100 — отсеивается. Критерии: цена, сроки, репутация, сертификаты, опыт. ИИ может считать автоматически по заданной шкале.

Compliance

Соответствие правилам — законы, стандарты, внутренние регламенты

Проверка, что договор не противоречит 44-ФЗ, что смета соответствует ГОСТ, что закупка прошла все согласования. ИИ сверяет документ с чеклистом требований.

SLA

Соглашение об уровне сервиса — скорость, доступность, штрафы

«Ответ на запрос — за 2 часа, доступность 99.9%, штраф за простой — 1% от месячной платы». ИИ может мониторить выполнение SLA и сигнализировать о нарушениях.

Пайплайн

«Конвейер» — цепочка шагов обработки данных

Шаг 1: получить email → Шаг 2: извлечь данные → Шаг 3: записать в CRM → Шаг 4: отправить уведомление. В n8n и Zapier пайплайны собираются визуально, без программирования.

🏷️ Сокращения из курса

GENAi

Внутренний сервис компании на базе OpenWebUI

Работает на ваших серверах (on-premise). Безопасен для конфиденциальных документов. Используйте GENAi для: договоров, отчётов, персональных данных, внутренних протоколов.

NDA

Соглашение о неразглашении — договор конфиденциальности

Подписывается с поставщиками ИИ, чтобы они не использовали ваши данные для обучения моделей.

ROI

Return on Investment — окупаемость инвестиций

Показывает, сколько денег приносит внедрение ИИ по сравнению с затратами. ROI = (Прибыль − Затраты) / Затраты × 100%.

CRM

Система управления отношениями с клиентами

Хранит информацию о клиентах, сделках, контактах. Примеры: amoCRM, Битрикс24, Salesforce.

ERP

Система планирования ресурсов предприятия

Объединяет учёт, производство, закупки, продажи. Примеры: 1С, SAP, Oracle.

🤖 Агент-системы и корпоративные инструменты

OpenClaw

Open-source фреймворк для запуска AI-агентов на собственном сервере

Позволяет создать цифрового сотрудника, который работает в Telegram, Email или корпоративном мессенджере. Данные не покидают компанию — агент работает on-premise. Использует файлы конфигурации (SOUL.md, SKILL.md, HEARTBEAT.md) для задания поведения.

Hermes Agent

Самообучающийся AI-агент от Nous Research

Агент, который создаёт новые навыки (SKILL.md) из своего опыта и запоминает предпочтения пользователя. Улучшается с каждой задачей, как опытный сотрудник.

GoClaw

Enterprise-платформа для команд агентов с изоляцией данных

Позволяет разным отделам иметь своих агентов с разными правами доступа. Например, агент бухгалтерии не видит данные инженеров. Подходит для крупных производственных компаний.

SOUL.md

«Душа» агента — файл, определяющий его характер и роль

Например: «Ты опытный инженер-технолог. Говоришь кратко, используешь данные АСУ ТП. Не делаешь предположений без фактов». Это как должностная инструкция для цифрового сотрудника.

SKILL.md

Файл-навык — пошаговая инструкция для типовой задачи агента

Например: «Как анализировать журнал отказов: 1) Загрузить данные из АСУ ТП, 2) Сгруппировать по оборудованию, 3) Выделить повторяющиеся отказы, 4) Дать рекомендации». Агент выполняет такой навык, как регламент.

HEARTBEAT.md

Расписание периодических проверок агента

Определяет, что агент делает автоматически и по какому расписанию. Например: каждый день в 7:30 проверить план добычи против факта, при отклонении >5% — отправить сводку. Аналог графика дежурств и планёрки.

MEMORY.md

Долговременная память агента — факты, решения, уроки

Агент записывает сюда важные факты, чтобы не забывать между сессиями. Например: «Печь №3 даёт брак при температуре >1200°» или «Скважина №47 нестабильна при давлении <15 атм». Это как база знаний опытного сотрудника.

AGENTS.md / USER.md / TOOLS.md

Системные файлы агента — правила, профиль пользователя и инструменты

AGENTS.md — общие правила для всех агентов (кодекс поведения). USER.md — профиль пользователя: кто он, чем занимается, что можно делать без спроса. TOOLS.md — список доступных инструментов: SAP, 1С, Grafana, Русал Пульс, Email.

Русал Пульс

Корпоративный мессенджер для внутренней коммуникации

Аналог Telegram/Slack, но развёрнут внутри компании. Агенты могут отправлять уведомления, отчёты и алерты в каналы Русал Пульс — данные не покидают периметр.

АСУ ТП

Автоматизированная система управления технологическим процессом

Система, которая собирает данные с датчиков и оборудования в реальном времени: температура, давление, расход, состояние агрегатов. Источник данных для агентов аналитики и предиктивного обслуживания.

On-premise (on-prem)

«На территории» — программы и данные внутри компании, не в облаке

Критично для производственных данных: параметры плавки, планы добычи, журналы отказов. On-prem агенты и модели работают на корпоративных серверах — данные не передаются внешним провайдерам.

Векторный поиск (Vector Search)

Поиск по смыслу, а не по точным словам

AI превращает текст в «точки» в многомерном пространстве. Слова со схожим значением («неустойка», «штраф», «пеня») оказываются рядом. Когда вы задаёте вопрос, AI ищет ближайшие точки — даже если формулировка отличается. Важно: нельзя «включить» векторный поиск командой в промпте — он работает автоматически при загрузке документов, или требует настройки RAG-системы от ИТ-отдела.
