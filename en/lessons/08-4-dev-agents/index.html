8.4 Development and Agents

15 min · for project managers and estimators · when a team is needed

In construction, AI is not just about prompts in ChatGPT. Real efficiency appears where the model integrates with your systems: estimation software, BIM, ERP, drawing databases. This requires a development team or ready-made agent solutions.

In this lesson: three complexity baskets — from tasks handled by a contractor to tasks requiring your own development team or agent system.

1. Where a Development Team is Needed

Some tasks in construction cannot be solved with off-the-shelf AI tools. Integration with internal systems, precise calculations, and legal liability are required.

Task Why Development is Needed What to Build
Estimates by GESN/FER Grand-Estimate and analogs use closed formats; a calculation engine is needed Integration with Grand-Estimate via API or plugin + custom calculation engine for verification
Index Updates per Minstroy Indices change quarterly; manual updates risk errors Minstroy index parser + auto-update of estimate norms
NMCC Calculation Initial maximum contract price is legally sensitive; an error = retendering Calculation module with source traceability, justification export, and audit log
Specification Extraction from Drawings Vision models produce a draft; production requires a verification pipeline Vision model (Qwen/GPT-4V) → structuring → manual check → export to estimate
BOR from BIM BIM is the single source of truth, but the bill of quantities is needed in estimation software BIM connector → BOR extraction → formatting for Grand-Estimate
ERP/Warehouse Reconciliation Warehouse materials ≠ estimate materials; two-way synchronization is needed ERP integration (1C, SAP) for reconciliation of balances and orders
Clash Detection Navisworks and Tekla find clashes, but integrating reports into workflow is a separate task Navisworks API → report extraction → automatic assignment of responsible parties → dashboard
3D Renders Generative tools (Stable Diffusion, DALL-E) need integration with BIM models Pipeline: BIM → view export → generative render → comparison with original geometry
Key Principle: if a task requires integration with two or more internal systems — this is work for a development team, not a "contractor in 2 weeks."

2. Where Agents are Needed

Agent systems are not just LLMs, but autonomous processes that can access systems, collect data, make decisions, and report to humans.

Project Risk Analysis

The agent analyzes the estimate, schedule, weather, and suppliers, and outputs a risk score for each risk with recommendations.

Agent at Work
1. Received estimate and schedule
2. Checked Minstroy indices — up to date
3. Checked weather forecast for foundation period — rain risk
4. Checked suppliers — 2 out of 5 had delays last quarter
5. Result: report with 3 critical risks + recommendations

Estimate Index Updates

Agent process: quarterly, the agent checks Minstroy indices, compares with current estimates, and suggests adjustments.

Agent at Work
1. Loaded current indices from Minstroy website
2. Compared with 15 active estimates
3. Found 3 estimates with outdated indices
4. Calculated delta (estimate is overstated by 8%)
5. Sent to estimator for approval

BOR Reconciliation with Procurement and Balances

The agent goes into ERP and checks: are materials ordered per BOR? Are they in stock? When is delivery?

Agent at Work
1. Received BOR for next month
2. Checked ERP — balances: cement shortage
3. Checked orders: cement delivery in 2 weeks
4. BOR requires cement in 1 week
5. Result: delay risk notification

Contractor Estimate Verification

The agent pulls market prices from open sources (or your database) and cross-checks with the contractor's estimate.

Agent at Work
1. Received contractor estimate for 45 million
2. Checked 120 positions against market price database
3. Found 15 positions inflated by 12-30%
4. Generated report with negotiation recommendations

3. Piloting: Where to Start

Don't try to automate everything at once. Construction is a conservative industry, and an error in an estimate costs more than a one-month delay.

Basket 1: Quick Effect, Low Risk

  • Specification extraction from drawings — Vision model + manual check. After 2-3 iterations, accuracy is 90%+.
  • Project risk analysis — agent based on open data + your estimates. Does not touch systems.
  • 3D renders for presentations — generative AI, does not affect calculations.

Basket 2: Medium Risk, Integration Required

  • Estimate index updates — needs Minstroy parser + access to estimates. Human-in-the-loop is mandatory.
  • BOR reconciliation with procurement — agent goes into ERP, needs access rights and audit log.
  • Clashes from Navisworks — BIM integration, but result is a report, not model changes.

Basket 3: High Risk, Team Required

  • GESN/FER estimates with auto-calculation — error = incorrect tender or schedule slip.
  • NMCC calculation — legal liability, needs traceability and expertise.
  • Contractor estimate verification with automatic claim generation — contract consequences.
Human-in-the-loop is mandatory for baskets 2 and 3. AI proposes, human confirms before execution. This is not a slowdown — it's insurance. In construction, a 5% estimate error can cost tens of millions.

Knowledge Check

Question 1: Which task definitely requires a development team rather than an off-the-shelf AI tool?

Comparing 3 commercial proposals by price
Integration with Grand-Estimate and a calculation engine
Generating a 3D render for a presentation

Question 2: Why is NMCC calculation in basket 3 (high risk)?

An error in NMCC can lead to retendering and legal consequences
NMCC is difficult to calculate in Excel
NMCC requires internet access

Question 3: What is "human-in-the-loop" in the context of agents for construction?

Hiring more people for oversight
AI proposes a solution, human confirms before execution
Training people to work with AI

Question 4: Which task from the list is best suited for a pilot (basket 1)?

Specification extraction from drawings with manual verification
Automatic NMCC calculation
Generating claims against a contractor
