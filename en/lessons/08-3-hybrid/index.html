11.3 Hybrid Quality Tasks: anonymized data

15 min · for quality specialists · cloud + on-prem combination

Гибридные задачи — это сценарии, где часть работы делается в облаке (общая структура, методология, литературный обзор), а финальная версия с реальными данными — в on-prem (закрытом контуре).

Принцип: Обезличенное описание + общая методология → облако. Конкретные параметры, рецептура, номера партий → on-prem.

1. Root-Cause + FMEA (Hybrid)

Cloud: Ishikawa diagram structure, 8D template, FMEA matrix. On-Prem: filling with real parameters of a specific batch.

Step 1: Cloud — Template Preparation
Prepare an 8D investigation template for metallurgical production:

D0: Launch criteria (when to start 8D)
D1: Team roles (manager, quality engineer, technologist, production, laboratory)
D2: Problem description form (5W2H)
D3: Temporary containment action options
D4: Ishikawa template (6M: Man, Machine, Material, Method, Milieu, Measurement)
D5-D7: Formats for corrective and preventive actions
D8: Closure criteria

All fields — fill with placeholder'ами [SPECIFY].
Step 2: On-Prem — Fill with Real Data
Fill the 8D template for a specific case:

- Партия: Batch-2026-1847
- Defect: cavities on ingot surface
- Линия: №3
- Date: 2026-07-15
- Parameters: [actual data from MES]

Conduct root-cause, fill Ishikawa, 5 Why, D5-D7.
Результат: ready template in cloud (saves 2 hours) + filled 8D in on-prem (saves another 4 hours).

2. Technical Specifications (Hybrid)

Cloud: general document structure, typical sections. On-Prem: specific technical parameters, tolerances, recipe.

Шаг 1: Облако — структура ТУ
Prepare a ТУ structure for an aluminum alloy for aviation:

1. Application area (example: aircraft structural parts)
2. Technical requirements:
   - Chemical composition (table: element, min, nominal, max)
   - Mechanical properties (σв, δ, HB, E)
   - Dimensional tolerances
3. Control methods (list of standard methods)
4. Packaging and marking (general requirements)
5. Technological instructions (without specific parameters — [SPECIFY])
6. Acceptance criteria (statistical, 100%)

All numerical values — [SPECIFY PRODUCTION].
Шаг 2: On-prem — заполнение параметрами
Fill ТУ for alloy "SuperAlum-3000":

Chemical composition:
- Al: [94.0-95.0]% — nominal 94.5%
- Mg: [2.5-3.0]% — nominal 2.8%
- Si: [0.8-1.5]% — nominal 1.2%
... [specific data from laboratory]

Mechanical properties:
- σв: ≥350 MPa (our data: 385)
- δ: ≥10% (our data: 13%)
... [specific data from tests]

Technological instructions:
- Casting temperature: 720-740°C (optimum 730°C)
- Cooling rate: 15-20°C/min
... [specific parameters from MES]

3. Alloy R&D (Hybrid)

Cloud: literature review, general metallurgical principles. On-Prem: comparison with real experimental data and decision making.

Шаг 1: Облако — литературный обзор
Conduct a literature review on aluminum alloys with Scandium (Sc):

1. What Sc alloys are described in literature (2020-2026)?
2. What mechanical properties are claimed (σв, δ, fatigue)?
3. What production methods (casting, rolling, extrusion)?
4. What application areas (aviation, automotive, sports)?
5. What problems (weldability, corrosion, Sc cost)?
6. Which companies are implementing (public information)?

Make a comparison table: alloy × properties × problems.
Step 2: On-Prem — Compare with Our Data
Compare our experimental data with literature:

Our alloy: Al-Mg-Sc (0.3% Sc)
Our data:
- σв = 428 MPa (average, 3 batches)
- δ = 10%
- Problem: cracking during welding (2/3 batches)

Literature data: [from cloud review]

Сделай:
1. Do our results match literature?
2. If not — what are the differences and why?
3. Recommendation: continue / change composition / abandon?
4. Next experimental steps (if continuing)

4. Training Materials (Hybrid)

Cloud: basic content, general principles. On-Prem: cases with real plant incidents as illustrations.

Шаг 1: Облако — базовый контент
Prepare a training module "Quality Control in Melting":

1. Main melting defects (types, causes, signs)
   - Gas inclusions
   - Oxide films
   - Chemical composition non-uniformity
   - Cavities (internal and surface)
2. Control methods (visual, UT, chemical analysis)
3. Acceptance criteria (general, without specific numbers)
4. Actions when defect is found (general algorithm)

Without reference to a specific plant — only general principles.
Step 2: On-Prem — Add Real Cases
Add real cases from our plant to the module:

Case 1: "Gas inclusions in batch Batch-2026-1847"
- What happened: [description, anonymized]
- Cause: [real cause from investigation]
- How found: [control method]
- What was done: [corrective actions]
- What we learned: [lesson for other masters]

Case 2: "Chemical composition non-uniformity in batch П-2026-1956"
- [similar structure]

Important: anonymize (remove names, specific customers, exact parameters if sensitive).

Anonymization Rules for Cloud

What must be removed before sending to cloud:
  • Customer names ("AutoProm LLC" → "Client A")
  • Batch numbers ("Batch-2026-1847" → "Batch X")
  • Specific chemical compositions ("Al ~94.5%" → "main element ~95%")
  • Supplier names ("UralRaw" → "Supplier 1")
  • Employee names ("Ivanov P.S." → "shift master 2")
  • Geolocation ("shop #3" → "shop Y")
What Can Stay:
  • Defect type ("cavities", "porosity")
  • General process type ("melting", "rolling")
  • General control methods ("UT", "chemical analysis")
  • Document structure (ТУ sections, 8D form)
