2.2 · Vector databases: how AI "understands" meaning

🎯 No code · ✅ Interactive quiz

🎯 Why does business need this?

Regular search looks for exact words.

Vector search searches by meaning.

Example: An employee asks "How to punish a supplier for being late?"

📚 What we are studying

Regular search vs Vector search

❌ Regular search

  • Searches for exact words
  • "Forfeit" ≠ "fine"
  • Skips synonyms
  • Doesn't understand the context
  • Requires precise wording

✅ Vector search

  • Searches for meaning
  • "Forfeit" = "fine" = "penalty"
  • Finds synonyms
  • Understands the context
  • Works with natural language

How it works (without math)

Analogy: Imagine that each word is a point in space.

  • "King" and "queen" are close points (relatives)
  • "King" and "car" are distant points (no connection)
  • "Forfeit" and "fine" are close points (synonyms)

AI turns your question into a point and looks for the closest document fragments to it.

❓ FAQ: Should I write "use vector search" in the prompt?

No, it won't help. When you write in ChatGPT, Claude, or Kimi — vector search has already happened (or hasn't happened) before you hit Enter. The AI has no "toggle vector search on" lever that responds to text commands.

How it actually works
What you doWhat happens
Write "use vector search"AI ignores it as a meaningless instruction — there's no such switch
Upload a PDF and ask a questionAutomatically the system finds relevant chunks via vector search
Write "search the internet"That's web search — a different mechanism, not document vector search

✅ How a non-IT specialist can actually use vector search

Variant 1: In a cloud chat (ChatGPT, Claude, Kimi + document upload)

Simply upload documents and ask specific questions. The system automatically uses vector search to find relevant fragments. Your prompt should be specific — not about "turn on search", but about "cite the page and quote the text".

Useful prompt for working with a document
Role: You are a supply contract lawyer.
Task: Check this contract for penalty clauses.
Format: Table — clause, risk, recommendation.

Important: indicate the page number and quote the text verbatim.

Vector search works here automatically — the system finds chunks about "penalty", "fines", "forfeits" and passes them to the AI.

Variant 2: In a corporate system (OpenWebUI / GENAI / agent)

When delegating to the IT department — use the term correctly. This is a technical specification, not a chat command:

Correct wording for an IT task
"We need the agent, when answering employee questions,
to first search for relevant documents via vector search,
then form an answer based on them. Do not hallucinate."

IT will understand: RAG (Retrieval-Augmented Generation) is needed — a system where vector search + answer generation work together.

Variant 3: When vector search is NOT suitable

Don't use it (and don't write about it in prompts) when:

  • You need an exact fact — "What is the USD exchange rate on July 15, 2026?" → Better use a central bank API or web search.
  • You need a calculation — "Calculate the NPV of this project" → Better use Excel + formulas.
  • The question is general — "What is vector search?" → The AI already knows from general knowledge, no documents needed.

✅ Checklist for accepting the result

  • Does the search find synonyms?
  • Does search understand context?
  • Is the response time acceptable? < 3 seconds)
  • Are the results relevant?
  • Does the system scale as documents grow?
  • Is it possible to filter by date/document type?
