2.2 · Vector databases: how AI “understands” meaning

🎯 No code · ✅ Interactive quiz

🎯 Why does business need this?

Regular search looks for exact words.

Vector search searches by meaning.

Example: An employee asks “How to punish a supplier for being late?”

📚 What we are studying

Regular search vs Vector search

❌ Regular search

  • Searches for exact words
  • "Forfeit" ≠ "fine"
  • Skips synonyms
  • Doesn't understand the context
  • Requires precise wording

✅ Vector search

  • Searches for meaning
  • "Forfeit" = "fine" = "penalty"
  • Finds synonyms
  • Understands the context
  • Works with natural language

How it works (without math)

Analogy: Imagine that each word is a point in space.

  • “King” and “queen” are close points (relatives)
  • "King" and "car" are distant points (no connection)
  • “Forfeit” and “fine” are close points (synonyms)

AI turns your question into a point and looks for the closest document fragments to it.

Example from life

💡 Example question
"What to do if the supplier is late?"

General search: Will not find anything (there is no word “late” in the contract)

Vector search: Find sections about “penalty”, “late delivery”, “penalties”

✅ Checklist for accepting the result

  • Does the search find synonyms?
  • Does search understand context?
  • Is the response time acceptable? < 3 секунд)
  • Are the results relevant?
  • Does the system scale as documents grow?
  • Is it possible to filter by date/document type?
