Level 16 — Compliance

16.5 Agent Scenarios for Compliance

⏱ 12 min · sanctions, AML, whistleblower, investigations
How is an agent different from a language model?
LLM analyzes on request. Agent works continuously: monitors sanctions lists, analyzes transactions, processes reports — without command "check now". More — Level 4.

Scenario 1: Sanctions Monitoring Agent (n8n + API)

Task: track sanctions list changes, check counterparties, alert on matches.

[Cron: every 4 hours]
↓
[API: sanctions lists update request]
├─→ OFAC SDN
├─→ EU sanctions
├─→ UN
├─→ National lists (compensatory)
└─→ Other national lists (China, UAE, etc.)
↓
[LLM: change analysis]
├─→ New persons/companies
├─→ Changes to existing entries
└─→ Sanctions lifted
↓
[Automatic company counterparty check]
├─→ Match with supplier/customer base
├─→ Match with beneficiary base
└─→ Transit chain check
↓
[On match: critical alert]
├─→ Email to CCO + lawyer
├─→ Block operations (ERP integration)
└─→ Initiate exit procedure
↓
[Daily report: changes, checks, risks]
Result: reaction to sanctions changes within hours, not days. Minimizes "inadvertent violation" risk.

Scenario 2: Transaction Analysis Agent (OpenClaw Agent)

Task: analyze AML alerts, form SAR/STR, maintain documentation.

[Trigger: AML system alert (suspicious transaction)]
↓
[Agent: context collection]
├─→ Counterparty transaction history
├─→ Geography (risky jurisdictions)
├─→ Amount vs client profile
├─→ Timing (non-standard)
└─→ Related persons
↓
[On-prem LLM: justification analysis]
├─→ False alert: reasons
├─→ Requires investigation: questions
└─→ Critical: SAR/STR recommendation
↓
[If confirmed: automatic SAR/STR formation]
├─→ Structure per regulator requirements
├─→ Transaction description
├─→ Suspicion justification
└─→ Attachments
↓
[Notification to compliance officer + log entry]

Scenario 3: Whistleblower Agent (Hybrid)

Task: process reports, classify, route, ensure confidentiality.

[Report received (form, email, phone)]
↓
[Agent: initial processing]
├─→ Text extraction (transcription if audio)
├─→ Classification: corruption / harassment / safety / finance / other
├─→ Urgency assessment
└─→ Duplicate check (similar reports)
↓
[On-prem LLM: anonymization + card formation]
├─→ Remove reporter identifiers (if not anonymous)
├─→ Structure facts
└─→ Form investigation questions
↓
[Routing]
├─→ Corruption → ethics committee
├─→ Harassment → HR + lawyer
├─→ Finance → internal audit
└─→ Safety → security service
↓
[Create task with deadline]
↓
[Monitoring: overdue → escalation to board of directors]
Critical: all whistleblower processing — on-prem. Reporter data never leaves the perimeter. Anonymization is mandatory.
