10.4 Development and Agents

15 min · for PR Directors and Communications Teams

This lesson covers PR tasks that cannot simply be handed off to the cloud. Some work is done by external AI, some only internally. And somewhere you need a development team and agent systems.

Key principle: hybrid approach. Drafts go outside, position alignment goes inside. Public sections go in the cloud, operational numbers stay in a closed loop.

1. Hybrid PR Tasks: Outside and Inside

PR works with information of varying sensitivity. Some materials can be generated in the cloud, others only in a closed loop.

Comments for Journalists

Outside (cloud): draft comments on general facts — market situation, general trends, neutral position.

Inside (on-prem): final position alignment, accounting for internal strategy, number adjustments, compliance check with legal restrictions.

Q&A for Speakers

Outside (cloud): general questions — company strategy, product lineup, market plans.

Inside (on-prem): sensitive topics — sanctions, litigation, emergencies, personnel decisions, negotiations with regulators. These Q&As must never leave the perimeter under any circumstances.

ESG Reporting

Outside (cloud): public sections — mission, sustainable development strategy, general goals.

Inside (on-prem): sections with real operational and environmental data — emissions, energy consumption, incidents, fines. Final report assembly is also done internally.

Incident Postmortem

Outside (cloud): analysis structure on an anonymized case — methodology, template, typical mistakes.

Inside (on-prem): real analysis with names, specific processes, internal data, and conclusions about personnel.

2. Where a Development Team is Needed

Some PR tasks require not just LLMs, but full-fledged development: integrations, pipelines, dashboards.

Media Monitoring Integration

Connecting to Mention, Brand Analytics, SCAN, Medialogia. Developing connectors, processing API responses, unified data format.

Without development: analysts manually download reports from 3–4 systems. With development: unified real-time data stream.

Automatic Press Release Generation

Pipeline: data from internal systems → draft release → alignment in on-prem LLM → publication.

Example: quarterly reports, personnel appointments, product launches — all generated from templates using internal data.

CRM Integration for Stakeholder Briefings

Before a meeting with a journalist or regulator, the system automatically compiles a dossier: interaction history, publications, sentiment, latest comments.

Reputation Metrics Dashboard

In real time: mention sentiment, reach, Share of Voice, topic trends. Updates every 15 minutes.

Automatic Postmortem Analysis

After an incident, the system structuredly collects data: timeline, who did what, what communications were sent, what decisions were made. Forms a framework for analysis.

3. Where Agent Systems are Needed

Agents are not just chatbots. These are autonomous processes that work 24/7, make decisions, and act without human involvement.

Monitoring Agent

Task: 24/7 scanning of media, social networks, forums, messengers.

Triggers: sharp rise in negativity, mentions of key persons, information leaks, competitive attacks.

Action: automatic alerts in Telegram/Slack to the PR Director with threat level and links.

Analyst Agent

Task: collecting reputation metrics for the week.

Output: weekly report for the board of directors — sentiment dynamics, top 5 crisis topics, recommendations for communication strategy.

GR Agent

Task: monitoring legislative initiatives.

Output: automatic briefing for the government relations department — what bills have been introduced, what threatens business, deadlines, recommended actions.

Crisis Agent

Task: trigger scenarios when negativity threshold is reached.

Action: automatically forms a position draft (based on approved templates), prepares Q&A, notifies the PR Director.

Important: the agent does not publish without a human — prepares everything for quick decision-making.

4. Piloting: Where to Start

You don't need to launch everything at once. Break implementation into baskets — from simple to complex, from quick ROI to long-term projects.

BasketWhat's InsideInfrastructureROI
Basket 1 Monitoring, press release generation, social media Cloud API (ChatGPT, Claude) Weeks
Basket 2 Crisis plans, internal communications, position alignment On-prem LLM Months
Basket 3 Media monitoring integration, dashboards, CRM Development team + server infrastructure Quarters
Basket 4 Full automation: agents, triggers, automatic briefings Agent platform + development 12–18 months
Tip: start with Basket 1. Launch cloud monitoring and draft generation — get your first ROI in 2–3 weeks. Simultaneously prepare on-prem for Basket 2. Don't try to build a full agent system right away.

General questions about company strategy

Question 2: What does the crisis agent do when negativity threshold is reached?

Automatically publishes a rebuttal on social media
Forms a position draft and notifies the PR Director

Question 3: Which piloting basket delivers ROI in weeks?

Basket 1: monitoring, press releases, social media in the cloud
Basket 3: media monitoring integration and dashboards
Basket 4: full automation with agents

Question 4: Why should ESG report final assembly be done inside the company?

Because cloud LLMs cannot work with PDFs
Because the report contains real operational and environmental data
Because ESG reports are not subject to audit
