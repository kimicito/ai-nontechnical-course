Level 17 — Personnel

17.5 Agent Scenarios for HR

⏱ 12 min · onboarding, recruitment, 360 assessment
How is an agent different from a language model?
LLM writes on request. Agent works cyclically: sends materials, checks completion, collects feedback, plans interviews — without HR involvement at each step. More — Level 4.

Scenario 1: Onboarding Agent (n8n + LMS)

Task: automate new hire's first days: send materials, check completion, collect feedback.

[Trigger: HR hiring order (HR system)]
↓
[Agent: create onboarding tracker]
├─→ Day 1: welcome letter, access, office map
├─→ Day 2: team introduction (auto-send to colleagues)
├─→ Week 1: training modules (LMS)
├─→ Week 2: manager meeting (auto-scheduling)
├─→ Month 1: interim feedback
└─→ 3 months: final assessment
↓
[Completion monitoring]
├─→ Module not completed on time → reminder
├─→ Test not passed → retake offer
└─→ All modules completed → notify HR
↓
[Feedback collection (auto-survey on day 7, 30, 90)]
↓
[Monthly report HR: onboarding status by new hires]
Result: HR doesn't spend time on onboarding routine. New hire receives consistent program without gaps.

Scenario 2: Recruitment Agent (OpenClaw Agent)

Task: analyze incoming resumes, conduct pre-screening, plan interviews, manage candidate.

[Resume received (email, job board, referral)]
↓
[Agent: data extraction]
├─→ Resume parsing (structuring)
├─→ Comparison with vacancy requirements
├─→ Mandatory criteria check
└─→ Experience assessment (relevance, depth)
↓
[On-prem LLM: pre-screening]
├─→ Match: yes / no / partial
├─→ Strengths
├─→ Risks and interview questions
└─→ Recommendation (interview / reject / reserve)
↓
[If recommendation "interview"]
├─→ Auto-check HR and manager calendars
├─→ Send invitation to candidate
└─→ Create task in HR system
↓
[If reject: personalized thank-you letter]
↓
[Weekly report: funnel, conversion, average time]

Scenario 3: 360 Assessment Agent (Hybrid)

Task: collect assessments, analyze, prepare reports, track completion.

[Trigger: 360 assessment round launch]
↓
[Agent: survey distribution]
├─→ Manager
├─→ Colleagues (3–5 people)
├─→ Subordinates (if any)
└─→ Self-assessment
↓
[Completion monitoring]
├─→ Not filled in 3 days → reminder
├─→ Not filled in 7 days → remind manager
└─→ All filled → launch analysis
↓
[On-prem LLM: analysis]
├─→ Aggregation by competencies
├─→ Comparison with previous period
├─→ Identify gaps (self vs others)
└─→ Form talking points for review
↓
[PDF report: individual + team summary]
↓
[Notification to manager + HR]
Important: all assessments are personal data. Collection and analysis strictly on-prem. Anonymized metrics (company averages) can be taken outside for benchmarking.
