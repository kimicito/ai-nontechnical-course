10.5 Agent Scenarios for PR

⏱ 12 min · media monitoring, crisis communications, ESG
How is an agent different from a language model?
LLM writes text on request. Agent works 24/7: scans hundreds of sources, classifies, alerts, prepares drafts — without command "check what's being written". More — Level 4.

Scenario 1: Media Monitoring Agent (n8n + RSS/API)

Task: track company, competitor, industry mentions; analyze sentiment; form digest.

[Cron: every 2 hours]
↓
[Parallel source polling]
├─→ RSS media (business newspapers)
├─→ Industry Telegram channels
├─→ Google News API
└─→ YouTube (mentions in videos)
↓
[LLM: filter by relevance]
├─→ Direct company mentions
├─→ Competitor mentions
├─→ Regulatory changes
└─→ Industry trends
↓
[LLM: sentiment analysis (positive / neutral / negative)]
↓
[Digest formation]
├─ Top-5 news
├─ Sentiment
└─ Reaction recommendations
↓
[Email to PR director + post in corporate Slack]
Result: PR gets digest instead of manually browsing 50+ sources. Saves 3–4 hours per day.

Scenario 2: Crisis Communications Agent (OpenClaw Agent)

Task: upon detecting negative wave — quickly assess scale, prepare response proposals, route for approval.

[Trigger: >10 negative mentions per hour OR mention in federal media]
↓
[Agent: emergency collection]
├─→ Where outbreak is (platforms, geography)
├─→ Key messages / memes
├─→ Influencers amplifying topic
└─→ Connection to real events
↓
[On-prem LLM: risk analysis]
├─→ Threat level (low / medium / critical)
├─→ Development scenarios
└─→ Response recommendations (silence / refutation / apology / action)
↓
[Response drafts]
├─ For media
├─ For social media
└─ Internal memo for employees
↓
[Notification to PR director: "Crisis: level X, proposals attached"]

Scenario 3: ESG Reporting Agent (Hybrid)

Task: collect data from different systems, form ESG report draft per GRI/SASB standards.

[Request: "Prepare annual ESG report"]
↓
[Agent: data collection from internal systems]
├─→ ERP: energy consumption, emissions
├─→ HR: gender equality, training
├─→ Procurement: supply chain, localization
└─→ Production: safety, waste
↓
[External LLM: analysis per GRI standard]
├─→ Which indicators are covered
├─→ Which data is missing
└─→ Comparison with previous year
↓
[On-prem LLM: report text formation]
↓
[PDF draft + improvement list]
Important: internal data (salaries, injuries, emissions) — on-prem. GRI standard analysis can be done by external AI on anonymized data.
