Level 12 — MRO (Maintenance, Repair, and Operations)

12.2 External MRO Tasks

⏱ 15 min · for reliability engineers and mechanics
Open Data (Cloud AI)
All tasks in this lesson use public sources, general methodologies, open market. You can use Claude, ChatGPT, Gemini.

Why Cloud is Better Than On-Prem for MRO

Cloud models win when you need to:

  • Web search: monitor MRO standards, equipment GOSTs, industry requirements
  • Translation: work with foreign spare parts catalogs, ISO, API
  • Generation: create training materials from scratch, standard procedures
  • Public data analysis: LME metals, equipment prices, benchmarking
  • Analog search: import substitution, cross-reference numbers
Important: before sending to cloud — check that the request contains no inventory numbers, downtime data, or plant financial indicators. If it does — anonymize or send to on-prem.

1. Development and Updating of Maintenance Schedules

General methodology for planned preventive maintenance scheduling — without plant-specific data.

Prompt Example (Cloud AI)
Prepare a methodology for developing a PPM schedule for a metallurgical plant:

1. Equipment criticality classification (A — critical, B — important, C — auxiliary)
2. Maintenance and repair frequency by equipment type
   - Current repair: monthly/quarterly
   - Medium repair: 1–2 times per year
   - Major repair: by operating hours or calendar period
3. Resource reservation (teams, spare parts, equipment)
4. Methodology for schedule adjustment during unplanned failures
5. KPI control (% completion vs plan, downtime)
6. Documentation forms (schedule, work order, certificate)

Without specific inventory numbers or plant data.

Result: methodology for enterprise implementation, saves 4–5 hours.

2. Development of General MRO Procedures

Document structure for typical equipment (pump, mill, compressor) — without plant-specific data.

Prompt Example (Claude with Web Search)
Prepare an MRO procedure structure for a horizontal centrifugal pump (general purpose, water/slurry transfer):

Procedure structure:
1. Scope (pump types, operating conditions)
2. Maintenance types and frequency (daily inspection, TO-1, TO-2, TO-3, major repair)
3. Work list for each maintenance type (checkpoints)
4. Acceptable parameter standards (vibration, temperature, flow)
5. Post-repair acceptance criteria
6. Record forms (inspection log, certificates)
7. Appendices (lubrication diagram, seal diagram)

Without specific inventory numbers or plant data.

Result: ready procedure structure, saves 2–3 hours.

3. Operating Instructions — General Part

Basic principles of equipment operation, if no plant-specific modifications exist.

Prompt Example
Prepare operating instructions for a plate heat exchanger (general purpose):

Sections:
1. Purpose and operating principle
2. Technical specifications (general ranges)
3. Personnel requirements
4. Startup and shutdown procedure
5. Operating control parameters (temperature, pressure, differential)
6. Preventive inspection (what to check, frequency)
7. Typical malfunctions and actions (without specific case references)
8. Safety requirements

Language: simple, for operators and mechanics.

Result: ready instructions, saves 3–4 hours.

4. Training Materials for Maintenance Personnel

Basic principles, safety, typical procedures — without references to real plant incidents.

Prompt Example
Prepare a training module "Safety During Repair of Rotating Equipment" for mechanics:

Format: 10 slides (text for each)
Content:
1. Types of hazards (mechanical, electrical, chemical)
2. LOTO (Lockout/Tagout) — shutdown procedure
3. Personal protective equipment
4. Control questions (5 questions with answers)

Without specific plant cases — only general principles.

Result: ready module, saves 4 hours.

5. Equipment and Spare Parts Analog Search (Import Substitution)

Open market — cross-reference search, analogs, replacements.

Prompt Example (Web Search)
Find analogs for SKF 22320 EK/C3 bearing for import substitution:

Requirements:
- Russian manufacturer or EAEU countries
- Similar dimensions: d=100, D=215, B=73 mm
- Type: spherical roller, bronze cage
- Acceptable load: not lower than original

Output table: manufacturer, designation, dimensions, approximate price, stock availability (if data available).

Result: analog list for procurement, saves 1–2 hours.

6. MRO Practice Benchmarking (RCM, TPM)

Open methodology — best practices in metallurgy, mining, energy.

Prompt Example (Web Search)
Benchmark MRO practices in metallurgy (2024–2026):

Questions:
1. Which major metallurgical companies implemented RCM (Reliability-Centered Maintenance)?
2. What results are claimed (downtime reduction, savings)?
3. What IT systems are used for MRO (SAP PM, 1C MRO, Infor)?
4. What KPI metrics are applied (MTBF, MTTR, OEE)?
5. What MRO conferences in metallurgy are scheduled for 2026?

Prepare a structured report with sources.

Result: industry practices overview, saves 3–4 hours.

7. RCM/TPM Transition Methodology Development

General framework — public approaches, without plant-specific data.

Prompt Example
Prepare a TPM (Total Productive Maintenance) implementation plan for a metallurgical plant:

Stages (by month, 12-month plan):
1. Preparation: personnel training, TPM team creation
2. Autonomous maintenance (operators perform basic care)
3. Planned maintenance (schedules, procedures)
4. Planned repair (major, medium)
5. Training and development (trainings, certification)
6. Early project management (reliability consideration in procurement)
7. Safety and quality (5S, visual control)

For each stage: goals, success metrics, typical problems.

Result: ready implementation plan, saves 5–6 hours.

8. Tender Documentation — General Sections

Standard ToR sections for repair work — without specific deal commercial terms.

Prompt Example
Prepare a standard ToR structure for equipment major repair:

Sections:
1. General provisions (scope, regulatory references)
2. Technical object description (without specific inventory numbers)
3. Work list (typical types: defect inspection, repair, testing)
4. Quality requirements (acceptance criteria, testing)
5. Documentation requirements (certificates, material certificates, photos)
6. Warranty obligations (typical periods)
7. Safety requirements
8. Contractor report form

Place for specific data insertion: [SPECIFY PLANT].

Result: standard ToR template, saves 2 hours.

Question 1: Why is spare parts analog search a cloud AI task?

Because it is public information — catalogs, cross-references, prices
Because suppliers are external contractors
Only if it is import substitution

Question 2: Which MRO task is definitely on-prem?

"Repair vs Replace" justification for a specific compressor
RCM methodology development for metallurgy
Standard ToR structure for overhaul

Question 3: Training materials for mechanics — which loop?

Cloud AI — basic principles, without specific plant incidents
On-prem — any materials for plant personnel
Hybrid — basic principles outside, plant cases inside
