8.5 Agent Scenarios for Construction

⏱ 12 min · automating construction processes with agents
How is an agent different from a language model?
LLM answers individual requests. Agent works cyclically: receives data, analyzes, makes decisions, takes actions — without human involvement at each step. More — Level 4.

Scenario 1: Defect Tracking Agent (n8n + Vision API)

Task: receive photos from site, classify defects, assign responsible persons, track resolution.

[Telegram bot on site: photo + defect description]
↓
[n8n: receive message]
↓
[Vision LLM: classify defect (crack, unevenness, material defect)]
↓
[LLM: determine criticality (critical / serious / cosmetic)]
↓
[Notion/Bitrix24: create task with photo, description, deadline]
↓
[Assign responsible by directory: defect type → department]
↓
[Notification to responsible in Telegram]
Result: defect is recorded in 2 minutes instead of 2 hours (photo → paper act → journal entry → routing).

Scenario 2: Estimate Checking Agent (OpenClaw Agent)

Task: upload estimate, check compliance with standards (local norms), find errors, prepare remarks.

[Upload estimate file (Excel, PDF) to system]
↓
[On-prem agent: extract positions]
├─→ Check: compliance with standards (internal base)
├─→ Check: volume calculations
├─→ Check: coefficients and adjustments
└─→ Check: regional pricing
↓
[LLM: form verification protocol]
├─ Found errors
├─ Correction recommendations
└─ References to regulatory documents
↓
[PDF protocol + notification to estimator]

Scenario 3: Schedule Control Agent (Hybrid)

Task: monitor work schedule, analyze delay risks, warn responsible persons.

[Integration with Project / MS Project / Primavera]
↓
[Agent: daily status reading]
├─→ Plan vs actual comparison
├─→ Critical path analysis
└─→ Identify tasks with delay risk (>3 days deviation)
↓
[LLM: formulate warning]
├─ Which work is at risk
├─ Why (reasons from data)
└─ What to do (options)
↓
[Email to project manager + dashboard update]
Important: schedule data is internal information. Agent works fully on-prem or through DMZ with filtering.
