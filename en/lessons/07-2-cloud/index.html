7.2 External Tasks: Cloud

15 min · for procurement specialists · open data

In this lesson — tasks where the procurement specialist works with public information: searching for suppliers, price benchmarking, preparing anonymized documents. This data does not contain trade secrets — it can be sent to cloud models.

Solution: cloud APIs: Claude, ChatGPT, Kimi, Qwen API. Web search is needed — so the model must be connected to the internet.

When Cloud Beats On-Prem

Cloud models win when you need:
  • Web search: find new suppliers, check market prices
  • Translation: work with foreign documents
  • Generation: create template text from scratch
  • Analysis of public data: raw material indices, exchange rates, customs codes
Important: before sending to the cloud — check that the request does not contain specific prices, supplier names, or volumes. If it does — anonymize it or send to on-prem.

1. Finding Analogs and Alternative Suppliers

The procurement specialist knows 2–3 suppliers for a position. Need to find more — for competition or backup. On-prem has no internet access, cloud — does.

Prompt Example (Claude with web search)
Find manufacturers of industrial centrifugal pumps 
series 32-160 in Russia and neighboring countries (Kazakhstan, Belarus).

Specify:
- Company name and website
- Where production is located (city)
- Whether there are certificates (GOST, ISO)
- Approximate price range (if publicly available)
- Procurement contacts (sales department)

Make a comparison table.
Result: a table of potential suppliers with contacts. The procurement specialist spends 10 minutes instead of 2 hours of manual searching.

2. Benchmarking Market Prices and Raw Material Indices

The procurement specialist is negotiating a price but doesn't know where the market is right now. LLM with web search finds current indices.

Prompt Example
What are the current (2024-2025) market prices for:
1. Steel rebar A500C (wholesale, per ton)
2. Portland cement M500 (wholesale, per ton)
3. Copper cathodes (LME + premium)

Also specify:
- Dynamics over the last 6 months (increase/decrease %)
- Forecast for the next quarter (if available)
- Main factors affecting the price
Important: the request does not contain specific prices of your suppliers — only market benchmarks. This is safe for the cloud.

3. Preparing an Anonymized Technical Specification (Draft)

Technical specification with specific numbers — goes to on-prem. But a draft structure of the TS, general requirements, typical parameters — can be done in the cloud, then finalized internally.

Prompt Example
Prepare a structure for a technical specification for the procurement 
of industrial pump equipment for chemical production.

Without specific numbers (volumes, prices, deadlines). 
Only typical parameters and requirements.

Structure:
1. General requirements
2. Technical parameters (power, capacity, materials)
3. Quality and certification requirements
4. Delivery conditions (standard)
5. Documentation requirements
6. Acceptance criteria

Specify which items will need to be supplemented with specific data inside the company.

4. Tender Documentation (General Sections)

Instructions for participants, evaluation criteria, application form — all of this is template-based and does not contain data about a specific deal.

Prompt Example
Create a tender documentation template for the procurement 
of industrial equipment by open competition method.

Include:
1. Instructions for participants (submission rules, deadlines)
2. Application form (general fields)
3. Evaluation criteria (price, deadlines, experience) — without weighting coefficients
4. Contract form (standard conditions)
5. List of documents for the application

Specify which sections need to be coordinated with the lawyer and the customer.

5. Auto-Responses to Clients and Platforms

Shipment status, delivery deadlines, requests for quotes — often template-based. LLM generates a response based on ERP status, but without specific numbers.

Prompt Example
Write a template email response to a shipment status inquiry.

Conditions:
- Order is in transit
- Supplier reported a delay
- Need to apologize, explain the reason, state the new deadline
- Offer compensation (standard)

Without specific dates, amounts, or supplier names. 
Recipient: internal customer within the company.

6. Translation and Localization of Documents

Foreign suppliers send instructions, specifications, and contracts in English, Chinese, and German. Cloud translation is faster and more accurate than standard translators for technical texts.

Prompt Example
Translate from English to Russian a technical specification 
for an industrial pump (file Spec.pdf).

Requirements:
- Preserve structure and section numbering
- Technical terms — per GOST/ISO
- Units of measurement — in SI (m³/h, bar, kW)
- Preserve tables in markdown format
- Mark unclear parts for clarification with the supplier

7. Checking Customs Requirements (General Regulations)

HS code, certification requirements, duties — this is public information. LLM finds current requirements for a product category.

Prompt Example
What customs requirements and certificates are needed 
for importing industrial centrifugal pumps into the Russian Federation (HS code 8413 70)?

Specify:
- Mandatory certification (GOST-R, TR CU)
- Whether SBKTS is needed
- Customs duty (rate)
- VAT on import
- Requirements for supplier documents (invoice, packing list)
- Specifics for certain countries (China, Germany, Italy)

Without specific cargo and amounts — only general regulations.

Knowledge Check

Question 1: Why is it better to search for new suppliers in the cloud rather than on-prem?

Because on-prem has no internet access for searching
Because on-prem works slower
Because the cloud is free

Question 2: Which model should NOT be used for price benchmarking?

Claude with web search
On-prem LLM without internet
ChatGPT with browsing

Question 3: Which query is safe for the cloud?

"Compare prices of my 3 suppliers: A, B, C" (with specific amounts)
"What are the market prices for rebar A500C in 2025?"
"Check the contract with supplier X against the template"

Question 4: Which task can be done in the cloud if the TS contains specific volumes?

Send the full TS with volumes to ChatGPT
Create a draft TS structure without numbers, then add volumes in on-prem
You cannot use AI for TS at all

Question 5: Which model is best for translating technical specifications?

Claude or GPT-4 — they handle technical terms better
Any free model — translation is a simple task
Only on-prem model, because specifications may contain prices
