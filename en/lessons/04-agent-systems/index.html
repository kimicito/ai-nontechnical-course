4.2 · Agent Systems for Corporate AI

15 min · How a digital employee is built from the inside

Up to 90% of tasks in a manufacturing company can be solved inside the perimeter — with own models, without sending data to the cloud. For this, agent systems exist — frameworks that run AI agents on corporate servers.

This lesson is not about installation and code. It is about how a digital employee is built and what "documents" it consists of — so you can set tasks for the IT department and verify the result.

🎯 Why does business need this?

On-premise = data never leaves the company. Failure logs, smelting parameters, mining plans, commercial forecasts — everything stays inside.

Agent system = digital employee with memory. It doesn't just answer questions — it remembers, learns, acts on schedule, and integrates with your systems (1C, SAP, process control system (SCADA)).

📚 What we are studying

Three types of agent systems

System Purpose Analogy
OpenClaw Universal agent with messengers (Telegram, corporate messenger, Email). Runs on your server. Personal assistant on all communication channels
Hermes Agent Self-learning agent. Creates skills from experience and remembers user preferences. Experienced employee who learns from tasks
GoClaw Enterprise platform for agent teams with data isolation between departments. Corporate personnel management system

Key agent files

An agent is not a "black box". It consists of documents, just as an employee consists of instructions and a personal file. Here is what each file means:

File What it is Business analogy
SOUL.md The "soul" of the agent — its character, communication style, role. Example: "You are an experienced process engineer, speak briefly and to the point". Job description + corporate culture
SKILL.md Skill — step-by-step instruction for a typical task. Example: "How to analyze the failure log". Regulation / standard operating procedure
HEARTBEAT.md Schedule of periodic checks. Example: every morning at 8:00 — check mining plan. Duty schedule and planning meetings
MEMORY.md Long-term memory — facts, decisions, lessons. Example: "Furnace #3 gives defects at temperature >1200°". Knowledge base / observation journal
AGENTS.md General rules for all agents in the system — what is allowed, what is not, red lines. Company code of conduct
USER.md User profile — who you are, what you do, what can be done without asking. Employee personal file + authority
TOOLS.md List of tools the agent has access to: SAP, 1C, Grafana, Email, corporate messenger. List of available programs and systems

🛠️ How to apply

Example: Mining control agent

Agent configuration

SOUL.md: "You are a mining analyst. Speak briefly, use data from the process control system (SCADA). Do not make assumptions without facts."

HEARTBEAT.md: Every day at 7:30 — check actual mining against plan, identify deviations >5%, send summary.

SKILL.md: "Analysis of mining log: 1) Load data from process control system (SCADA), 2) Compare with plan by drills, 3) Identify deviations, 4) Give recommendations, 5) Send to corporate messenger."

MEMORY.md: "Well #47 gives unstable flow rate at pressure <15 atm. In March 2026 there was a rod break."

Result: The chief engineer receives a summary in the corporate messenger before the shift starts — with facts, deviations, and recommendations. Manual reconciliation time is reduced from 2 hours to 5 minutes.

Example: Quality control agent

Agent configuration

SOUL.md: "You are a quality engineer. Check smelting parameters against the standard. Point out only deviations."

HEARTBEAT.md: Every 4 hours — check temperature, pressure, chemical composition of the last batch. On deviation — immediate notification.

TOOLS.md: Access to process control system (SCADA) database, corporate messenger for alerts, Excel for reports.

Result: Defects are detected in real time, not at shipment. Savings on returns and re-melting.

✅ Checklist for accepting the result

  • Does the agent run on a corporate server (on-prem)?
  • Does data not go to external clouds?
  • Does SOUL.md reflect the role and style needed by the business?
  • Does HEARTBEAT.md cover key checks (not too frequent)?
  • Does SKILL.md describe tasks step by step, like a regulation?
  • Does MEMORY.md accumulate useful facts (not garbage)?
  • Is TOOLS.md limited to necessary systems (principle of least privilege)?
  • Is there a "human in the loop" for critical decisions?

📝 Checking understanding

Question 1: What does the SOUL.md file define?

List of tools the agent has access to
Character, communication style and role of the agent
Schedule of periodic checks

Question 2: What is HEARTBEAT.md for?

Store the agent's long-term memory
Set the schedule of periodic agent tasks
Describe step-by-step instructions for skills

Question 3: Why is it critical to use on-prem agents for production data?

On-prem agents work faster than cloud ones
Data (mining parameters, plans, failures) does not leave the company perimeter
On-prem agents are cheaper to maintain

💡 Summary

Agent system is not magic, but a set of documents (SOUL, SKILL, HEARTBEAT, MEMORY) that define the behavior of a digital employee.

On-prem guarantees that your production data stays inside the company.

Key manager skill: being able to formulate the content of these documents — like job descriptions for a human, but for an agent.
