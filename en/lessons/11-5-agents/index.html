11.5 Agent Scenarios for Quality

⏱ 12 min · agents for quality control and claims
How is an agent different from a language model?
LLM analyzes on request. Agent works continuously: receives data, classifies, routes, tracks deadlines — without human involvement. More — Level 4.

Scenario 1: Claim Processing Agent (n8n + LLM)

Task: upon claim receipt — classify, assign responsible, track deadlines.

[Email / ECM: new customer claim]
↓
[n8n: extract attachments and text]
↓
[LLM: classification]
├─→ Type: product quality / logistics / documents / price
├─→ Criticality: critical / serious / standard
└─→ Potential loss amount
↓
[Assign responsible by directory]
↓
[Create task in ECM with deadline]
├─ Critical: 24 hours
├─ Serious: 72 hours
└─ Standard: 5 days
↓
[Notify responsible + manager for critical]
↓
[Daily monitoring: overdue → escalation]
Result: claim response time reduced from 1–2 days to hours. No claim is lost.

Scenario 2: Root-Cause Analysis Agent (OpenClaw Agent)

Task: upon defect data receipt — automatically perform cause analysis, build Ishikawa diagram, propose solutions.

[Input: defect description + data (photos, process parameters)]
↓
[On-prem agent: collect historical data]
├─→ Similar defects for period
├─→ Process changes (raw materials, equipment, personnel)
└─→ Correlations (temperature, pressure, speed)
↓
[LLM: build Ishikawa diagram]
├─→ 6M: Man, Machine, Material, Method, Measurement, Environment
├─→ Probable root causes with justification
└─→ References to similar cases
↓
[Propose corrective actions]
├─ Short-term (eliminate consequences)
└─ Long-term (prevent recurrence)
↓
[Report to quality engineer]

Scenario 3: 8D Report Preparation Agent (Hybrid)

Task: collect data on all 8D stages, form report, track action completion.

[Trigger: 8D card created in system]
↓
[Agent: collect data by stages]
D1: Team → responsible list
D2: Problem description → extract from request
D3: Interim measures → check completion
D4: Root cause → from root-cause agent
D5–D7: Corrective actions → track
D8: Team congratulations → form
↓
[LLM: form report text]
↓
[PDF + notify team]
↓
[Monitoring: D5–D7 overdue → reminder]
