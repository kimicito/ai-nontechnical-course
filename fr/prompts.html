🎯 Exemples de Prompts pour différentes professions

Modèles prêts à l'emploi pour des tâches réelles. Copiez, collez dans GENAi (ou un autre chat IA), remplacez [texte entre crochets] par vos données.

📋 Comment utiliser ces exemples

Chaque prompt est construit selon la formule RCOFE (Rôle-Contexte-Objet-Format-Évaluation) et adapté pour des solutions on-premise comme GENAi :

  • Aucun besoin d'API, de programmation ou de JSON
  • Fonctionne dans n'importe quelle interface de chat (OpenWebUI, ChatGPT, Claude)
  • Pour les documents confidentiels, utilisez GENAi en interne
  • Vérifiez toujours le résultat avant de l'envoyer au client / au tribunal / en reporting

Responsable RH
Recrutement, intégration, marque employeur
Rédiger un plan d'intégration (onboarding) d'un nouveau collaborateur
Rôle : Tu es un responsable RH expérimenté dans l'intégration de collaborateurs dans une entreprise IT. Tâche : Élabore un plan d'intégration de 5 jours pour [poste, par exemple : "chargé de clientèle"]. Contexte : - Entreprise : [nom/description] - Format de travail : [bureau/hybride/télétravail] - Outils clés : [liste : CRM, chat d'entreprise, etc.] - Parrain : [nom/poste] Format : Tableau avec colonnes : Jour | Heure | Activité | Responsable | Objectif | Résultat Évaluation : Vérifie que tous les processus clés (accès, rencontre avec l'équipe, premières tâches) sont répartis logiquement sur les jours.
Préparer une FAQ sur une nouvelle politique pour les collaborateurs
Rôle : Tu es un communicant RH. Tu sais expliquer des politiques complexes en langage simple. Tâche : Rédige une FAQ pour les collaborateurs sur le thème : [thème, par exemple : "nouvelle politique de télétravail"]. Contexte : - Ce qui a changé : [brève description des changements] - Top 5 des questions des collaborateurs : [liste] - Ton : chaleureux, bienveillant, sans jargon bureaucratique Format : 7-10 questions avec réponses. Chaque réponse — 3 phrases maximum. À la fin — où s'adresser pour des questions complémentaires. Évaluation : Vérifie que les réponses ne se contredisent pas et couvrent les scénarios principaux.
Rédiger une annonce de poste interne
Rôle : Tu es un responsable de marque employeur. Tu rédiges des annonces pour lesquelles on a envie de postuler. Tâche : Rédige une description de poste interne pour le poste de [poste]. Contexte : - Département : [nom] - Tâches clés : [3-5 points] - Prérequis : [obligatoires et souhaités] - Pourquoi c'est génial : [2-3 atouts uniques du poste/projet] Format : Brève introduction (1 paragraphe) → Responsabilités (liste) → Prérequis (liste) → Ce que nous offrons (liste) → Comment postuler. Évaluation : Vérifie que le texte sonne vivant, sans clichés du type "entreprise dynamique en pleine croissance".

Juriste
Contrats, risques, conformité
Réaliser une première vérification des risques d'un contrat
Rôle : Tu es un juriste d'entreprise avec de l'expérience dans [secteur, par exemple : "IT/télécoms"]. Tâche : Effectue une analyse préliminaire du contrat ci-joint. Identifie les risques pour [partie : "client" ou "prestataire"]. Format : Tableau avec colonnes : - Clause du contrat (citer le texte) - Risque (bref) - Niveau de risque (Élevé/Moyen/Faible) - Recommandation (comment atténuer) Limites : - Ne donne pas de conseils juridiques — seulement des signaux d'alerte - Si tu n'es pas sûr de l'interprétation — indique explicitement "Nécessite une vérification par un spécialiste" - Indique quelles clauses standard sont absentes (force majeure, modalité de modification, etc.) Évaluation : Revérifie que toutes les citations correspondent exactement au texte du contrat.
Rédiger un protocole de désaccord pour un contrat
Rôle : Tu es un juriste en droit des contrats. Tu prépares des documents pour des négociations. Tâche : Sur la base du contrat ci-joint, rédige un protocole de désaccord. Contexte : - Notre position : [brève description de ce que nous cherchons à obtenir] - Points de litige clés : [liste des numéros de clauses ou des thèmes] - Droit applicable : [par exemple, "Code civil français"] Format : Tableau — Clause du contrat (telle quelle) | Notre position | Justification | Proposition de rédaction Introduction : 1-2 paragraphes sur l'objectif du protocole et l'approche générale des négociations. Évaluation : Vérifie que les formulations proposées sont juridiquement correctes et ne créent pas de nouveaux risques.
Préparer un guide sur les CND pour les collaborateurs
Rôle : Tu es un juriste en propriété intellectuelle. Tu expliques le complexe de manière simple. Tâche : Rédige un guide pour les collaborateurs : "Ce qu'il ne faut pas divulguer selon la CND". Contexte : - Type d'entreprise : [produit IT / conseil / fabrication] - Actifs principaux : [code/base clients/savoir-faire/données financières] - Ton : amical mais clair. Sans panique, mais avec conscience des conséquences Format : 1. Qu'est-ce qu'une CND (2 phrases) 2. Ce qui est considéré comme confidentiel (liste avec exemples) 3. Ce qu'on peut faire (liste verte) 4. Ce qu'on ne peut pas faire (liste rouge) 5. Que faire en cas de doute (algorithme d'action) 6. À qui s'adresser pour des questions Évaluation : Vérifie que le texte ne contredit pas la législation en vigueur [indiquer le pays].

Responsable marketing
Contenu, SMM, email marketing
Rédiger un post LinkedIn à partir d'un contenu existant
Rôle : Tu es un content marketer en B2B. Tu rédiges des posts qui génèrent de l'engagement et des leads. Tâche : Crée un post LinkedIn à partir du contenu suivant : [insérer le texte du cas/artcile/actualité]. Contexte : - Cible : [par exemple : "DRH de PME"] - Ton : expert mais conversationnel. Sans jargon administratif - Objectif du post : [informer / attirer des leads / annoncer un produit] - Hashtags : maximum 5, pertinents pour le sujet Format : - Accroche (première phrase — captivante) - Idée principale (2-3 paragraphes courts) - Conclusion / appel à l'action - Hashtags Contrainte : Maximum 150 mots. Le post doit être lisible en diagonal — utilise des paragraphes de 1-2 phrases. Évaluation : Vérifie que le post ne sonne pas comme une publicité, mais apporte de la valeur au lecteur.
Créer une campagne d'emailing pour un segment de clients
Rôle : Tu es un email marketer. Tes emails sont ouverts et cliqués. Tâche : Rédige une séquence de 3 emails pour le segment : [description du segment, par exemple : "clients inactifs depuis 90 jours"]. Contexte : - Produit : [brève description] - Offre : [ce que nous proposons] - Ton : [chaleureux/formel/ludique] - Longueur : chaque email — 150 mots maximum Format pour chaque email : - Objet de l'email (3 variantes au choix) - Aperçu (preview text) - Corps de l'email - CTA (bouton/lien avec texte) Contraintes : - Email 1 : rappel de la valeur - Email 2 : preuve sociale ou cas d'usage - Email 3 : dernière chance + offre Évaluation : Vérifie que les objets ne partent pas dans les spams (pas de "GRATUIT", "URGENT" en majuscules).
Élaborer un plan de calendrier éditorial pour un mois
Rôle : Tu es un stratège de contenu. Tu planifies des calendriers qui nourrissent l'entonnoir. Tâche : Établis un plan de contenu pour [mois] pour [canal : LinkedIn / blog / Telegram]. Contexte : - Thématique : [par exemple : "IA pour les entreprises"] - Cible : [description] - Fréquence de publication : [par exemple : "3 fois par semaine"] - Formats : [posts/cas d'usage/sondages/vidéos] Format : Tableau — Date | Thème | Format | Objectif (notoriété/leads/engagement) | Appel à l'action | Statut En complément : - 3 idées de posts "viraux" - 2 thèmes pour des collaborations avec des experts - 1 proposition de repurposing (comment utiliser un même contenu dans 3 formats) Évaluation : Vérifie que les thèmes couvrent les différentes étapes de l'entonnoir (pas seulement la vente).

Responsable financier / FP&A
Analyse, reporting, prévision
Rédiger un executive summary à partir d'un long rapport
Rôle : Tu es un analyste financier. Tu prépares des summaries pour le PDG, qui n'a que 5 minutes. Tâche : Transforme le rapport ci-joint en un executive summary d'une page. Contexte : - Audience : [PDG / investisseurs / banque] - Ce qui est important : [métrique clé/événement] - Ton : direct, sans flou, chiffres dans le premier paragraphe Format : 1. Conclusion principale (1-2 phrases, avec le chiffre clé) 2. 3 métriques clés (réalisé vs plan vs prévision) 3. Ce qui a bien marché (2-3 points) 4. Ce qui nécessite attention (2-3 points avec recommandations) 5. Prochaines étapes (3 actions avec responsables et délais) Contrainte : Maximum 400 mots. Chaque point — 2 phrases maximum. Évaluation : Vérifie que tous les chiffres correspondent exactement au rapport source. N'arrondis pas sans le mentionner.
Analyser les écarts budgétaires (variance analysis)
Rôle : Tu es un analyste FP&A. Tu identifies les causes des écarts et proposes des actions. Tâche : Analyse les écarts entre le réalisé et le budget pour [période]. Données : [insérer le tableau : rubrique | budget | réalisé | écart | écart %] Format : Tableau — Rubrique | Écart | Cause probable | Recommandation | Responsable Exigences : - Focus sur les écarts > [seuil, par exemple : 10% ou 500K €] - Pour chaque rubrique — 1 cause supposée maximum - La recommandation doit être concrète (pas "analyser", mais "négocier avec le fournisseur X avant le [date]") Évaluation : Vérifie que les écarts sont calculés correctement et que les causes sont logiques.
Préparer un commentaire sur la prévision de chiffre d'affaires
Rôle : Tu es un directeur financier. Tu expliques les chiffres de manière à ce que le business comprenne. Tâche : Rédige un commentaire sur la prévision de chiffre d'affaires pour [trimestre/année]. Contexte : - Prévision : [montant] - Scénario de base : [hypothèses] - Scénario optimiste : [hypothèses] - Scénario pessimiste : [hypothèses] - Drivers clés : [facteurs de croissance] - Risques : [principaux] Format : 1. Conclusion générale (2 phrases) 2. Scénario de base : logique et drivers clés 3. Sensibilité : comment évolue le résultat si [facteur] change de ±10% 4. Risques et atténuations 5. Ce qui est attendu du business (demandes de données/décisions) Ton : analytique mais accessible. Sans jargon financier. Évaluation : Vérifie que les scénarios sont cohérents entre eux et que les drivers mènent logiquement au résultat.

Comptable
Comptabilité, reporting, clôture mensuelle
Établir une checklist de clôture mensuelle
Rôle : Tu es un chef comptable. Tu connais toutes les étapes de la clôture mensuelle. Tâche : Établis une checklist de clôture mensuelle pour [type d'entreprise, par exemple : "société commerciale en IS"]. Contexte : - Système de comptabilité : [ERP / SAP / autre] - Spécificités : [import/export / nombreux fournisseurs / coût de revient complexe] - Délai de clôture : [par exemple : "5 jours ouvrés"] Format : Tableau — Jour | Procédure | Responsable | Point de contrôle (quoi vérifier) | Erreurs fréquentes Exigences : - Indique la séquence (quoi d'abord : amortissement ou réévaluation devises ?) - Mets en évidence les étapes critiques dont dépend le reste - Ajoute une colonne "Si ça ne concorde pas — que faire" Évaluation : Vérifie que la séquence est logique et que rien n'est oublié.
Analyser les fluctuations dans un rapport
Rôle : Tu es un comptable analyste. Tu détectes les anomalies dans les chiffres. Tâche : Analyse le rapport ci-joint et trouve les écarts significatifs. Contexte : - Type de rapport : [balance / grand livre / état] - Périodes de comparaison : [mois en cours vs précédent / mois en cours vs même mois N-1] - Matérialité : [seuil, par exemple : "100K € ou 10%"] Format : Tableau — Rubrique | Période en cours | Période précédente | Écart | % | Cause probable | Vérification nécessaire ? Exigences : - Trie d'abord par écart absolu (les plus grands en haut) - Pour chaque rubrique > seuil — propose une cause - Indique explicitement : "Nécessite un complément d'information auprès du responsable" ou "Variation normale" Évaluation : Vérifie que les calculs d'écarts sont corrects.
Vérifier la conformité des écritures à la norme
Rôle : Tu es un comptable méthodologue. Tu connais [norme : PCG / IFRS / US GAAP]. Tâche : Vérifie si les écritures ci-jointes respectent les exigences de la norme. Contexte : - Norme : [préciser] - Opération : [description de l'opération économique] - Écritures : [Débit | Crédit | Montant | Commentaire] Format : Tableau — Écriture | Correct/Erreur | Si erreur — quoi est incorrect | Variante correcte | Référence à la norme (point/article) Contraintes : - Si tu n'es pas sûr — écris "Nécessite consultation du méthodologue" - Ne donne pas de conseils juridiques - Indique quels documents doivent justifier l'opération Évaluation : Vérifie que les références à la norme sont à jour (si tu cites un point précis).

Responsable commercial
Propositions, négociations, CRM
Préparer une proposition commerciale
Rôle : Tu es un responsable commercial B2B. Tu rédiges des propositions qui concluent des affaires. Tâche : Rédige une proposition commerciale pour [société client]. Contexte : - Notre produit : [description] - Besoin du client : [déduit de la correspondance/entretien] - Budget client : [fourchette, si connu] - Délais : [si connus] - Concurrents : [ceux que le client étudie, si connu] Format : 1. Salutation (personnalisée) 2. Compréhension de la problématique du client (montre que tu as écouté) 3. Solution (comment notre produit résout le problème) 4. Contenu de la proposition (liste + prix) 5. Délais de déploiement 6. Prochaine étape (CTA concret) Ton : confiant mais pas intrusif. Focus sur la valeur, pas sur les fonctionnalités. Évaluation : Vérifie que la proposition s'adresse à une douleur spécifique du client, et n'est pas un modèle générique.
Rédiger un email de suivi après un rendez-vous
Rôle : Tu es un responsable commercial. Tes emails reçoivent des réponses. Tâche : Rédige un follow-up après un rendez-vous avec [nom/poste du client]. Contexte : - Sujet du rendez-vous : [de quoi on a parlé] - Points clés : [3-5 points] - Promesses de notre part : [ce qu'on s'est engagés à envoyer/faire] - Engagements du client : [ce que le client s'est engagé à faire] - Prochaine étape : [concrètement] - Ton : [formel/amicable] Format : - Remerciement pour le temps (1 phrase) - Bref résumé du rendez-vous (3-4 points) - Ce que nous faisons (avec dates) - Ce qu'on attend du client (avec dates) - Proposition de prochain contact Contrainte : Maximum 200 mots. Les clients ne lisent pas les longs emails. Évaluation : Vérifie que toutes les dates et promesses sont alignées avec le compte-rendu du rendez-vous.
Analyser un entonnoir de vente
Rôle : Tu es un analyste commercial. Tu identifies les goulots d'étranglement dans l'entonnoir. Tâche : Analyse l'entonnoir de vente pour [période]. Données : [insérer le tableau : étape | quantité | taux de conversion | panier moyen | cycle] Format : 1. Vue d'ensemble (2-3 phrases) 2. Tableau — Étape | Conversion | Problème (oui/non) | Hypothèse de cause | Recommandation 3. Top 3 priorités d'amélioration 4. Prévision : que se passe-t-il si on améliore le taux de conversion de [X]% à chaque étape Exigences : - Compare avec la période précédente (si les données existent) - Indique à quelle étape on perd le plus d'argent - Les recommandations — concrètes, pas "il faut plus de leads" Évaluation : Vérifie que les calculs de conversion sont corrects et que les priorités sont financièrement justifiées.

Chef de projet
Planification, risques, communication
Établir un plan de projet avec ressources et risques
Rôle : Tu es un chef de projet avec de l'expérience dans [secteur : IT/construction/marketing]. Tâche : Établis un plan de projet : [nom du projet]. Contexte : - Objectif du projet : [objectif SMART] - Budget : [montant] - Délais : [date début — date fin] - Équipe : [rôles et nombre] - Contraintes clés : [par exemple : "ne pas interrompre les ventes en cours"] Format : 1. WBS (structure de découpage du travail) — liste des phases avec sous-tâches 2. Tableau — Tâche | Responsable | Délais | Dépendances | Statut 3. Risques — tableau : Risque | Probabilité | Impact | Atténuation | Propriétaire du risque 4. Chemin critique (3-5 tâches dont dépend le délai) 5. Points de contrôle (quoi vérifier à chaque étape) Évaluation : Vérifie que le plan est réaliste (pas 3 tâches en 1 jour pour une seule personne).
Conduire une rétrospective de sprint/projet
Rôle : Tu es un Scrum Master. Tu conduis des rétrospectives après lesquelles l'équipe progresse. Tâche : Prépare un modèle de rétrospective pour [équipe/projet] sur [période]. Contexte : - Ce qui était prévu : [objectifs/engagements] - Ce qui a réussi : [faits, métriques] - Ce qui n'a pas réussi : [problèmes] - Composition de l'équipe : [rôles] Format : 1. Tableau de bord du sprint (3-4 chiffres : prévu/réalisé, vélocité, bugs) 2. Ce qui a bien fonctionné (3 points avec précision, pas "l'équipe est géniale") 3. Ce qui peut être amélioré (3 points avec cause racine) 4. Expérimentations pour le prochain sprint (2-3 actions avec propriétaires et métriques de succès) 5. Remerciements (espace pour les collègues) Ton : constructif, sans recherche de coupables. Évaluation : Vérifie que les actions du point 4 sont mesurables et assignées à des personnes concrètes.
Rédiger un rapport de statut de projet (status update)
Rôle : Tu es un chef de projet. Tu rédiges des status updates qui sont lus. Tâche : Prépare un status update hebdomadaire pour [audience : client/direction/équipe]. Contexte : - Projet : [nom] - Phase actuelle : [nom] - Avancement : [% réalisé] - Budget : [dépensé/total] - Principales réalisations de la semaine : [2-3 points] - Bloqueurs : [ce qui bloque] - Besoin d'aide : [de qui et sur quoi] Format : 1. Feu tricolore (🟢/🟡/🔴) sur 4 dimensions : délai, budget, qualité, risques 2. Ce qui a été fait (3-5 points) 3. Ce qui est prévu (3-5 points avec délais) 4. Bloqueurs et besoin d'aide 5. Décisions clés à prendre Contrainte : Maximum 1 page. La direction ne lit pas les rapports longs. Évaluation : Vérifie que le statut est objectif — pas de problèmes cachés derrière un feu vert.

Recruteur
Recherche, entretiens, évaluation de candidats
Élaborer des questions comportementales pour un entretien
Rôle : Tu es un recruteur IT. Tu conçois des questions qui prédisent la réussite sur le poste. Tâche : Crée 8 questions comportementales pour un entretien sur le poste de [poste]. Contexte : - Compétences clés : [liste : par exemple, "gestion de conflits", "décomposition des tâches", "travail dans l'incertitude"] - Niveau : [junior/intermédiaire/senior/lead] - Format de l'entretien : [45 minutes / 1 heure] - Entreprise : [brève description de la culture] Format : Tableau — Compétence | Question | Ce qu'on écoute dans la réponse | Signaux d'alerte Exigences : - Questions selon la méthode STAR (Situation-Tâche-Action-Résultat) - Pour chaque compétence — 1-2 questions - Ajoute 1-2 questions sur l'adéquation culturelle Évaluation : Vérifie que les questions sont ouvertes (pas "oui/non") et mesurent ce qui est important pour le poste.
Analyser le CV d'un candidat
Rôle : Tu es un recruteur. Tu analyses les CV pour leur adéquation au poste. Tâche : Compare le CV du candidat avec les exigences du poste. Contexte : - Poste : [poste, exigences clés] - CV : [insérer le texte du CV ou les points clés] Format : Tableau — Exigence | Adéquation (Oui/Non/Partiel) | Confirmation dans le CV | Questions pour l'entretien En complément : - Évaluation globale de l'adéquation (%) - Points forts du candidat (2-3 points) - Risques potentiels (2-3 points) - Recommandation : "Inviter" / "Inviter avec réserves" / "Refuser" Évaluation : Vérifie que l'évaluation est objective et basée uniquement sur les données du CV (pas de suppositions).
Rédiger un refus à un candidat (en préservant la réputation)
Rôle : Tu es un responsable RH. Tu sais dire "non" de manière à ce que la personne respecte l'entreprise. Tâche : Rédige un email de refus au candidat [nom]. Contexte : - Poste : [nom] - Étape : [à quelle étape le refus — après le CV/entretien/test] - Raison : [manque d'expérience en X / un autre candidat a été choisi / décision du client] - Candidat : [brève description, pourquoi il est bon, même s'il n'a pas été sélectionné] Format : - Remerciement pour l'intérêt (1 phrase) - Décision (honnête mais tactique) - Ce qu'on a apprécié chez le candidat (concret — pas de généralités) - Retour d'expérience, si pertinent (1 point constructif) - Invitation à rester en contact (si le candidat est fort) Ton : respectueux, chaleureux. Sans modèles "nous conserverons votre CV". Évaluation : Lis du point de vue du candidat. Si tu recevais un tel email — quel sentiment ça provoquerait ?

Responsable RP
Communiqués de presse, gestion de crise, médias
Rédiger un communiqué de presse sur un événement
Rôle : Tu es un responsable RP. Tu rédiges des communiqués de presse que les médias publient. Tâche : Rédige un communiqué de presse sur le thème : [événement, par exemple : "lancement d'un nouveau produit / levée de fonds / partenariat"]. Contexte : - Entreprise : [nom, description de l'activité] - Actualité : [ce qui s'est passé, faits, chiffres] - Citation du dirigeant : [idée clé, si disponible] - Médias cibles : [affaires/tech/grand public] Format (communiqué de presse standard) : 1. Titre (informatif, sans hyperbole) 2. Lead (qui, quoi, quand, où, pourquoi — 1 paragraphe) 3. Texte principal (2-3 paragraphes avec faits) 4. Citation (1 paragraphe) 5. À propos de l'entreprise (boilerplate) 6. Contacts presse Contraintes : - Maximum 500 mots - Sans superlatifs ("meilleur", "premier", "révolutionnaire") — sauf preuve - Tous les chiffres et faits vérifiables Évaluation : Vérifie que le lead répond aux 5W1H et contient l'actualité principale.
Préparer une réponse à un retour négatif sur les réseaux/dans les médias
Rôle : Tu es un responsable RP en gestion de crise. Tu travailles vite et juste. Tâche : Prépare une réponse à une publication/commentaire. Contexte : - Source : [où c'est publié, crédibilité] - Fond de la réclamation : [bref] - Faits : [ce qui s'est réellement passé] - Notre position : [ce qu'on est prêts à reconnaître/ce qu'on conteste] - Audience : [clients/partenaires/grand public] Format : 1. Évaluation rapide : niveau de menace (faible/moyen/élevé) et pourquoi 2. 3 variantes de réponse : - Variante A : diplomatique, douce - Variante B : directe, factuelle - Variante C : brève, sans émotions 3. Recommandation : quelle variante choisir et pourquoi 4. Prochaines étapes (quoi faire après publication de la réponse) Contrainte : Maximum 150 mots par réponse. En crise, la brièveté est la sœur du talent. Évaluation : Vérifie que la réponse ne sonne pas comme une excuse et n'ajoute pas d'huile sur le feu.
Établir un plan de communication pour un lancement
Rôle : Tu es un stratège RP. Tu planifies des communications qui atteignent les objectifs. Tâche : Établis un plan de communication pour le lancement de [produit/événement]. Contexte : - Date de lancement : [quand] - Cible : [qui] - Message clé : [1 phrase] - Canaux : [lesquels on utilise] - Budget : [si limité] Format : Tableau — Phase | Date | Canal | Format | Message | Responsable | KPI Phases : - Pré-lancement (2 semaines avant) : teasers, annonce - Jour J : communiqué de presse, posts, newsletter - Post-lancement (1-2 semaines) : cas d'usage, témoignages, bilan En complément : - Matrice de messages : quoi dire aux différentes audiences - Plan B : quoi faire si le lancement est reporté Évaluation : Vérifie que le plan est réaliste en délais et en ressources.

Analyste business
Analyse de marché, processus, exigences
Réaliser une analyse SWOT d'une entreprise/d'un produit
Rôle : Tu es un analyste business. Tu fais des SWOT sur lesquels on prend des décisions. Tâche : Établis une analyse SWOT pour [entreprise/produit/direction]. Contexte : - Secteur : [nom] - Taille : [chiffre d'affaires, employés, zone géographique] - Produits clés : [liste] - Concurrents : [3-5 principaux] - Marché : [tendances, dynamique] Format : Tableau 2×2 — Forces | Faiblesses | Opportunités | Menaces Exigences : - Dans chaque case — 4-5 points, triés par impact - Chaque point — concret, avec explication "pourquoi" - Forces et Faiblesses — facteurs internes - Opportunités et Menaces — facteurs externes En complément : - 2 stratégies SO (on utilise nos forces pour les opportunités) - 2 stratégies WT (comment neutraliser les faiblesses et les menaces) Évaluation : Vérifie que les facteurs sont objectifs et basés sur des données, pas des suppositions.
Rédiger des exigences fonctionnelles (BRD)
Rôle : Tu es un analyste business. Tu rédiges des exigences que les développeurs implémentent sans questions. Tâche : Établis un document d'exigences fonctionnelles pour la fonctionnalité : [nom de la fonctionnalité]. Contexte : - Produit : [nom] - Utilisateur : [qui va l'utiliser] - Problème : [ce qui ne va pas actuellement] - Résultat attendu : [ce que ça donnera après implémentation] - Contraintes : [délais, budget, intégrations] Format : 1. Description générale (2-3 paragraphes) 2. Objectifs business (SMART) 3. Scénarios utilisateurs (2-3 scénarios au format "En tant que [rôle], je veux [action], afin de [objectif]") 4. Exigences fonctionnelles (liste avec priorités : Must/Should/Could) 5. Exigences non fonctionnelles (performance, sécurité, scalabilité) 6. Critères d'acceptation (quand considère-t-on la tâche comme terminée) 7. Questions ouvertes Évaluation : Vérifie que les exigences sont uniques et vérifiables (pas de mots "pratique", "rapide" sans chiffres).
Analyser le marché des fournisseurs/concurrents
Rôle : Tu es un analyste de marché. Tu collectes et structures les données pour la prise de décision. Tâche : Compare [fournisseurs / concurrents / solutions] pour [besoin de l'entreprise]. Contexte : - Ce qu'on cherche : [par exemple : "un CRM pour une entreprise B2B de 100 personnes"] - Critères : [liste : prix, fonctionnalités, intégrations, support, délai de déploiement] - Pondération des critères : [si disponible] - Budget : [fourchette] Format : 1. Tableau comparatif — Critère | Option A | Option B | Option C | Vainqueur 2. Analyse détaillée par critère (2-3 phrases) 3. Risques cachés (ce qui peut mal tourner avec chaque option) 4. Recommandation avec justification 5. Prochaines étapes (quoi faire pour prendre la décision) Contraintes : - N'utilise pas de données de plus de [année] si on parle de prix - Si tu n'es pas sûr d'un chiffre — indique "Nécessite un complément d'information" - Cite les sources, si disponibles Évaluation : Vérifie que la comparaison est objective et couvre tous les critères.

Appels d'offres, négociations, contrats
Comparer les propositions commerciales de fournisseurs
Rôle : Tu es un spécialiste des achats. Tu analyses les propositions pour choisir la meilleure offre. Tâche : Compare les propositions commerciales de [nombre] fournisseurs. Données : [insérer le tableau : fournisseur | prix | délai | conditions de paiement | garanties | services supplémentaires] Contexte : - Ce qu'on achète : [description] - Facteurs critiques : [par exemple : "le délai de livraison est 20% plus important que le prix"] - Contraintes : [budget, certification obligatoire, etc.] Format : Tableau — Critère | Poids | Fournisseur A | Fournisseur B | Fournisseur C | Vainqueur Bilan : - 1er-2e-3e place avec justification - Recommandation : avec qui négocier - Ce qu'il faut encore clarifier avec le vainqueur Évaluation : Vérifie que la comparaison couvre tous les critères et que les poids sont appliqués correctement.
Vérifier les conditions d'un contrat de fourniture
Rôle : Tu es un acheteur avec un sens juridique. Tu vérifies les contrats avant signature. Tâche : Analyse les conditions du contrat de fourniture. Contexte : - Notre rôle : [acheteur / vendeur] - Produit/service : [description] - Risques typiques : [par exemple : "retard de livraison", "produit défectueux", "modification de prix"] Format : Tableau — Condition | Ce qui est écrit | Norme/Risque | Recommandation Points clés à vérifier : - Prix et modalité de modification - Délais de livraison et responsabilité en cas de retard - Conditions de paiement (acompte/post-paiement/accord de crédit) - Transfert de risque (Incoterms) - Garanties et procédure de réclamation - Force majeure et mode de résolution des litiges Évaluation : Vérifie qu'il n'y a pas de risques disproportionnés pour notre partie.
Rédiger un cahier des charges (TDR pour le fournisseur)
Rôle : Tu es un ingénieur d'affaires. Tu rédiges des TDR sur lesquels les fournisseurs donnent des prix précis. Tâche : Rédige un cahier des charges pour l'achat de [produit/service]. Contexte : - Ce qu'on achète : [description détaillée] - Volume : [quantité, périodicité] - Exigences : [techniques/qualité/logistique] - Normes : [ISO/spécifications] Format : 1. Informations générales (sur l'entreprise, sur l'achat) 2. Exigences techniques (liste avec tolérances) 3. Exigences d'emballage et d'étiquetage 4. Logistique (conditions de livraison, délais, adresse) 5. Documentation (quels documents sont nécessaires : certificats, fiches techniques) 6. Procédure de réception et de réclamation 7. Forme de réponse du fournisseur (ce qu'il doit indiquer dans sa réponse) Contraintes : - Ne mentionne pas de marques spécifiques (sauf si critique) - Utilise des exigences mesurables (pas "haute qualité", mais "teneur en X ≥ 99%") Évaluation : Vérifie que le TDR ne contient pas de contradictions et que toutes les exigences sont vérifiables.

Accompagnement clients, rétention, upsell
Préparer un plan de rendez-vous avec un client clé
Rôle : Tu es un responsable clientèle. Tu prépares des rendez-vous après lesquels le client veut continuer à travailler avec nous. Tâche : Établis un plan de rendez-vous avec le client [nom de l'entreprise]. Contexte : - Type de rendez-vous : [régulier / QBR / sur un problème / sur une extension] - Historique : [depuis combien de temps on travaille ensemble, ce qui a été acheté, tickets support] - Participants côté client : [rôles] - Objectif du rendez-vous : [ce qu'on veut obtenir] Format : 1. Ordre du jour (timing par point) 2. Pour chaque point : objectif, questions clés, quelles informations préparer 3. Documents pour le rendez-vous (quels rapports/présentations apporter) 4. Objections anticipées et réponses 5. Prochaine étape (vers quoi on mène le rendez-vous) Contrainte : Rendez-vous de 60 minutes maximum. L'ordre du jour doit tenir dans ce temps. Évaluation : Vérifie que l'ordre du jour est orienté vers la valeur pour le client, pas seulement notre intérêt.
Rédiger un email proposant d'élargir la collaboration
Rôle : Tu es un account manager. Tu sais proposer plus de manière à ce que le client te remercie. Tâche : Rédige un email au client [nom] proposant d'élargir la collaboration. Contexte : - Ce que le client utilise déjà : [produit/service] - Résultats : [chiffres concrets, si disponibles] - Ce qu'on propose : [nouveau produit/module/volume] - Bénéfice pour le client : [économie/croissance/réduction des risques] Format : - Rappel des résultats actuels (1 paragraphe avec chiffres) - Observation/insight ("Nous avons remarqué que...") - Proposition (bref, avec le bénéfice) - Preuve sociale ("Le client X l'utilise déjà et obtient...") - Appel à l'action (prochaine étape concrète) Ton : partenarial, pas intrusif. Le client est déjà avec nous — pas besoin de "vendre". Évaluation : Lis du point de vue du client. Est-ce que ça ressemble à une préoccupation pour son business ou à une vente ?
Analyser la satisfaction du client (sur la base de données)
Rôle : Tu es un analyste de l'expérience client. Tu trouves des patterns dans les données clients. Tâche : Analyse les données du client [nom]. Données : [insérer : NPS, tickets support, utilisation des fonctionnalités, extensions/réductions, dates] Format : 1. Évaluation globale de la santé du client (🟢/🟡/🔴) 2. Tableau — Métrique | Actuel | Tendance | Évaluation 3. Signaux (2-3 observations qui peuvent indiquer un risque ou une opportunité) 4. Recommandations : - Quoi faire dans les 30 prochains jours - Quoi proposer (upsell/formation/rendez-vous) 5. Qui de l'équipe doit prendre en charge Évaluation : Vérifie que les conclusions sont basées sur les données, pas sur l'intuition.

⚡ Comment adapter à votre propre tâche

Chaque prompt ci-dessus est un modèle. Pour obtenir le meilleur résultat :

  • Remplacez tout ce qui est entre crochets [comme ceci] par vos données
  • Ajoutez du contexte — plus de détails sur votre entreprise/situation, plus la réponse sera précise
  • Utilisez le few-shot — joignez 1-2 exemples du résultat attendu
  • Demandez une reformulation — si le résultat n'est pas idéal, demandez "plus court", "ajouter des chiffres", "réécrire pour [audience]"
  • Vérifiez toujours — l'IA aide à préparer, mais la responsabilité du résultat reste sur vous
