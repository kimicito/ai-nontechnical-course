1.1 · Ingénierie prompte pour un manager

🎯 Sans code · ✅ Quiz interactif

🎯 Pourquoi c'est important pour l'entreprise

80 % du résultat de l'IA ne dépend pas du modèle, mais de la manière dont vous l'avez posé.

Le manager dépense en moyenne2-3 heures par jourpour : rédiger des lettres, analyser des documents, comparer des propositions, préparer des rapports. Une bonne invite réduit ce temps de 3 à 5 fois.

Exemple de sauvegarde :

La comparaison de 3 devis de fournisseurs prend 2 heures manuellement. Avec une bonne invite - 15 minutes. Économie:1 heure 45 minutes × 10 fois par mois = 17,5 heures.

📚 Ce que nous étudions

Formule pour une bonne invite : RCOFE

LettreSignificationQuestion
RRôleQui est responsable ? "Vous êtes un acheteur expérimenté"
CContexteCe qui se passe? "Choisir l'emballage des cookies"
OObjectifQue faut-il obtenir ? "Comparez 3 phrases"
FFormatSous quelle forme ? "Tableau : Critères | Fournisseur 1 | Fournisseur 2"
EEval (Auto-vérification)Vérifiez le résultat : "Vérifiez que toutes les colonnes sont remplies, pas de différences illogiques"

Comparaison : mauvaise ou bonne invite

❌ Mauvaise invite
"Analysez les propositions et dites-moi laquelle est la meilleure"

Résultat : une réponse générale sans précisions, il faudra redemander 3 fois

✅ Bonne invite (RCOFE)
Vous êtes un acheteur FMCG expérimenté avec 10 ans d'expérience.
Nous sélectionnons un fournisseur d'emballages en carton pour cookies.
Reçu 3 offres en pièces jointes.

Tâche : comparer les offres selon des critères et recommander la meilleure.

Critères :
1. Prix unitaire (frais de port compris)
2. Commande minimale
3. Délai de production
4. Conditions de paiement
5. Évaluation du fournisseur

Format : tableau comparatif + recommandation claire et justifiée.

Limitation : si la période est > 14 jours, marquez-la comme un risque.

Eval : vérifiez que toutes les colonnes du tableau sont remplies, qu'il n'y a pas de différences de prix illogiques et que tous les chiffres s'additionnent correctement.

🛠️ Comment postuler

Étape 1 : Définir le rôle

L'IA réagit mieux lorsqu'elle sait qui elle « fait semblant d'être ». The role sets the expertise and tone.

  • "Tu es avocat"- analyse juridique, formulation soignée
  • "Vous êtes un spécialiste du marketing"- création, métriques, CTA
  • "Vous êtes analyste financier"- chiffres, retour sur investissement, risques

Étape 2 : Donnez le contexte

Sans contexte, l'IA donne une réponse générale. Avec contexte - spécifique.

💡 Exemple

Sans contexte :« Comment réduire les coûts ? → conseils généraux

Avec contexte :"Nous sommes un fabricant de biscuits, 50 salariés, la logistique représente 30% du budget. Comment réduire les coûts ?" → étapes spécifiques

Étape 3 : Spécifiez le format

Le format permet de gagner du temps lors du reformatage.

FormatQuand utiliserExemple de demande
TableauComparaison, analyse"Résultat dans le tableau : Critère | Option 1 | Option 2"
ListePlanifier étape par étape"Donnez 5 étapes sous forme de liste numérotée"
JSONPour les programmeurs/automatisation"Renvoyer le résultat en JSON avec des champs..."
EmailLettre prête"Écrivez une lettre au fournisseur dans un style commercial"

Étape 4 : Ajoutez l'auto-vérification (Eval)

Eval — c'est une demande à l'IA de vérifier sa réponse avant de la sortir. Cela réduit les erreurs et les hallucinations de 30 à 50 %.

💡 Exemples d'Eval
Vérifiez que toutes les colonnes du tableau sont remplies.
Vérifiez qu'il n'y a pas de différences de coût illogiques.
Vérifiez que tous les chiffres s'additionnent correctement.
Si un élément n'est pas spécifié dans le document — écrivez « Non spécifié ».

Pourquoi le faire : L'IA "invente" parfois des données manquantes ou saute une ligne dans un tableau. Eval l'oblige à revérifier — comme si vous demandiez à un collègue de "revérifier".

👨‍💻 Comment déléguer à un développeur

Modèle de tâche pour le développeur

"Nous devons créer une bibliothèque d'invites pour le département [NOM].

Exigences :
1. Modèles pour les tâches typiques : [liste]
2. Système de validation : contrôle d'exhaustivité du RCOF
3. Stockage dans [Notion/Confluence/SharePoint]
4. Versioning : historique des modifications
5. Accès : [qui peut modifier]

Exemple d'invite (joindre) :
[insérer une bonne invite]

Date limite : [date]
Responsable : [nom]"

🔒 Quand le prompt ne suffit pas : contrat de comportement

La formule RCOFE fonctionne parfaitement pour les tâches ponctuelles. Mais si vous écrivez le même prompt pour la 3ème fois de suite — il est temps de formaliser un contrat de comportement.

Signal : « Je demande pour la 5ème fois à l'IA de vérifier les factures » → ce n'est pas une tâche ponctuelle, c'est un processus régulier.

6 éléments du contrat

ÉlémentQuestionExemple
1. ActivationQuand déclencher le processus ?« Rapport de ventes reçu par email »
2. EntréeQu'envoie-t-on en entrée ?« Excel avec colonnes Plan | Réel | Écart »
3. LimitesQue NE PAS faire ?« Ne fais pas de recommandations de licenciement. N'utilise pas de données externes »
4. ModeAnalyse / brouillon / final ?« Analyse uniquement — ne modifie pas le fichier source »
5. SortieFormat et où enregistrer ?« Tableau des 3 écarts + cause probable. Envoie sur Telegram »
6. StopOù attendre l'humain ?« Si écart > 30 % — arrête-toi, attends la décision du CFO »

Évolution : du RCOFE au contrat

🔄 Étape 1 — Prompt ponctuel (RCOFE)
Tu es analyste financier.
Entrée : rapport de ventes T2.
Tâche : trouve les 3 principaux écarts par rapport au plan.
Format : tableau.
Eval : vérifie que les chiffres sont corrects.

✅ Convient pour un seul rapport. Mais si le rapport arrive chaque vendredi — il faudra copier le prompt encore et encore.

🔄 Étape 2 — Contrat pour processus régulier
ACTIVATION : Fichier "Rapport_ventes_*.xlsx" reçu par email
ENTRÉE : Tableau avec colonnes Plan | Réel | Écart (%)
LIMITES :
  - Analyse uniquement les chiffres du fichier
  - NE fais pas de recommandations de licenciement
  - N'utilise PAS de données externes
MODE : Analyse uniquement (ne modifie pas le fichier)
SORTIE : Tableau avec 3 écarts + cause probable
STOP : Si écart > 30 % → arrête-toi, attends le CFO

✅ Écrit une fois — fonctionne chaque semaine. Peut être confié au développeur pour être transformé en skill.

💡 Règle de décision

1-2 fois → RCOFE (prompt ponctuel)
3+ fois → Contrat (processus régulier)
Chaque semaine → Skill (automatisation via développeur)

✅ Checklist d'acceptation

  • L'invite contient-elle les 5 éléments RCOFE ?
  • Le rôle est-il approprié à la tâche ? (pas « vous êtes une IA », mais « vous êtes un analyste »)
  • Le contexte est-il suffisamment précis ? (il y a des chiffres, des délais, des restrictions)
  • L'objectif est-il clair ? (une tâche, pas 5 à la fois)
  • Le format est-il pratique pour la copie vers Excel/Word/CRM ?
  • Y a-t-il des restrictions ? (ce qu'il faut exclure, limites, cas extrêmes)
  • L'avez-vous vérifié avec 3 exemples réels ? Est-ce stable ?

🎯 L'essentiel de la leçon

L'invite estSavoirs traditionnels pour l'IA. Utilisez la formuleRCOFE : Rôle, Contexte, Objectif, Format, Eval. Plus l'invite est précise, meilleur est le résultat. Une mauvaise invite vous fait perdre du temps en clarification. Une bonne invite permet d'économiser de l'argent2-3 heures par jour.
