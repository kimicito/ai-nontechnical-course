1.3 · Chaînes de requêtes : du simple au complexe

🎯 Sans code · ✅ Quiz interactif

🎯 Pourquoi c'est important pour l'entreprise

Tâches complexesne peut pas être résolu en une seule fois. Essayer de tout demander en même temps conduit à des réponses superficielles.

Exemple:L'analyse du marché des fournisseurs comprend : recherche → comparaison → évaluation des risques → recommandation. Chaque étape s’appuie sur la précédente.

Économie:Une chaîne de 4 invites = 30 minutes. Analyse manuelle = 4 heures.

📚 Ce que nous étudions

Qu'est-ce qu'une chaîne de requêtes

Chaîneest une séquence d'invites, où chacune suivante utilise le résultat de la précédente.

🔍Collection
→
📊 Structure
→
🔎 Analyse
→
✅Solution

Exemple : sélection du fournisseur d'emballage

Étape 1 : Collecte d'informations
Vous êtes un acheteur FMCG.
Retrouvez les 5 plus grands fabricants d'emballages en carton à Moscou.
Pour chacun : nom, site internet, numéro de téléphone, minimum de commande, délai de production.
Étape 2 : Structuration
Sur la base des résultats de l'étape précédente, créez un tableau :
| Fournisseur | Site Web | Téléphone | Min. commander | Date limite | Remarques |

Marquez les fournisseurs avec un délai > 14 jours.
Étape 3 : Analyse des risques
Analysez le tableau :
1. Quels risques voyez-vous ? (délais, salaire minimum, joueurs inconnus)
2. Qui est le leader en termes de rapport prix/durée ?
3. Que dois-je vérifier en plus ?
Étape 4 : Recommandation
Sur la base de l’analyse, recommandez 2 fournisseurs pour le premier contact.
Justifiez votre choix. Indiquez les questions à poser lors de l'appel.

🛠️ Comment postuler

Règle : une invite = une tâche

❌ Incorrect :"Trouver des fournisseurs, les comparer, évaluer les risques et recommander les meilleurs"

✅ Exactement :Divisé en 4 invites distinctes

Pourquoi : L’IA donne une réponse superficielle lorsqu’elle en demande trop à la fois.

Modèles de chaîne commerciale

Chaîne "Tender"

  1. Recueillir les exigences du fournisseur
  2. Faire une liste de critères d'évaluation
  3. Recueillir des propositions (ou trouver des fournisseurs)
  4. Comparer par critères
  5. Évaluer les risques
  6. Formuler une recommandation

Chaîne "Rapport"

  1. Collecter les données pour la période
  2. Calculer les indicateurs clés
  3. Identifier les tendances et les anomalies
  4. Comparer avec la période précédente
  5. Formuler des conclusions et des recommandations

Chaîne "Accord"

  1. Analyser les termes clés
  2. Trouver les risques pour chaque côté
  3. Comparez avec le modèle/les contrats précédents
  4. Formuler des commentaires
  5. Préparer une liste de questions pour la contrepartie

👨‍💻 Comment déléguer à un développeur

Modèle de tâche

« Nous devons automatiser la chaîne d'analyse [processus].

Chaîne actuelle (manuelle) :
1. [étape 1] → [étape 2] → [étape 3]

Exigences :
1. Sauvegarde des résultats intermédiaires entre les étapes
2. Possibilité de s'arrêter et de vérifier le résultat après chaque étape
3. Si le résultat d'une étape est incorrect, vous pouvez la répéter
4. Rapport final au format [préciser]
5. Notification par email une fois terminé

Intégrations : [1C / Excel / CRM / email]
Date limite : [date]"

✅ Checklist d'acceptation

  • Chaque étape a-t-elle une tâche d’entrée et de sortie claire ?
  • Est-il possible de vérifier le résultat de l’étape intermédiaire ?
  • Si l’étape 2 est incorrecte, est-il possible de redémarrer uniquement cette étape ?
  • Le résultat final est-il construit sur toutes les étapes précédentes ?
  • N’existe-t-il pas d’étapes pouvant être combinées sans perte de qualité ?
  • La chaîne couvre-t-elle l’intégralité de la tâche du début à la fin ?

🎯 L'essentiel de la leçon

Tâches complexesle briser en chaîne: collecte → structuration → analyse → recommandation. Vérifiez le résultat de chaque étape. Une tâche = une invite. Cela donne une qualité qui ne peut être obtenue avec une seule demande.
