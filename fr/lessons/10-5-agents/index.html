Niveau 10 — Relations Publiques

10.5 Scénarios d'agents pour les Relations Publiques

⏱ 12 min · suivi médiatique, communications de crise, ESG
En quoi un agent diffère-t-il d'un modèle de langage ?
Le LLM rédige du texte sur demande. L'agent travaille 24/7 : scanne des centaines de sources, classe, alerte, prépare des ébauches — sans la commande « vérifie ce qui s'écrit ». Plus — Niveau 4.

Scénario 1 : Agent de suivi médiatique (n8n + RSS/API)

Tâche : suivre les mentions de l'entreprise, concurrents, secteur ; analyser la tonalité ; former un digest.

[Cron : toutes les 2 heures]
↓
[Sondage parallèle des sources]
├─→ RSS médias (Les Echos, Le Monde, Reuters)
├─→ Canaux Telegram du secteur
├─→ Google News API
└─→ YouTube (mentions dans les vidéos)
↓
[LLM : filtrage par pertinence]
├─→ Mentions directes de l'entreprise
├─→ Mentions des concurrents
├─→ Changements réglementaires
└─→ Tendances du secteur
↓
[LLM : analyse de tonalité (positif / neutre / négatif)]
↓
[Formation du digest]
├─ Top-5 actualités
├─ Tonalité
└─ Recommandations de réaction
↓
[Email au directeur RP + post dans Slack interne]
Résultat : le RP reçoit un digest au lieu de consulter manuellement 50+ sources. Économie de 3–4 heures par jour.

Scénario 2 : Agent de communications de crise (Agent OpenClaw)

Tâche : lors de la détection d'une vague négative — évaluer rapidement l'ampleur, préparer des propositions de réponse, router vers approbation.

[Déclencheur : >10 mentions négatives par heure OU mention dans un média national]
↓
[Agent : collecte d'urgence]
├─→ Où se produit l'explosion (plateformes, géographie)
├─→ Messages clés / mèmes
├─→ Influenceurs amplifiant le sujet
└─→ Lien avec des événements réels
↓
[LLM on-prem : analyse du risque]
├─→ Niveau de menace (faible / moyen / critique)
├─→ Scénarios d'évolution
└─→ Recommandations de réponse (silence / démenti / excuses / action)
↓
[Ébauches de réponses]
├─ Pour les médias
├─ Pour les réseaux sociaux
└─ Note interne pour les employés
↓
[Notification au directeur RP : « Crise : niveau X, propositions jointes »]

Scénario 3 : Agent de reporting ESG (hybride)

Tâche : collecter les données de différents systèmes, former l'ébauche du rapport ESG selon les standards GRI/SASB.

[Demande : « Préparer le rapport ESG annuel »]
↓
[Agent : collecte de données depuis les systèmes internes]
├─→ ERP : consommation énergétique, émissions
├─→ RH : parité, formation
├─→ Achats : chaîne d'approvisionnement, localisation
└─→ Production : sécurité, déchets
↓
[LLM externe : analyse selon le standard GRI]
├─→ Quels indicateurs sont couverts
├─→ Quelles données manquent
└─→ Comparaison avec l'année précédente
↓
[LLM on-prem : rédaction du texte du rapport]
↓
[Ébauche PDF + liste des points à compléter]
Important : les données internes (salaires, blessures, émissions) — on-prem. L'analyse selon le standard GRI peut se faire en IA externe sur des données anonymisées.
