8.1 Réalité : ce que l'IA peut faire dans la construction aujourd'hui

15 min · pour estimateurs, chefs de chantier, ingénieurs · construction de capitaux

La thèse principale de cette leçon : un devis n'est pas du texte. Les LLM ne calculent pas selon la GESN/FER, ne connaissent pas les coefficients et ne comprennent pas la méthode des ressources. Si vous donnez à un réseau de neurones un devis sous forme de texte et que vous attendez un calcul précis — vous êtes condamné aux hallucinations.

Mais cela ne signifie pas que l'IA est inutile dans la construction. Elle est utile — mais pour les tâches où un calcul normatif précis n'est pas nécessaire. Décomposons où elle fonctionne maintenant et où elle ne fonctionne pas encore.

Pourquoi la construction est plus difficile que les achats

Dans les achats, il y a des chiffres concrets : prix, délai de livraison, quantité. Un LLM compare 5 devis — et le fait bien. Dans la construction, tout est différent :

Ce qui rend la construction difficile pour les LLM :
  • Bases de données normatives : GESN, FER, TSN, FSSC — des dizaines de milliers d'éléments avec coefficients
  • Méthode des ressources : coût = norme × prix de la ressource × coefficients. Les LLM ne connaissent pas les prix actuels des ressources
  • Coefficients : Kp (profit), Knr (frais généraux), Kpz (chargement), indices de conversion — ils varient selon la région et l'année
  • Devis locaux : chaque projet est unique, aucun devis n'est identique à un autre
  • Normes de devis ≠ prix du marché : la GESN fournit les normes de main-d'œuvre et de machinerie, mais les prix des matériaux viennent du marché
Exemple d'erreur : Vous téléchargez un devis PDF sur ChatGPT et demandez "vérifiez le calcul". Le LLM voit du texte, mais pas ce qui se cache derrière les tableaux : quelles normes ont été appliquées, quels coefficients, quels indices. Il peut dire "tout est correct" ou trouver une erreur arithmétique évidente — mais manquer une erreur de devis de 30 %.

Trois catégories de tâches

Divisons les tâches du service construction en trois groupes — selon la préparation de l'IA à les résoudre.

🟢 Catégorie 1 : Victoires rapides — Maintenant

Les LLM gèrent cela sans personnalisation. ROI en jours et semaines.

  • Recherche dans la base de données normative : "Dans quelle collection GESN puis-je trouver la construction de pieux forés ?"
  • Décodage des éléments de devis : "Que signifie l'élément FERm 01-01-001-01 ?"
  • Conversion VOR en texte : "Sur la base du DCE, rédigez un plan de méthode"
  • Génération de documents textuels : notes explicatives, plans de méthode, lettres
  • Comparaison de devis locaux par texte : "Devis A et Devis B — quelle est la différence d'approche ?"
  • Vérification logique (pas de calcul !) : "Le devis de fondation comprend des travaux de toiture — est-ce une erreur ?"
  • Extraction de données des dessins (OCR) : reconnaître les dimensions, repères, quantités

🟡 Catégorie 2 : Nécessite un développement — Mois

Nécessite l'intégration avec un logiciel de devis, des bases de données normatives ou une logique personnalisée.

  • Calcul automatique de devis à partir du VOR : lier les volumes de travaux à la GESN/FER, appliquer les coefficients
  • Validation de devis basée sur les normes : vérification automatique : les taux sont-ils correctement sélectionnés, des travaux manquent-ils
  • Prévision de coûts : modèle entraîné sur des données historiques prédit le devis final à partir de l'ébauche du VOR
  • Correspondance dessin-devis : vérifier que tous les éléments du dessin sont pris en compte dans le devis
  • Extraction de VOR à partir de modèles BIM : quantification automatique à partir de modèles 3D

🔴 Catégorie 3 : Nécessite des agents — Années

Nécessite un système multi-agents avec accès aux données externes et prise de décision autonome.

  • Automatisation complète du devis : du dessin au devis approuvé sans intervention humaine
  • Surveillance automatique des prix des ressources : un agent surveille le marché et met à jour le devis en temps réel
  • Prévision des risques de construction : un agent analyse la météo, les fournisseurs, les sous-traitants et avertit des retards
  • Gestion automatique des modifications : quand le projet change, un agent recalcule tout : devis, planning, ressources

Où faire un pilote : Catégorie 1

Ne commencez pas par le calcul automatique de devis — c'est la Catégorie 2, cela nécessite un développement. Commencez par des tâches qui produisent des résultats en une semaine.

Étape 1 : Recherche dans la base de données normative

Exemple de prompt
Je dois trouver une norme pour la construction d'une fondation à semelle filante en béton armé monolithique.

Questions :
1. Dans quelle collection GESN ou FER dois-je chercher ?
2. Quels taux sont adaptés pour une fondation de 1,5 m de profondeur et 0,4 m de large ?
3. Quelles ressources sont incluses dans la norme (béton, armatures, coffrage, main-d'œuvre) ?
4. Quels coefficients sont généralement appliqués à cette norme ?

Le LLM ne donnera pas un calcul exact, mais il vous orientera dans la bonne direction — ce qui fera gagner 30 minutes de recherche dans les collections.

Étape 2 : Décodage des éléments de devis

Exemple de prompt
Voici une liste d'éléments d'un devis local :
- FERm 08-02-001-01
- GESNm 81-05-001-01
- TSN-2001-08-02-001

Décodez chaque élément :
1. Nom complet des travaux
2. Unité de mesure
3. Ressources principales (matériaux, main-d'œuvre, machinerie)
4. Domaine d'application typique

Étape 3 : Vérification logique d'un devis

Exemple de prompt
J'ai téléchargé un PDF d'un devis local pour des travaux de réparation capitale.

Vérifiez la logique (NE PAS calculer les sommes) :
1. Toutes les sections du devis correspondent-elles aux types de travaux dans la note explicative ?
2. Y a-t-il des travaux qui ne devraient pas figurer dans ce type de réparation ?
3. Y a-t-il des travaux manquants typiques pour une réparation capitale ?
4. La séquence des travaux correspond-elle à la technologie ?

Fournissez une liste des éléments suspects avec explications.

Étape 4 : Génération de notes explicatives

Exemple de prompt
Sur la base du devis local (PDF joint), rédigez une note explicative :

1. Dispositions générales (objet, client, entrepreneur)
2. Base d'élaboration du devis
3. Liste des taux et normes appliqués
4. Particularités d'application des coefficients
5. Coût total et durée des travaux

Format : officiel, selon la GOST R 21.1101-2013.

Ce qu'il NE FAUT PAS faire avec les LLM dans la construction

Signaux d'alerte — Tâches où les LLM vont vous induire en erreur :
  • Calculer un devis "à partir de zéro" à partir de texte : les LLM ne connaissent pas les prix actuels des ressources, les indices et les coefficients
  • Vérifier l'arithmétique dans les devis complexes : peuvent manquer une erreur dans un coefficient mais "confirmer" le total
  • Sélectionner des normes par description des travaux : le LLM peut suggérer un taux similaire mais incorrect
  • Utiliser pour la justification de devis devant un tribunal ou une expertise : les conclusions du LLM n'ont pas de force juridique
Règle de sécurité : Un LLM est un assistant de l'estimateur, pas un remplaçant. Il accélère les tâches routinières (recherche, texte, vérification logique), mais le calcul précis reste l'affaire des humains et des logiciels de devis (GRAND-Estimate, A0, Smeta.ru).

Vérification des connaissances

Question 1 : Pourquoi un LLM ne peut-il pas calculer précisément un devis selon la GESN/FER ?

Parce que la GESN interdit l'utilisation de l'IA
Parce que les LLM ne connaissent pas les prix actuels des ressources, les coefficients et les indices
Parce que les devis sont trop longs pour la fenêtre de contexte

Catégorie 1 — Victoires rapides
Catégorie 2 — Nécessite un développement
Catégorie 3 — Nécessite des agents
