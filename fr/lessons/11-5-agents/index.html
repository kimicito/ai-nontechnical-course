11.5 Scénarios d'agents pour la Qualité

⏱ 12 min · agents pour le contrôle qualité et les réclamations
En quoi un agent diffère-t-il d'un modèle de langage ?
Le LLM analyse sur demande. L'agent travaille en continu : reçoit des données, classe, route, suit les délais — sans participation humaine. Plus — Niveau 4.

Scénario 1 : Agent de traitement des réclamations (n8n + LLM)

Tâche : lors de la réception d'une réclamation — classifier, assigner un responsable, suivre les délais.

[Email / SÉD : nouvelle réclamation client]
↓
[n8n : extraction des pièces jointes et du texte]
↓
[LLM : classification]
├─→ Type : qualité produit / logistique / documents / prix
├─→ Criticité : critique / sérieuse / standard
└─→ Montant potentiel du préjudice
↓
[Assignation du responsable selon le répertoire]
↓
[Création de la tâche dans SÉD avec deadline]
├─ Critique : 24 heures
├─ Sérieuse : 72 heures
└─ Standard : 5 jours
↓
[Notification au responsable + au manager en cas de criticité]
↓
[Surveillance quotidienne : dépassées → escalade]
Résultat : le temps de réaction à une réclamation passe de 1–2 jours à quelques heures. Aucune réclamation n'est perdue.

Scénario 2 : Agent d'analyse des causes racines (Agent OpenClaw)

Tâche : lors de la réception de données sur un défaut — analyser automatiquement les causes, construire un diagramme d'Ishikawa, proposer des solutions.

[Entrée : description du défaut + données (photo, paramètres de processus)]
↓
[Agent on-prem : collecte des données historiques]
├─→ Défauts analogues sur la période
├─→ Changements dans le processus (matière, équipement, personnel)
└─→ Corrélations (température, pression, vitesse)
↓
[LLM : construction du diagramme d'Ishikawa]
├─→ 6M : Homme, Machine, Matière, Méthode, Mesure, Environnement
├─→ Causes racines probables avec justification
└─→ Références à des cas similaires
↓
[Proposition d'actions correctrices]
├─ Court terme (élimination des conséquences)
└─ Long terme (prévention de la répétition)
↓
[Rapport à l'ingénieur qualité]

Scénario 3 : Agent de préparation des rapports 8D (hybride)

Tâche : collecter les données pour toutes les étapes 8D, former le rapport, suivre l'exécution des actions.

[Déclencheur : création d'une fiche 8D dans le système]
↓
[Agent : collecte de données par étape]
D1 : Équipe → liste des responsables
D2 : Description du problème → extraction de la demande
D3 : Mesures temporaires → vérification de l'exécution
D4 : Cause racine → depuis l'agent root-cause
D5–D7 : Actions correctrices → suivi
D8 : Félicitations à l'équipe → formation
↓
[LLM : rédaction du texte du rapport]
↓
[PDF + notification à l'équipe]
↓
[Surveillance : D5–D7 non exécutés dans les délais → rappel]
