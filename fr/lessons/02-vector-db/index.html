2.2 · Bases de données vectorielles : comment l'IA « comprend » le sens

🎯 Sans code · ✅ Quiz interactif

🎯 Pourquoi c'est important pour l'entreprise

Une recherche régulière recherchemots exacts. Si vous recherchez « pénalité », mais que le contrat indique « amende pour retard de paiement », une recherche régulière ne la trouvera pas.

Recherche de vecteursrecherches parsignification. Il comprend que « forfait », « amende », « pénalité » sont à peu près la même chose.

Exemple:Un employé demande « Comment punir un fournisseur pour son retard ? » - La recherche vectorielle trouvera une section sur les sanctions, même si le document ne contient pas les mots « punir » ou « en retard ».

📚 Ce que nous étudions

Recherche régulière vs recherche vectorielle

❌ Recherche régulière

  • Recherche des mots exacts
  • "Forfait" ≠ "amende"
  • Les synonymes de Sauter
  • Ne comprend pas le contexte
  • Nécessite une formulation précise

✅ Recherche de vecteurs

  • Recherches de sens
  • "Forfait" = "amende" = "pénalité"
  • Trouve des synonymes
  • Comprend le contexte
  • Fonctionne avec le langage naturel

Comment ça marche (sans mathématiques)

Analogie:Imaginez que chaque mot est un point dans l'espace.

  • « Roi » et « reine » sont des points proches (parents)
  • "King" et "car" sont des points éloignés (pas de connexion)
  • « Forfait » et « amende » sont des points proches (synonymes)

L'IA transforme votre question en un point et recherche les fragments de document les plus proches.

❓ Question fréquente : faut-il écrire « utilise la recherche vectorielle » dans le prompt ?

Non, cela n'aidera pas. Lorsque vous écrivez dans ChatGPT, Claude ou Kimi, la recherche vectorielle a déjà eu lieu (ou n'a pas eu lieu) avant que vous appuyiez sur Entrée. L'IA n'a pas de levier « activer la recherche vectorielle » sur commande dans le texte.

Comment cela fonctionne réellement
Vous faitesCe qui se passe
Vous écrivez « utilise la recherche vectorielle »L'IA ignore comme instruction sans sens — elle n'a pas ce commutateur
Vous téléchargez un PDF et posez une questionAutomatiquement, le système recherche les fragments pertinents via la recherche vectorielle
Vous écrivez « cherche sur internet »C'est une recherche web — un autre mécanisme, pas la recherche vectorielle par documents

✅ Comment un non-spécialiste IT peut réellement utiliser la recherche vectorielle

Variante 1 : Dans un chat cloud (ChatGPT, Claude, Kimi + téléchargement de documents)

Chargez simplement des documents et posez des questions spécifiques. Le système automatiquement utilise la recherche vectorielle pour trouver les fragments pertinents. Votre prompt doit être spécifique — pas « active la recherche », mais « indique la page et cite le texte ».

Prompt utile pour travailler avec un document
Rôle : Tu es un avocat spécialisé en contrats de fourniture.
Tâche : Vérifie ce contrat sur les clauses de pénalité.
Format : Tableau — clause, risque, recommandation.

Important : indique le numéro de page et cite le texte mot pour mot.

La recherche vectorielle fonctionne ici automatiquement — le système trouve les fragments sur « pénalité », « amendes », « pénalités » et les transmet à l'IA.

Variante 2 : Dans un système corporate (OpenWebUI / GENAI / agent)

Lorsque vous déléguez une tâche au service informatique — utilisez le terme correctement. C'est un cahier des charges, pas une commande dans un chat :

Formulation correcte pour une tâche informatique
« Il faut que l'agent, lorsqu'il répond aux questions des employés,
cherche d'abord les documents pertinents via la recherche vectorielle,
puis formule une réponse sur leur base. Ne pas générer de tête. »

L'informatique comprendra : il faut connecter RAG (Retrieval-Augmented Generation) — un système où la recherche vectorielle + la génération de réponse travaillent ensemble.

Variante 3 : Quand la recherche vectorielle N'EST PAS adaptée

Ne l'utilisez pas (et n'en parlez pas dans le prompt), lorsque :

  • Un fait précis est nécessaire — « Quel est le taux USD le 15 juillet 2026 ? » → Mieux vaut une API de banque centrale ou une recherche web.
  • Un calcul est nécessaire — « Calculez le NPV de ce projet » → Mieux vaut Excel + formules.
  • La question est générale — « Qu'est-ce que la recherche vectorielle ? » → L'IA le sait déjà des connaissances générales, les documents ne sont pas nécessaires.

👨‍💻 Comment déléguer à un développeur

Modèle de tâche

"Nous avons besoin d'une recherche de documents en entreprise avec une base de données vectorielles.

Exigences :
1. Recherche par signification (pas seulement des correspondances exactes)
2. Prise en charge des langues russe et anglaise
3. Evolutivité : de 1 000 à 100 000 documents
4. Intégration avec [système existant]
5. Temps de réponse< 2 secondes

Exemple de demande : [donner]
Résultat attendu : [donner]

Balle préféréebase de données : [Pinecone/Chroma/Weaviate - ou laisser le choix au développeur]
Délai : [date]"

✅ Checklist d'acceptation

  • La recherche trouve-t-elle des synonymes ? (« pénalité » = « amende »)
  • La recherche comprend-elle le contexte ? (les questions avec des mots différents donnent les mêmes résultats)
  • Le délai de réponse est-il acceptable ? (< 3 secondes)
  • Les résultats sont-ils pertinents ? (les 3 premières réponses sont utiles)
  • Le système évolue-t-il à mesure que les documents se développent ?
  • Est-il possible de filtrer par date/type de document ?

🎯 L'essentiel de la leçon

Recherche de vecteurspermet à l'IA de comprendre le sens plutôt que de rechercher des mots exacts. Ceci est essentiel pour travailler avec des documents : les employés posent des questions avec leurs propres mots et l'IA trouve les bonnes réponses. Le gestionnaire n'a pas besoin de sélectionner une base - c'est la tâche du développeur. Besoin de comprendrePour quoic'est nécessaire etcomment vérifier le résultat.
