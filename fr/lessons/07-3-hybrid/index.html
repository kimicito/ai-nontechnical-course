7.3 Tâches hybrides : Anonymisation des données

15 min · pour spécialistes des achats · frontière des contours

Certaines tâches se situent à la frontière : elles peuvent être réalisées dans le cloud, mais uniquement si les données sont anonymisées. Remplacez le nom du fournisseur par « Fournisseur A », supprimez les prix spécifiques — et la tâche devient sûre pour un modèle externe.

Dans cette leçon : comment anonymiser les données, quelles tâches sont autorisées en mode hybride, et comment préparer une demande pour les développeurs pour l'automatisation.

Règle : Anonymiser avant le cloud

Ce qui NE DOIT PAS être envoyé dans le cloud (même partiellement) :
  • Prix spécifiques des devis
  • Noms de fournisseurs (sauf s'il s'agit de géants comme Siemens)
  • Volumes d'achat pour des articles spécifiques
  • Conditions contractuelles (pénalités, remises, délais de paiement)
  • Position de négociation, BATNA, faiblesses du fournisseur
Ce qui PEUT être envoyé après anonymisation :
  • Comparaison de la structure des devis (sans prix) — « Fournisseur A vs Fournisseur B »
  • Analyse de l'exhaustivité de la réponse au CDC (sans noms)
  • Répétition de négociation en jeu de rôle (scénario anonymisé)
  • Préparation d'arguments généraux (sans chiffres spécifiques)

Checklist d'anonymisation (5 étapes)

  1. Remplacer les noms : « LLC PromTechSnab » → « Fournisseur A »
  2. Remplacer les prix par des prix relatifs : « 1 200 000 ₽ » → « prix de base +20 % » ou « Prix X »
  3. Supprimer les volumes : « 50 unités » → « lot de taille moyenne »
  4. Supprimer les dates de contrat : « jusqu'au 31.12.2025 » → « contrat jusqu'à la fin de l'année »
  5. Vérifier : si vous envoyez le texte anonymisé à un collègue — comprendra-t-il de quoi il s'agit, sans connaître le fournisseur ? Si oui — il peut aller dans le cloud.

1. Répétition de négociation en jeu de rôle

Un spécialiste des achats se prépare pour des négociations complexes. Il a besoin de répéter un scénario. Les données peuvent être anonymisées et un modèle cloud peut être utilisé comme « adversaire. »

Exemple de prompt (anonymisé)
Effectuez une répétition de négociation en jeu de rôle avec moi.

Scénario (anonymisé) :
- Je suis un spécialiste des achats dans une entreprise industrielle
- Je dois réduire le prix pour la fourniture d'équipement
- J'ai 2 fournisseurs alternatifs avec des prix inférieurs de 15 %
- Le fournisseur actuel (Fournisseur A) a retardé la livraison la dernière fois
- J'ai besoin d'un délai de 3 semaines, leur standard est de 6 semaines
- Je suis prêt à discuter du volume, mais pas à divulguer de chiffres spécifiques

Vous jouez le rôle du Fournisseur A. Votre objectif est de maintenir le prix et les délais.
Mon objectif est de réduire le prix et d'accélérer les délais.

Commencez les négociations. Répondez comme un vrai fournisseur — 
avec des objections, des contre-arguments, des tentatives d'évitement.
Résultat : 15 à 20 minutes de répétition avec un adversaire « vivant. » Le spécialiste des achats pratique les arguments, apprend à dévier les objections et se prépare pour la vraie conversation.

2. Comparaison de la structure des devis (sans prix)

Vous devez comprendre quel fournisseur a répondu plus complètement au CDC, pas qui est moins cher. Les prix sont supprimés, seule la structure et l'exhaustivité sont comparées.

Exemple de prompt (anonymisé)
Comparez 3 devis commerciaux par structure et exhaustivité de réponse.

Données anonymisées :
- Fournisseur A, Fournisseur B, Fournisseur C
- Prix supprimés, seulement relatifs : « sous le marché », « marché », « au-dessus du marché »

Comparez selon les critères :
1. Exhaustivité de la réponse au CDC (tous les points sont-ils couverts)
2. Présence de descriptions techniques (oui/non, qualité)
3. Présence de certificats (liste)
4. Conditions de garantie (durée, portée)
5. Services supplémentaires (installation, formation, service)

Ne citez pas de prix spécifiques. Donnez une recommandation sur l'exhaustivité.

3. Préparation d'arguments généraux

Un spécialiste des achats a besoin d'une liste d'arguments pour les négociations en termes généraux — non liés à une affaire spécifique. Un modèle cloud génère la base, puis le système local ajoute les spécificités.

Exemple de prompt (anonymisé)
Je me prépare pour des négociations sur l'achat d'équipement industriel.

Contexte (anonymisé) :
- Marché : 3 à 4 grands fournisseurs, forte concurrence
- Ma position : grand client, achats réguliers
- Problème : le fournisseur veut augmenter le prix et les délais

Préparez des arguments généraux pour les négociations :
1. Arguments pour la réduction de prix (marché, volumes, concurrence)
2. Arguments pour l'accélération des délais (planification, réserves, pénalités)
3. Contre-arguments aux objections typiques des fournisseurs
4. Formulations polies pour une position ferme

Sans prix, noms ou volumes spécifiques. 
Uniquement des modèles que j'adapterai ensuite.

Demande aux développeurs pour l'automatisation de l'anonymisation

Si les spécialistes des achats anonymisent souvent des documents manuellement — cela peut être automatisé. Voici un modèle de demande pour l'équipe informatique.

Modèle de demande pour les développeurs
Tâche : créer un script/application pour anonymiser 
des documents d'achat avant envoi aux LLM cloud.

Entrée : fichier (PDF, Word, Excel) avec un devis commercial.
Sortie : le même fichier avec des données remplacées.

Règles de remplacement :
1. Noms d'entreprises → « Fournisseur A », « Fournisseur B », etc.
2. Montants → relatifs (« prix de base », « +15 % », « -20 % »)
3. Volumes → « petit/moyen/gros lot »
4. Dates → relatives (« au cours du trimestre en cours », « jusqu'à la fin de l'année »)
5. Coordonnées de contact → supprimer complètement
6. Numéros de contrat → « Contrat N »

Supplément : vérifier — que le fichier de sortie 
ne contient aucune donnée originale (recherche par mots-clés).

Intégration : glisser-déposer le fichier → recevoir la version anonymisée 
en 5 secondes. Sauvegarder l'historique d'anonymisation.

Priorité : élevée (utilisée quotidiennement par 5 à 10 spécialistes des achats).
💡 Astuce : même si l'informatique ne peut pas construire une application complète, vous pouvez commencer par un simple script Python. Il remplacera 80 % du travail manuel. Puis affinez-le.

Vérification des connaissances

Question 1 : Lequel des éléments suivants NE DOIT PAS être envoyé dans le cloud même après anonymisation ?

Nom du fournisseur → « Fournisseur A »
Clauses de pénalité spécifiques d'un contrat actif
« Prix 10 % au-dessus du marché »

Question 2 : Quelle est la première étape de la checklist d'anonymisation ?

Supprimer toutes les dates
Remplacer les noms d'entreprises par « Fournisseur A », « Fournisseur B »
Supprimer tous les chiffres du document

Question 3 : Comment vérifiez-vous si un document est suffisamment anonymisé ?

L'envoyer dans le cloud et voir si le système le bloque
Le montrer à un collègue — il comprend l'essentiel, mais ne connaît pas le fournisseur ?
Supprimer tous les chiffres et noms — cela suffit

Question 4 : Laquelle des suivantes est une tâche hybride (anonymisation + cloud) ?

Vérifier un contrat pour des écarts par rapport à un modèle
Répétition de négociation en jeu de rôle avec un scénario anonymisé
Rechercher de nouveaux fournisseurs sur internet

Question 5 : Que devez-vous faire si les spécialistes des achats anonymisent constamment des documents manuellement ?

Préparer une demande aux développeurs pour l'automatisation par script
Continuer manuellement — c'est plus fiable
Envoyer les originaux dans le cloud — c'est plus rapide
