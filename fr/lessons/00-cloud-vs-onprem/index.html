0.3 Cloud vs On-premise : GENAi pour les données sensibles

🎯 Pas de code · ✅ Quiz à la fin

🎯 Pourquoi c'est important pour l'entreprise

60% des entreprisesn’utilisez pas l’IA par crainte de fuite de données. Mais entre « envoyer tout à ChatGPT » et « ne pas utiliser l'IA », il existe une troisième voie :hébergement sur place.

⚠️ Cas réel :Le responsable des achats a téléchargé une liste de fournisseurs avec des prix sur ChatGPT. Au bout de 3 mois, le concurrent connaissait sa stratégie tarifaire. Données divulguées lors de la formation du modèle.

📚 Cloud vs On-premise : en termes simples

☁️ Nuage

Qu'est-ce que c'est:Vous envoyez des données aux serveurs d'une autre entreprise (OpenAI, Anthropic, Google).

✅ Pas cher, rapide, aucun service informatique nécessaire

❌ Les données dépassent le périmètre de l’entreprise

🏢 Sur site (propre serveur)

Qu'est-ce que c'est:Le modèle d'IA fonctionne sur un serveur au sein de l'entreprise ; les données ne quittent pas le bureau.

✅ Contrôle total des données, confidentialité

❌ Plus cher, vous avez besoin d'un serveur et d'un administrateur

Analogie pour les négociations

💡 ANALOGIE

Cloud = location de bureaux dans un centre d'affaires.C'est pratique, mais le concierge voit tous vos visiteurs et les voisins entendent à travers le mur.

Sur site = votre maison.Vous êtes propriétaire, mais vous réparez vous-même la robinetterie et payez les réparations du toit.

🛠️ GENAI : juste milieu

GENAi est une Plateforme IA d'entreprise (comme ChatGPT, mais à l'intérieur de l'Entreprise). Les employés voient le chat habituel, mais les données restent au sein de l'entreprise. Vous trouverez toujours le bouton « GENAi » sur la page principale de Rusal Online.

Quand choisir GENAI :

  • Travailler avec des secrets commerciaux (prix, fournisseurs, contrats)
  • Données personnelles des employés ou des clients
  • États financiers avant publication
  • Plans et prévisions stratégiques
  • Obligation réglementaire de stocker les données dans le pays (RF, RPC, UE)

Matrice des risques

🔴 ÉLEVÉ

Contrats, prix, M&A

🟡 MOYEN

Rapports internes, RH

🟢 FAIBLE

Marketing, données publiques

👨‍💻 Comment attribuer une tâche à un développeur

Modèle pour le directeur informatique : 
"Nous devons déployer un LLM local pour le département [NOM].

Exigences :
- Nombre d'utilisateurs : [N] personnes
- Documents : jusqu'à [N] pages à la fois
- Confidentialité : CRITIQUE (secret commercial)
- Langues : RU, EN
- Budget matériel : [montant]
- Durée : [N] jours

S'il vous plaît suggérer :
1. Option modèle (Qwen / Lama / Mistral)
2. Configuration du serveur
3. Coûts de licence
4. Plan de sauvegarde
5. Instructions pour les utilisateurs"

✅ Checklist d'acceptation

  • Les données ne quittent-elles pas le serveur de l'entreprise ? (vérifier les paramètres de sécurité (pare-feu))
  • Y a-t-il une sauvegarde ? (et si le serveur brûle)
  • Les utilisateurs peuvent-ils travailler sans un informaticien ? (l'interface est claire)
  • Vitesse de réponse< 10 secondes? (vérifier sous charge maximale)
  • Existe-t-il un processus de mise à jour du modèle ? (de nouvelles versions sont publiées chaque mois)

🧰 Outils

  • GENAi — Plateforme IA d'entreprise (comme ChatGPT, mais à l'intérieur de l'Entreprise). Les employés voient un chat familier, mais les données restent à l'intérieur de l'entreprise. Vous trouverez toujours le bouton « GENAi » sur la page principale de Rusal Online.

🎯 L'essentiel de la leçon

Pour les données publiques - modèles cloud (ChatGPT, Claude). Pour les confidentiels - sur site (GENAi + Qwen/Llama). Il n’existe pas de solution universelle : choisissez par classe de données. Et ne téléchargez jamais de secrets commerciaux sur des chatbots publics.
