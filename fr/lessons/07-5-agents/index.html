7.5 Scénarios d'agents pour les Achats

⏱ 12 min · comment les agents automatisent les processus d'achat
En quoi un agent diffère-t-il d'un modèle de langage ?
Le LLM répond à des requêtes individuelles. L'agent travaille de manière cyclique : il se fixe des tâches, appelle des outils, obtient des résultats et décide quoi faire ensuite. Un agent peut visiter un site web, télécharger un fichier, envoyer un email — sans intervention humaine à chaque étape.

Plus sur les agents — Niveau 4 : Agents et orchestration.

Scénario 1 : Agent de monitoring des appels d'offres (n8n + LLM)

Tâche : vérifier quotidiennement les marchés publics et appels d'offres commerciaux, filtrer par mots-clés, envoyer un bref digest à l'acheteur.

Architecture de l'agent

[n8n Cron : quotidien à 9h00]
↓
[Requête HTTP : plateforme marchés publics / API client]
↓
[LLM Node : filtrage par mots-clés, évaluation pertinence]
↓
[Telegram / Email : digest « 5 appels d'offres aujourd'hui »]
↓
[Google Sheets : enregistrement pour analyse]

Ce que fait l'agent

  • Télécharge XML/JSON de la plateforme de marchés publics
  • Le LLM lit les descriptions de lots, compare avec le profil de l'acheteur
  • Évalue la pertinence sur une échelle 1–10
  • Envoie uniquement les appels d'offres avec score ≥7
  • Enregistre dans un tableau pour statistiques d'efficacité
Résultat : l'acheteur reçoit un digest prêt au lieu de 2 heures de consultation manuelle des plateformes. Économie de 10 heures par semaine.

Scénario 2 : Agent d'analyse des fournisseurs (style OpenClaw)

Tâche : à la réception d'une demande pour un nouveau fournisseur — collecter automatiquement les informations publiques et préparer un profil de due diligence.

Architecture de l'agent

[Demande dans le système : nom de l'entreprise, SIRET]
↓
[Agent OpenClaw : appels parallèles]
├─→ [Navigateur : site de l'entreprise → extraire activité, contacts]
├─→ [API : registre des entreprises → vérifier statut, fondateurs]
├─→ [API : bureau de crédit → indicateurs financiers]
└─→ [Recherche : actualités, litiges]
↓
[LLM On-prem : synthèse en profil unifié]
↓
[Rapport PDF dans dossier + notification à l'acheteur]

Sécurité du contour

  • Collecte publique de données — contour externe (navigateur, API)
  • Synthèse du profil, évaluation des risques — LLM on-prem
  • Décision d'admission — humain (human-in-the-loop)
Important : le fait même de vérifier un fournisseur et ses résultats est une information interne. Le profil est préparé en on-prem, même si les données sont collectées depuis des sources publiques.

Scénario 3 : Agent de comparaison de propositions commerciales (hybride)

Tâche : recevoir les propositions de 3+ fournisseurs, structurer, comparer, identifier les risques cachés, préparer une recommandation.

Architecture de l'agent

[Intégration Email : nouveau mail avec sujet « Proposition »]
↓
[Agent : extraction des pièces jointes (PDF, Excel, Word)]
↓
[LLM On-prem : extraction des paramètres des propositions]
├─ Prix, délais, conditions de paiement
├─ Garanties, pénalités, force majeure
└─ Conditions cachées (reconduction automatique, arbitrage exclusif)
↓
[LLM : tableau comparatif + SWOT de chaque proposition]
↓
[Notification à l'acheteur : « Propositions prêtes pour comparaison »]

Caractéristique clé

L'agent travaille à l'intérieur du contour : reçoit les emails, traite les documents, n'envoie pas le contenu des propositions à l'extérieur. Le LLM analyse les conditions sans voir les noms des concurrents lors de la comparaison (si une confidentialité supplémentaire est requise).

Résultat : comparaison de 5 propositions en 15 minutes au lieu de 3–4 heures d'analyse manuelle. L'agent ne manque pas les petits caractères et les conditions cachées.
