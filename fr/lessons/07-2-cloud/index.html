7.2 Tâches externes : Cloud

15 min · pour les spécialistes des achats · données ouvertes

Dans cette leçon — des tâches où le spécialiste des achats travaille avec des informations publiques : recherche de fournisseurs, benchmarking des prix, préparation de documents anonymisés. Ces données ne contiennent pas de secrets commerciaux — elles peuvent être envoyées aux modèles cloud.

Solution : APIs cloud : Claude, ChatGPT, Kimi, API Qwen. La recherche web est nécessaire — le modèle doit donc être connecté à Internet.

Quand le Cloud l'emporte sur l'On-Prem

Les modèles cloud gagnent quand vous avez besoin de :
  • Recherche web : trouver de nouveaux fournisseurs, vérifier les prix du marché
  • Traduction : travailler avec des documents étrangers
  • Génération : créer des textes de modèle à partir de zéro
  • Analyse de données publiques : indices de matières premières, taux de change, codes douaniers
Important : avant d'envoyer vers le cloud — vérifiez que la demande ne contient pas de prix spécifiques, de noms de fournisseurs ou de volumes. Si c'est le cas — anonymisez-la ou envoyez-la vers l'on-prem.

1. Recherche d'analogues et de fournisseurs alternatifs

Le spécialiste des achats connaît 2 à 3 fournisseurs pour une position. Il faut en trouver davantage — pour la concurrence ou en secours. L'on-prem n'a pas accès à Internet, le cloud — oui.

Exemple de prompt (Claude avec recherche web)
Trouver des fabricants de pompes centrifuges industrielles 
 série 32-160 en Russie et dans les pays voisins (Kazakhstan, Biélorussie).

 Préciser :
 - Nom de l'entreprise et site web
 - Où se trouve la production (ville)
 - S'il existe des certificats (GOST, ISO)
 - Fourchette de prix approximative (si publiquement disponible)
 - Contacts de l'approvisionnement (service commercial)

 Faire un tableau comparatif.
Résultat : un tableau de fournisseurs potentiels avec leurs contacts. Le spécialiste des achats passe 10 minutes au lieu de 2 heures de recherche manuelle.

2. Benchmarking des prix du marché et des indices de matières premières

Le spécialiste des achats négocie un prix mais ne sait pas où se trouve le marché actuellement. Le LLM avec recherche web trouve les indices actuels.

Exemple de prompt
Quels sont les prix actuels du marché (2024-2025) pour :
 1. Armature en acier A500C (gros, par tonne)
 2. Ciment Portland M500 (gros, par tonne)
 3. Cathodes de cuivre (LME + prime)

 Préciser également :
 - La dynamique au cours des 6 derniers mois (augmentation/diminution en %)
 - Les prévisions pour le trimestre prochain (si disponibles)
 - Les principaux facteurs affectant le prix
Important : la demande ne contient pas de prix spécifiques de vos fournisseurs — seulement des benchmarks du marché. C'est sans danger pour le cloud.

3. Préparation d'un cahier des charges anonymisé (brouillon)

Le cahier des charges avec des chiffres spécifiques — va vers l'on-prem. Mais une structure brouillon du CdC, des exigences générales, des paramètres typiques — peut être faite dans le cloud, puis finalisée en interne.

Exemple de prompt
Préparer une structure pour un cahier des charges pour l'approvisionnement 
 en équipement de pompes industrielles pour la production chimique.

 Sans chiffres spécifiques (volumes, prix, délais). 
 Seulement les paramètres et exigences typiques.

 Structure :
 1. Exigences générales
 2. Paramètres techniques (puissance, capacité, matériaux)
 3. Exigences de qualité et de certification
 4. Conditions de livraison (standard)
 5. Exigences de documentation
 6. Critères d'acceptation

 Préciser quels éléments devront être complétés par des données spécifiques à l'intérieur de l'entreprise.

4. Documentation d'appel d'offres (sections générales)

Instructions pour les participants, critères d'évaluation, formulaire de candidature — tout cela est basé sur des modèles et ne contient pas de données sur une transaction spécifique.

Exemple de prompt
Créer un modèle de documentation d'appel d'offres pour l'approvisionnement 
 en équipement industriel par méthode de concours ouvert.

 Inclure :
 1. Instructions pour les participants (règles de soumission, délais)
 2. Formulaire de candidature (champs généraux)
 3. Critères d'évaluation (prix, délais, expérience) — sans coefficients de pondération
 4. Formulaire de contrat (conditions standard)
 5. Liste des documents pour la candidature

 Préciser quelles sections doivent être coordonnées avec l'avocat et le demandeur.

5. Réponses automatiques aux clients et aux plateformes

Le statut de l'expédition, les délais de livraison, les demandes de devis — souvent basés sur des modèles. Le LLM génère une réponse basée sur le statut ERP, mais sans chiffres spécifiques.

Exemple de prompt
Rédiger un modèle d'e-mail de réponse à une demande de statut d'expédition.

 Conditions :
 - La commande est en transit
 - Le fournisseur a signalé un retard
 - Il faut s'excuser, expliquer la raison, indiquer le nouveau délai
 - Proposer une compensation (standard)

 Sans dates spécifiques, montants ou noms de fournisseurs. 
 Destinataire : client interne au sein de l'entreprise.

6. Traduction et localisation de documents

Les fournisseurs étrangers envoient des instructions, des spécifications et des contrats en anglais, chinois et allemand. La traduction cloud est plus rapide et plus précise que les traducteurs standard pour les textes techniques.

Exemple de prompt
Traduire de l'anglais vers le russe une spécification technique 
 pour une pompe industrielle (fichier Spec.pdf).

 Exigences :
 - Préserver la structure et la numérotation des sections
 - Termes techniques — selon GOST/ISO
 - Unités de mesure — en SI (m³/h, bar, kW)
 - Préserver les tableaux au format markdown
 - Marquer les parties peu claires pour clarification avec le fournisseur

7. Vérification des exigences douanières (réglementations générales)

Code SH, exigences de certification, droits — ce sont des informations publiques. Le LLM trouve les exigences actuelles pour une catégorie de produits.

Exemple de prompt
Quelles sont les exigences douanières et les certificats nécessaires 
 pour l'importation de pompes centrifuges industrielles en Fédération de Russie (code SH 8413 70) ?

 Préciser :
 - Certification obligatoire (GOST-R, TR CU)
 - Si le SBKTS est nécessaire
 - Droit de douane (taux)
 - TVA à l'importation
 - Exigences pour les documents du fournisseur (facture, liste de colisage)
 - Spécificités pour certains pays (Chine, Allemagne, Italie)

 Sans cargaison spécifique ni montants — seulement des réglementations générales.

Vérification des connaissances

Question 1 : Pourquoi vaut-il mieux rechercher de nouveaux fournisseurs dans le cloud plutôt que sur l'on-prem ?

Parce que l'on-prem n'a pas accès à Internet pour la recherche
Parce que l'on-prem fonctionne plus lentement
Parce que le cloud est gratuit

Question 2 : Quel modèle ne devrait PAS être utilisé pour le benchmarking des prix ?

Claude avec recherche web
LLM on-prem sans Internet
ChatGPT avec navigation

Question 3 : Quelle requête est sûre pour le cloud ?

« Comparez les prix de mes 3 fournisseurs : A, B, C » (avec montants spécifiques)
« Quels sont les prix du marché pour l'armature A500C en 2025 ? »
« Vérifiez le contrat avec le fournisseur X par rapport au modèle »

Question 4 : Quelle tâche peut être réalisée dans le cloud si le CdC contient des volumes spécifiques ?

Envoyer le CdC complet avec volumes à ChatGPT
Créer une structure brouillon de CdC sans chiffres, puis ajouter les volumes en on-prem
On ne peut pas utiliser l'IA pour le CdC du tout

Question 5 : Quel modèle est le meilleur pour traduire des spécifications techniques ?

Claude ou GPT-4 — ils gèrent mieux les termes techniques
N'importe quel modèle gratuit — la traduction est une tâche simple
Seulement le modèle on-prem, car les spécifications peuvent contenir des prix
