Niveau 16 — Compliance

16.5 Scénarios agents pour le compliance

⏱ 20 min · agents pour la surveillance des sanctions, l'analyse AML, le traitement des signalements whistleblower
Agent vs LLM
Un LLM répond à une question isolée. Un agent effectue une séquence d'actions : surveille, analyse, alerte, crée des tâches. Pour les scénarios agents compliance, voir Niveau 4 — Qu'est-ce qu'un agent.

Scénario 1 : Agent de surveillance des sanctions (n8n/no-code)

Problème : les listes de sanctions changent quotidiennement, et les mises à jour manuelles prennent trop de temps.

Architecture de l'agent

Flux de données
1. Sources : listes publiques OFAC, UE, ONU, listes nationales
2. Déclencheur : mise à jour des listes (webhook/API)
3. Filtrage : correspondance avec la base de contreparties de l'entreprise
4. Analyse LLM : évaluation du risque (anonymisée)
5. Action : création d'une alerte dans le système compliance
6. Notification : alerte à l'officier compliance

Données sensibles : la liste des contreparties de l'entreprise — information interne. Le matching avec des noms spécifiques — on-prem.

Scénario 2 : Agent d'analyse AML (full AI agent)

Problème : le volume des transactions nécessite une analyse automatique des patterns suspects.

Architecture de l'agent

Flux de données
1. Déclencheur : nouvelle transaction dans le système
2. Analyse rule-based : vérification automatique des seuils
3. Analyse LLM : comparaison avec le profil du client
4. Évaluation : niveau de risque (faible/moyen/élevé)
5. Action : génération d'un rapport préliminaire SAR/STR
6. Notification : tâche pour l'analyste AML avec priorité

Données sensibles : toutes les données des transactions, les profils des clients — uniquement on-prem.

Scénario 3 : Agent de traitement des signalements whistleblower (hybride)

Problème : les signalements arrivent via différents canaux et nécessitent une analyse structurée rapide.

Architecture de l'agent

Flux de données
1. Déclencheur : nouveau signalement (formulaire web, email, téléphone)
2. Extraction : texte du signalement
3. Analyse LLM (on-prem) :
   - Résumé des allégations
   - Évaluation de la crédibilité
   - Classification du risque
4. Recherche interne (on-prem) :
   - Historique des incidents similaires
   - Politiques applicables
5. Action : création d'une tâche d'enquête avec priorité
6. Notification : alerte au responsable compliance

Sécurité : tout le traitement — on-prem. Les signalements contiennent des allégations sensibles sur des personnes spécifiques.

Vérification des connaissances

Question 1 : Pourquoi l'analyse AML — uniquement on-prem ?

Parce que les transactions et les profils clients sont confidentiels
On peut anonymiser les transactions pour le cloud
Parce que l'AML nécessite une vitesse maximale

Question 2 : L'agent de surveillance des sanctions — quel contour pour les listes ?

Cloud — les listes de sanctions sont publiques
On-prem — les listes peuvent contenir des données sensibles
Hybride — listes à l'extérieur, matching à l'intérieur
