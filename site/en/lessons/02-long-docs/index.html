2.3 · Working with long documents: limitations and solutions

🎯 No code · ✅ Interactive quiz

🎯 Why does business need this?

Contracts, reports, tender documentation - often hundreds of pages.

Problem: If the document is too long, the AI ​​“forgets” the beginning when it reads to the end.

Solution: Breakdown, RAG, special models with more context.

📚 What we are studying

3 ways to work with long documents

Method 1: Selecting a Model with More Context

Simple solution: use Kimi or Gemini for long documents.

💡 When to use
  • Document< 500 страниц
  • Need quick analysis without setup
  • No sensitive data (cloud services)

Method 2: Chunking

We divide the document into logical parts and analyze them one by one.

💡 Example of contract breakdown
Part 1: Preamble, subject of the agreement

Each part is analyzed separately, then the results are combined.

Method 3: RAG (Retrieval-Augmented Generation)

We use vector search: AI finds relevant fragments and analyzes only them.

💡 When to use
  • Archive of hundreds of documents
  • Need a search for meaning
  • Regular work with documents

🛠️ How to apply

Scenario: Analysis of tender documentation (300 pages)

Task: Analyze tender documentation for compliance with requirements.

Algorithm
  1. Upload document to Kimi or Gemini (supports large files)
  2. Set the prompt: “Analyze the tender documentation. Highlight the key requirements for the supplier, evaluation criteria, deadlines. Output in a table.”
  3. If the answer is cut off, clarify: “Continue the analysis of section [name]”

Scenario: Analysis of the archive of contracts (50 documents)

Task: Find all contracts with penalties above 1%.

Algorithm
  1. Upload all contracts into the RAG system
  2. Ask the question: “In which contracts does the penalty exceed 1% per day of delay?”
  3. Check answers (AI will indicate sources)

👨‍💻 How to delegate to a developer

Task template

“We need a system for working with long documents.

✅ Checklist for accepting the result

  • AI processes the document completely, without cropping?
  • Is context between sections preserved?
  • Are sources indicated (pages, sections)?
  • Can I ask a clarifying question about the document?
  • Is the processing time acceptable? < 1 минуты на 100 страниц)
  • Works with PDF, DOCX, TXT formats?
