6.3 · AI-политика компании: правила игры

🎯 Без кода · ✅ Интерактивный квиз

🎯 Зачем это бизнесу

Без правил использования AI компания рискует: утечкой данных, ошибочными решениями, нарушением законодательства. AI-политика — это не бюрократия, это страховка.

Пример: Компания без политики: сотрудник отправил клиентские данные в бесплатный ChatGPT → Утечка → Штраф 500 000 ₽ по 152-ФЗ.

Компания с политикой: сотрудник знал, что нельзя → Использовал локальное решение → Нет утечки.

Что должна содержать AI-политика

РазделСодержание
1. ЦельЗачем используем AI, какие задачи решаем
2. Разрешённые инструментыКакие сервисы можно использовать (ChatGPT Enterprise, локальные и т.д.)
3. Запрещённые данныеЧто нельзя отправлять в облачный AI
4. ПроцессыВ каких задачах AI может работать автономно, где нужен человек
5. ОтветственностьКто отвечает за результат, кто проверяет
6. ОбучениеКак часто обучать, кто проводит

Пример: категории данных по уровням доступа

🔴 Красные (строго локально):

  • Персональные данные клиентов
  • Коммерческая тайна
  • Договоры NDA
  • Финансовая отчётность

🟡 Жёлтые (деидентификация + Enterprise):

  • Внутренние регламенты
  • Шаблоны договоров
  • Аналитические отчёты

🟢 Зелёные (можно в облако):

  • Общие знания, справочная информация
  • Черновики без конфиденциальных данных
  • Обучающие материалы

🛠️ Как применить

Шаблон AI-политики (краткая версия)

Пример политики
1. Цель: Используем AI для автоматизации рутины 
   и повышения качества анализа.

2. Разрешённые инструменты:
   - ChatGPT Enterprise (для общих задач)
   - Локальная модель (для конфиденциальных)
   - n8n / Zapier (для автоматизации)

3. Запрещено:
   - Отправлять персональные данные в бесплатный ChatGPT
   - Использовать AI для финальных решений без проверки
   - Доверять AI юридические и финансовые расчёты без верификации

4. Процессы:
   - Рутина (< 50 000 ₽): AI автономно
   - Важные (> 50 000 ₽): AI + человек
   - Критичные (> 500 000 ₽): человек

5. Ответственность:
   - Сотрудник: соблюдение политики
   - Руководитель: проверка критичных решений
   - IT: техническая поддержка

6. Нарушения:
   - Первое: предупреждение + обучение
   - Повторное: дисциплинарная ответственность

Как внедрить политику

  1. Соберите команду: IT, юрист, HR, бизнес-репрезентанты
  2. Напишите черновик: Используйте шаблон выше
  3. Согласуйте с юристом: Соответствие 152-ФЗ, NDA
  4. Утвердите руководством: Приказ / распоряжение
  5. Обучите сотрудников: Не просто "прочитайте", а практика
  6. Проверяйте: Аудит раз в квартал

👨‍💻 Как делегировать разработчику

Шаблон задачи

"Нужна система контроля соблюдения AI-политики.

Требования:
1. Автоматическая проверка входных данных 
   (не содержат ли конфиденциальную информацию)
2. Блокировка запрещённых операций
3. Логирование всех AI-запросов
4. Отчётность для аудита
5. Интеграция с [Active Directory / SSO]

Политика (прикрепить): [файл]
Срок: [дата]"

✅ Чеклист принятия результата

  • Политика утверждена руководством?
  • Все сотрудники ознакомились и подписали?
  • Есть обучение (не просто "прочитайте")?
  • Технические ограничения настроены? (что нельзя отправить)
  • Есть процесс проверки и аудита?
  • Нарушения фиксируются и обрабатываются?

🎯 Главное из урока

AI-политика = правила дорожного движения. Без них — хаос и аварии. С ними — эффективное и безопасное использование. Политика должна быть понятной, практичной и обязательной. Не просто "запрещаем всё", а "разрешаем то, что безопасно, с контролем".
